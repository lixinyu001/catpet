## 1. 整体架构

### 1.1 单 HTML 文件结构

游戏采用单 HTML 文件架构，所有界面通过 JavaScript 动态渲染到 `#app` 容器中。

**HTML 骨架**（`src/index.html`）：

| 元素 | ID | 作用 |
|---|---|---|
| 应用根容器 | `#app` | 所有页面内容由 `render()` 注入此处 |
| Buff 显示栏 | `#buffBar` | 固定在顶部居中，显示激活的 buff 芯片 |
| Toast 容器 | `#toast-container` | 固定在顶部居中，所有 Toast 提示插入此处 |

### 1.2 脚本加载顺序

```html
<script src="js/config.js"></script>   <!-- 常量与配置 -->
<script src="js/systems.js"></script>  <!-- 天赋、buff 等系统函数 -->
<script src="js/state.js"></script>    <!-- 游戏状态、存档读写 -->
<script src="js/pet.js"></script>      <!-- 宠物相关函数 -->
<script src="js/battle.js"></script>   <!-- 战斗系统、自动开箱、日志 -->
<script src="js/economy.js"></script> <!-- 经济系统 -->
<script src="js/ui.js"></script>      <!-- UI 渲染与事件 -->
<script src="js/main.js"></script>     <!-- 初始化入口 -->
```

### 1.3 全局窗口状态

UI 模块在 `window` 上挂载若干状态变量，用于跨 render 调用保持界面状态：

| 变量 | 类型 | 作用 |
|---|---|---|
| `window._charSheet` | `'equipment' \| 'forge' \| 'socket'` | 角色页当前 sheet（任务 13/14，v2.0.3 新增 `socket` 打孔镶嵌 sheet） |

| `window._skillBookPage` | `Object<category, page>` | 技能书商店分页（任务 15） |

| `window._bloodOrbImplantPetId` | `petId \| undefined` | 血统植入弹窗目标宠物（任务 7） |
| `window._activitySheet` | `'treasure' \| 'tower' \| 'arena' \| 'advance' \| 'formation' \| 'skillbook' \| 'petcave'` | 活动页当前 sheet（v2.0.3 替换 `bloodline` 为 `advance`） |

### 2.2.1 导航红点提示系统（v3.1.0 新增）

> **v3.1.0 新增**：底部导航栏的 4 个主功能模块（人物装备、天赋、成就、宠物）新增红点角标，实时提示待处理事项。

**红点模块与触发条件**（`ui.js` `getNavBadge(tabId)`）：

| 模块 | tabId | 触发函数 | 判定逻辑 |
|---|---|---|---|
| **人物装备** | `character` | `checkEquipUpgradeable()` | 背包中存在综合评分优于当前穿戴的装备 |
| **天赋** | `talent` | `checkTalentUnspent()` | 当前激活方案中 `getTalentPoints() > 0`（剩余未分配天赋点） |
| **成就** | `achievement` | `checkAchievementClaimable()` | 存在已达成但未领取奖励的里程碑 |
| **宠物** | `pets` | `checkPetUnspentPoints()` | 当前上阵宠物中任意一只 `freeAttrPoints > 0` |

**红点显示样式**：

```html
<span class="inline-block w-2 h-2 bg-red-500 rounded-full ml-0.5 animate-pulse"></span>
```

- 位置：标签文字右侧 `ml-0.5`，紧凑布局
- 样式：红色圆点，脉冲动画（`animate-pulse`）
- 大小：`2x2`（宽高各 2 个 Tailwind 单位，约 8px）
- 嵌入方式：`ui.js:1596` 导航渲染时调用 `getNavBadge(t.id)`，若有待办则追加红点 HTML

**红点消失逻辑**：

- **人物装备**：穿戴背包中的更优装备后，红点自动消失
- **天赋**：分配完剩余天赋点后，红点自动消失  
- **成就**：领取完所有可领取奖励后，红点自动消失
- **宠物**：分配完所有上阵宠物的剩余属性点后，红点自动消失

**判定时机**：

- 导航栏重绘时（切换页面、功能状态变化）实时重新判定
- 不依赖缓存，始终读取最新状态（装备背包/天赋点数/成就领取状态/宠物属性点）

### 2.3 main-content（主内容区）

```html
<main class="flex-1 p-4 max-w-5xl mx-auto w-full space-y-4">
  <!-- 各页面具体内容 -->
</main>
```

最大宽度 `max-w-5xl`（约 64rem），居中布局，所有页面通用。

### 3.3 渲染流程

```js
function render() {
  try {
    const app = document.getElementById('app');
    switch (currentScreen) { /* 分发 */ }
    if (viewingPetId) app.innerHTML += renderPetDetailModal();        // 宠物详情弹窗
    if (window._bloodOrbImplantPetId) app.innerHTML += renderBloodOrbImplantModal();  // 血统植入弹窗
    bindEvents();           // 占位空函数，子类可扩展
    renderBuffBar();        // 渲染顶部 buff 栏
  } catch(e) {
    // 渲染出错时显示"返回主页"按钮
  }
}
```

### 3.4 路由切换函数 `navigateTo(screen)`

`ui.js` 第 6319 行：

```js
function navigateTo(screen) {
  currentScreen = screen;
  render();
  if (screen === 'main' && liveBattle) {
    setTimeout(() => renderBattleArena(), 50);  // 主页时延迟刷新战斗场景
  }
}
```

## 4. 主页（main）

`renderMainScreen()`（`ui.js` 第 1311 行）是游戏入口页，集成战斗、进度、掉落日志等功能。

### 4.1 顶部状态栏

- 删除存档按钮（红色边框）→ `confirmDeleteSave()`
- 游戏标题 `暗影纪元`（字体 `font-fantasy`，金色）
- 玩家等级 + 转生星标（`★`×rebirth）
- 💎 钻石、🪙 金币、战力数字
- 🎁 兑换码按钮 → `openRedeemModal()`

### 4.2 新手礼包

仅当 `team.length === 0 && !G.newPlayerGiftClaimed` 时显示。包含动画 `animate-gift-glow`，点击 `claimNewPlayerGift()` 领取 3 只初始宠物（小史莱姆/小哥布林/小精灵）+ 1000 金币 + 2 颗宠物蛋。

### 4.3 战斗面板

| 元素 | 说明 |
|---|---|
| 地图选择 `<select id="mapSelect">` | 列出 `MAPS` 中 `player.level >= minLv - 5` 的地图，每选项含地图名+当前层数+等级范围 |
| 关卡进度 | 第 X/Y 层 + 阶段标识：`小怪 n/m` / `⭐ 精英怪` / `👑 BOSS` / `✅ 已通关` |
| 小怪进度条 | `progress-bar` + `progress-fill`（青蓝渐变） |
| 开始/停止挂机按钮 | `toggleAutoBattle()`，按钮文本随 `autoBattleInterval` 切换 |
| 速度按钮组 | 1x / 2x / 4x / 8x / 16x / 32x（解锁条件见 §20） |
| 状态文字 | `挂机中...` 或 `已停止` |
| 自动开箱按钮 | `toggleAutoChests()`，颜色随状态切换 |
| 经验进度条 | `G.player.exp / G.player.expToNext` |

### 4.4 战斗竞技场

```html

    <!-- 由 renderBattleArena() 注入 -->
```

详细结构见 §14。

### 4.5 出战宠物

3 列网格展示当前队伍，每个槽位卡片显示：

- 品质标签（按 `pet.rarity` 取色）
- 站位标签（前/中/后）
- 宠物名 / 种族·等级 / 成长值
- 战力

底部阵法选择：`<select>` 列出所有已学阵法 + "不使用阵法"选项，激活阵法时显示效果倍率与描述，并提供"📜 阵法管理"按钮跳转 formation 页。

### 4.6 未开宝箱区

仅当 `G.chests.length > 0` 时显示。每个宝箱点击调用 `openChestFromBag(id)`，并提供"一键开启"按钮 `openAllChests()`。

### 4.7 掉落日志（任务 25）

```html
<div class="bg-card border border-game rounded-xl p-4">
  <h2 class="font-bold text-lg mb-2">📦 掉落日志</h2>
  <div class="battle-log scrollbar-thin" id="battleLog"
       style="display:flex;flex-direction:column-reverse;">

```

实现：`G.battleLog.filter(l => l.type === 'loot').slice(-40).reverse()`。新条目通过 `flex-direction:column-reverse` 自动滚到底部。无掉落时显示"等待掉落..."。

> **任务 25**：原为"战斗日志"，已改为"掉落日志"，仅展示 loot 类型记录，避免与战斗过程信息混淆。

## 24. 存档管理

### 24.1 存档键

存档唯一标识：`localStorage` 键名 `'shadow_era_save'`，值为 `JSON.stringify(G)`。

### 24.2 保存 `saveGame()`

`state.js` 第 80 行：

```js
function saveGame() {
  if (window.__DELETING_SAVE__) return;  // 删除存档时跳过自动保存
  try { localStorage.setItem('shadow_era_save', JSON.stringify(G)); } catch(e) {}
}
```

### 24.3 读取 `loadGame()`

`state.js` 第 86 行：

- 读取 `localStorage.getItem('shadow_era_save')`
- JSON 解析失败或结构异常（无 `player` 字段）→ 抛错并清空存档
- 合并默认状态 `DEFAULT_STATE` 与读取数据
- 执行一系列迁移函数：`migratePetAttributes` / `migrateInnateSkills` / `migrateGemSlots` / `restoreEquipmentAffixes` / `migrateTreasureMaps` / `migratePetRarity`
- 调用 `checkDailyReset()` / `checkArenaWeeklyReset()` 重置每日/每周数据

### 24.4 自动保存

`main.js` 第 39 行定时保存：

```js
setInterval(() => { saveGame(); }, 30000);  // 每 30 秒
```

`main.js` 第 42 行关闭/刷新前保存：

```js
window.addEventListener('beforeunload', () => {
  stopLiveBattle();
  if (autoBattleInterval) clearInterval(autoBattleInterval);
  if (window.__DELETING_SAVE__) return;  // 删除时跳过
  saveGame();
});
```

### 24.5 删除存档 `confirmDeleteSave()`

`ui.js` 第 1102 行，**二次确认 + 防刷新自动保存**：

```js
function confirmDeleteSave() {
  if (!confirm('⚠️ 确定要删除存档吗？\n\n所有宠物、装备、进度将永久丢失，无法恢复！')) return;
  if (!confirm('⚠️ 最后确认：真的要删除所有进度从零开始吗？\n\n此操作不可撤销！')) return;
  // 设置标志位，防止 beforeunload 事件重新保存
  window.__DELETING_SAVE__ = true;
  try { localStorage.removeItem('shadow_era_save'); } catch(e) {}
  showToast('存档已删除，正在重新开始...', 'success');
  setTimeout(function() { location.reload(); }, 800);  // 800ms 后刷新
}
```

**关键机制**：设置 `window.__DELETING_SAVE__ = true` 后，`saveGame()` 与 `beforeunload` 监听器都会跳过保存，防止删除的存档被刷新前的自动保存覆盖回来。

### 24.6 存档预检查

`index.html` 第 553 行 `precheck()` 在游戏脚本加载前执行：

```js
(function precheck() {
  try {
    var raw = localStorage.getItem('shadow_era_save');
    if (raw) {
      var parsed = JSON.parse(raw);
      if (!parsed || typeof parsed !== 'object' || !parsed.player) {
        localStorage.removeItem('shadow_era_save');  // 结构异常，清除
      }
    }
  } catch(e) {
    try { localStorage.removeItem('shadow_era_save'); } catch(e2) {}
  }
})();
```

### 24.7 手动清空

`window.clearSaveAndReload()`（`index.html` 第 546 行）：

```js
window.clearSaveAndReload = function() {
  try { localStorage.removeItem('shadow_era_save'); } catch(e) {}
  try { localStorage.clear(); } catch(e) {}
  location.reload();
};
```

错误处理 UI 中的"清除存档并刷新"按钮调用此函数。

## 25. 错误处理

### 25.1 错误捕获流程

游戏启动时 `window.__GAME_LOADED__ = false`，`main.js` 第一行设为 `true`，若执行到末尾仍为 `false`，则进入错误恢复 UI。

### 25.2 三层错误处理

#### 第一层：`window.addEventListener('error')`（`index.html` 第 569 行）

```js
window.addEventListener('error', function(e) {
  if (window.__GAME_LOADED__) return;
  try { localStorage.removeItem('shadow_era_save'); } catch(e2) {}
  // 显示"游戏加载失败"红色 UI，含错误消息与行号
});
```

UI 包含：
- ⚠️ 大图标
- "游戏加载失败"红色标题
- 错误消息（`e.message`）+ 行号（`e.lineno`）
- "存档已自动清除"黄色提示
- "清除存档并刷新"按钮（`window.clearSaveAndReload`）
- "清除所有缓存并刷新"按钮（`localStorage.clear(); location.reload()`）
- "或按 Ctrl+Shift+R 强制刷新"提示

#### 第二层：`main.js` 初始化 try/catch

```js
(function init() {
  try {
    loadGame();
    checkDailyReset();
    /* ... */
    render();
  } catch(e) {
    // 重置为默认状态，重新保存并渲染
    G = JSON.parse(JSON.stringify(DEFAULT_STATE));
    saveGame();
    render();
  }
})();
```

#### 第三层：脚本解析失败兜底（`index.html` 第 592 行）

```html
<script>
if (!window.__GAME_LOADED__) {
  try { localStorage.removeItem('shadow_era_save'); } catch(e) {}
  // 显示"游戏脚本解析失败" UI
}
</script>
```

若 `main.js` 因语法错误（如不支持的 `?.` 可选链）未能执行，`__GAME_LOADED__` 保持 `false`，此兜底脚本会显示"游戏脚本解析失败"UI，提示用户升级浏览器（Chrome/Edge 最新版）。

### 25.3 渲染错误处理

`render()` 函数（`ui.js` 第 989 行）try/catch 包裹：

```js
} catch(e) {
  console.error('Render error:', e);
  const errMsg = (e && e.message) ? e.message : String(e || '未知错误');
  document.getElementById('app').innerHTML = '<div>...页面渲染出错...<button onclick="currentScreen=\'main\';render()">返回主页</button></div>';
}
```

任何 render 异常都会被捕获，避免白屏，并提供"返回主页"逃生按钮。
