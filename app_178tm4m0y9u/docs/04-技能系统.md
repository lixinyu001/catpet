## 一、技能总览与分类

游戏技能分为 **5 大类**，由 `type` 字段标识，类型图标与标签如下：

| 类型 | type 字段 | 图标 | 标签 | 是否占技能格 | 数量 | 来源 |
|------|-----------|------|------|--------------|------|------|
| 血统 | `bloodline` | 👑 | 血统 | ❌ 不占格 | 27 | `BLOODLINE_SKILLS` |
| 主动 | `active` | ⚔️ | 主动 | ✅ 占格 | 75 | `ACTIVE_SKILLS` |
| 被动 | `passive` | 🛡️ | 被动 | ✅ 占格 | 约 75 | `PASSIVE_SKILLS` |
| 光环 | `aura` | ✨ | 光环 | ✅ 占格 | 18 | `AURA_SKILLS` |
| 天生 | — | ★ | 天生 | ✅ 占格 | — | 派生自主动/被动/光环 |

> **天生技能**并非独立类型，而是宠物生成时从图鉴 `innateSkills` 必带 + 技能库随机抽取得到的技能，会被标记 `isInnate: true`，其底层 `type` 仍为 `active`/`passive`/`aura` 之一。血统技能不占技能格，独立于 6 格技能栏之外。

**技能合并入口**（`config.js`）：

```js
const ALL_SKILLS = [...ACTIVE_SKILLS, ...PASSIVE_SKILLS, ...AURA_SKILLS];
function getSkillById(id) {
  return ALL_SKILLS.find(s => s.id === id) || BLOODLINE_SKILLS.find(s => s.id === id);
}
```

### 技能分级（tier）

被动 / 光环技能按 `tier` 分 3 级，命名规则 `原名` / `高级原名` / `超级原名`，id 后缀 `_2` / `_3`：

| tier | 前缀 | 颜色 | 标签 |
|------|------|------|------|
| 1 | 初级 | `#22c55e` 绿 | 初级 |
| 2 | 高级 | `#a855f7` 紫 | 高级 |
| 3 | 超级 | `#f59e0b` 金 | 超级 |

`getSkillTier(id)`：以 `_3` 结尾返回 3，`_2` 结尾返回 2，否则 1。`getSkillBaseId(id)` 去掉 `_(2|3)` 后缀得到基础 id，用于"同名技能去重"。

### 2.1 主动技能描述格式

主动技能 `desc` 字段在加载时统一追加触发概率与品质标签，最终格式为：

```
[品质名] 【伤害类型·目标·特性】技能简述
伤害公式：...
触发概率信息
⏱ 触发概率：基础X%（类型上限Y%，受技能触发加成提升，多技能独立判断，最多3个主动技能生效）
```

**描述构成要素**：

1. **品质前缀**：`[普通]` / `[优秀]` / `[稀有]` / `[史诗]` / `[传说]`
2. **【标签】**：格式 `【伤害类型·目标·特性】`，例如：
   - `【物理·单体】`、`【法术·单体·必中】`、`【物理·单体·高暴击】`
   - `【物理·群体·秒3】`、`【法术·群体·秒4·控制】`、`【法术·全体·吸血】`
   - `【治疗·单体·冷却】`、`【增益·自身·反击·冷却】`、`【控制·全体·嘲讽】`、`【物理·单体·必暴·融合限定】`
3. **伤害公式**：明示 power 来源（攻击力 / 灵力 / 血量 / 魔法 / 防御 / 速度）与系数
4. **触发概率**：附加于描述末尾

**伤害类型判定**（`config.js` `isSkillMagic` / `isSkillSupport`）：

| 类型 | 判定条件 | 基础触发 | 上限 |
|------|----------|----------|------|
| 法术 | 元素 ∈ {冰,雷,暗,光,毒} 或 辅助类 | 20% | 40% |
| 物理 | 元素 ∉ 法术元素集（火/水/土/无/混沌/时空/奥术等） | 8% | 15% |
| 辅助 | category ∈ {single_heal, aoe_heal, single_buff, aoe_buff} | 25% | 50% |

### 2.2 伤害公式与暴击

**基础伤害公式**（`battle.js` `executeActiveSkill`，第 898-908 行）：

```
真实伤害（trueDmg）:
  rawDmg = power × dmgPct × skillDmgMult × (1 + skillDmgBonus)

普通伤害:
  rawDmg = power × dmgPct × skillDmgMult × (1 + skillDmgBonus) − def × (0.6 − ignoreDefPct)

最终:
  rawDmg = round(rawDmg × random(0.90, 1.10))   // ±10% 波动
  若暴击: rawDmg = round(rawDmg × critMult)
  dmg = max(1, rawDmg)
```

**参数说明**：

| 参数 | 含义 | 来源 |
|------|------|------|
| `power` | 攻击力来源 | 法术→`getPetSpiritPower`（灵力）；物理→`getPetAtkPower`（攻击力） |
| `dmgPct` | 伤害系数 | 技能配置 |
| `skillDmgMult` | 被动技能伤害倍率 | `1 + passives.skillDmg` |
| `skillDmgBonus` | 天赋法术穿透 | `stats.skillDmg`（来自 `pet_skill_dmg` 天赋） |
| `def` | 目标防御 | 含 `defReduced` 减防效果：`def × (1 − defReduced)` |
| `ignoreDefPct` | 无视防御比例 | 技能 `ignoreDefPct`（默认 0） |

**暴击公式**：

```
critChance = 0.10 + passives.critRate + skill.bonusCrit + stats.critRate(玩家暴击率20%)
critMult   = 1.8 + passives.critDmg + stats.critDmg
```

> 注：技能描述中写"暴击伤害×1.5"为文案表述，实际 `critMult` 基础值为 **1.8**（`battle.js` 第 883 行）。

**多段伤害**（`hits` 字段）：

- 每击独立判定暴击；
- 递减系数 `decayPerHit`（如千刀斩 `0.20`）：第 i 击（从 0 起）倍率 = `(1 − decayPerHit)^i`；
- 无 `decayPerHit` 时每击等倍。

**斩杀**（`executeThreshold` / `executeMult`）：目标当前血量 < 最大血量 × `executeThreshold` 时，伤害 × `executeMult`。

### 2.2.1 AOE 伤害系数平衡调整（v3.1.0）

> **v3.1.0 调整**：为优化战斗节奏平衡，下调 AOE 技能伤害系数，提升单体技能相对价值。

**调整规则**：

| 技能类型 | 调整比例 | 判定标准 |
|---------|---------|---------|
| 法术类 AOE | ×0.8（-20%） | `desc` 标注【法术】或 `powerAttr: 'mp'`（魔法系） |
| 物理类 AOE | ×0.9（-10%） | `desc` 标注【物理】或 `powerAttr` 为 `hp/hpDiff/def/spd`（血量/血量差/防御/速度系） |
| 单体及非伤害技能 | 不变 | `category` 为 `single_atk/single_heal/single_buff/aoe_heal/aoe_buff` 或无 `dmgPct` |

**涉及技能**（共 28 个，`skill_data.js` 已同步更新 `dmgPct` 和 `desc`）：

- **法术 AOE（13个）**：暴风雪(0.6→0.48)、闪电链(1.1→0.88)、暗影爆发(1.0→0.80)、毒雾(0.5→0.40)、群体冰冻(0.6→0.48)、催眠术(0.4→0.32)、奔雷咒(1.25→1.00)、蝶潮(0.45→0.36)、佛行震地(0.5→0.40)、雷霆万钧(0.55→0.44)、奥术爆发(0.80→0.64)、魔能弹幕(0.60→0.48)、法力风暴(0.50→0.40)。
- **物理 AOE（15个）**：陨石天降(0.8→0.72)、烈焰风暴(0.85→0.765)、地裂术(1.5→1.35)、地震术(0.7→0.63)、烈焰冲撞(1.2→1.08)、碧水滔天(0.55→0.495)、泰山压顶(1.35→1.215)、搜魂斩(0.9→0.81)、混元一击(2.0→1.80)、灭世之炎(2.2→1.98)、血池爆裂(0.30→0.27)、生命洪流(0.35→0.315)、守护之怒(2.00→1.80)、闪电突袭(2.00→1.80)、影分身(2.50→2.25)。

> **设计意图**：通过差异化调整比例（法术-20% > 物理-10%），避免群体技能过强导致单体技能价值被掩盖，同时保持法术 AOE 相对更高的爆发预期。

### 3.2 被动技能效果加权

```js
function getPassiveEffects(pet) {
  const skills = getAllSkills(pet);
  const effects = {};
  skills.filter(s => s.type === 'passive' && s.effect).forEach(s => {
    Object.entries(s.effect).forEach(([k, v]) => {
      if (effects[k] === undefined || v > effects[k]) effects[k] = v;
    });
  });
  return effects;
}
```

> **注意**：同 baseId 不同 tier 的被动只生效最高 tier（`getEffectiveSkills` 标记 `isEffective`），低 tier 显示为"未生效"（半透明）。

**光环效果叠加**（`pet.js` `getAuraEffects` 第 575 行）：扫描全队所有光环技能，**同键取最大值**（非累加）：

```js
function getAuraEffects(team) {
  const effects = {};
  team.forEach(pet => {
    getAllSkills(pet).filter(s => s.type === 'aura').forEach(s => {
      Object.entries(s.effect).forEach(([k, v]) => {
        if (!effects[k] || v > effects[k]) effects[k] = v;
      });
    });
  });
  return effects;
}
```

**光环释放规则**（`battle.js` `tryExecuteAuraSkills` 第 562 行）：

1. **无冷却且未生效时当前回合必定释放**：在主动技能检查之前执行，**不占用主动技能触发次数**；
2. **已生效光环避免重复释放**：通过 `liveBattle.activeAuras[skill.id]` 标记，已生效则跳过；
3. **仅处理无冷却光环**：`if (skill.cd && skill.cd > 0) return;`（当前所有光环 `cd=0`）；
4. **释放动作**：调用 `executeAuraSkill` 显示日志、动画、记录生效状态。

**光环释放流程**：

```
executePetAction(pet)
  → tryExecuteAuraSkills(pet)        # 1. 光环（100%释放，不占主动次数）
    → 遍历无冷却光环，跳过已生效
    → 标记 activeAuras[skill.id]=true
    → executeAuraSkill：公告+日志+动画
  → tryUseActiveSkill(pet)            # 2. 主动技能（概率触发，最多3个）
  → executeNormalAttack(pet)          # 3. 普攻
```

**光环释放展示**：紫色公告 `✨ {宠物} 释放光环【{光环名}】：全队攻击力+15%`，并对自身播放攻击动画。

> **注意**：光环的属性加成通过 `getAuraEffects` 被动叠加到全队属性上（在 `getPetAtkPower` / `getPetSpiritPower` 中读取 `auras.teamAtk` 等）；`executeAuraSkill` 仅负责显示性释放动作。

## 六、天生技能系统

### 6.1 必带机制

天生技能来源于宠物图鉴 `dex.innateSkills`（`config.js` PET_DEX 定义），生成时**必带**（`pet.js` `generatePetBase` 第 112-120 行）：

```js
if (dex.innateSkills && dex.innateSkills.length > 0) {
  dex.innateSkills.forEach(function(sid) {
    var sk = ALL_SKILLS.find(function(s) { return s.id === sid; });
    if (sk && !usedBaseIds.has(getSkillBaseId(sid))) {
      innateSkills.push({ ...sk, isInnate: true });
      usedBaseIds.add(getSkillBaseId(sid));
    }
  });
}
```

- 按 baseId 去重，避免同 baseId 不同 tier 重复；
- 标记 `isInnate: true`，UI 中显示"天生"（紫色标签）。

### 6.2 技能库随机抽取

每个宠物有固定技能库 `getPetSkillLib(name)`（`config.js` 第 351 行），生成时从技能库随机补充：

1. 必带天生技能；
2. 候选池按特长+品阶累加（`SPEC_SKILL_POOL` / `_T3` / `_T4` / `_T5`）；
3. 按 baseId 去重；
4. 用宠物名生成确定性种子，**同名宠物技能库永远一致**；
5. 从池中按种子确定性选取补充技能至 `maxSkills`。

### 6.3 总技能数随机（至少1，最多6）

```js
// 从技能库随机抽取 extraCount 个
```

## 十、技能继承（融合）

宠物融合时技能继承逻辑（`pet.js` 第 716-750 行）：

1. **收集两宠所有普通技能**（不含血统）：`getNormalSkills` 过滤 `type !== 'bloodline'`；
2. **按 baseId 聚合**：统计每 baseId 出现次数与最高 tier；
3. **继承规则**：

| 情形 | 概率 | 结果 |
|------|------|------|
| 相同技能（两只都有，`count ≥ 2`）且 maxTier<3 | **50%** | 升级 tier+1（`upgradedId`，上限 3） |
| 相同技能未升级 / 不同技能 | **60%** | 保留原技能 |
| 否则 | — | 不保留 |

4. **保底**：若结果为空，至少保留 1 个技能（随机选）；
5. **不限数量**：融合后技能可超过 6 格（`newPet.innateSkills = resultSkills.map(s => ({...s, isInnate: true}))`）。

**升级 id 计算**：

```js
const newTier = Math.min(entry.maxTier + 1, 3);
const upgradedId = s.id.replace(/(_\d)?$/, newTier > 1 ? '_' + newTier : '');
```

> **注意**：融合限定宠物（`generateFusionOnlyPet`）的技能不参与继承，直接使用图鉴定义的独有天生技能。

## 十二、战斗力计算技能加成

战斗力由 6 部分组成，技能加成为第 6 部分（`pet.js` `getCombatPowerBreakdown` 第 453 行）：

```
总战力 = round(statCp × skillMult)
技能加成战力 = round(statCp × (skillMult − 1))
```

**`skillMult` 由 `getSkillPowerMultiplier(pet)` 计算**（第 376 行），基础值 1.0，累加如下：

### 12.1 主动技能加成

```
avgDmgPct = totalDmgPct / activeCount
mult += max(0, avgDmgPct − 1.0) × 0.5       // 平均伤害系数超 1.0 部分 ×0.5
mult += min(activeCount, 8) × 0.03           // 技能数量每个+3%（上限8个）
```

**totalDmgPct 累加规则**：

| 技能类型 | 贡献 |
|----------|------|
| 含 dmgPct | `dmgPct` + 多段额外 `dmgPct × (hits − 1) × 0.6` |
| 含 healPct | `healPct × 0.4` |
| 纯 buff | `0.4` |

### 12.2 被动技能加成（按效果类型加权）

| 效果键 | 权重 | 说明 |
|--------|------|------|
| atkBonus | ×1.0 | 攻击加成（直接输出） |
| critRate | ×0.6 | 暴击率（暴击 1.8 倍） |
| critDmg | ×0.3 | 暴击伤害 |
| skillDmg | ×0.8 | 技能伤害（直接影响主动输出） |
| ignoreDef | ×0.5 | 无视防御 |
| doubleChance | ×0.6 | 连击/追击 |
| tripleChance | ×0.8 | 三连 |
| skillDouble | ×0.5 | 法连 |
| vampPct | ×0.4 | 吸血（续航） |
| firstStrike | ×0.2 | 先手 |
| dmgReduce | ×0.2 | 减伤（生存，低权重避免坦克虚高） |
| regenPct | ×0.3 | 再生 |
| parryChance | ×0.2 | 招架 |
| dodgeBonus | ×0.3 | 闪避 |
| reflectPct | ×0.2 | 反震 |
| antiCrit | ×0.1 | 抗暴击 |

### 12.3 异化品质额外加成

```
if (pet.rarity === 'special') mult ×= 1.2;   // 异化品质额外 ×1.2
```

异化宠物在生成时 1.5% 概率判定（`generatePetBase`），融合时 3% 概率判定，资质 ×1.2、成长 ×1.3（上限 3.5）。

### 二、图鉴库系统建立（需求12）

v2.0.5 建立**统一技能库**架构，所有宠物/怪物/技能数据集中管理于 `config.js`，作为游戏的核心数据图鉴库。

#### 1. SKILL_LIBRARY 统一技能库

`config.js` 顶部新增统一技能库：

```js
// 需求12：统一技能库（SKILL_LIBRARY）- 商城/宠物/怪物的技能信息均来自此库
const ALL_SKILLS = [...ACTIVE_SKILLS, ...PASSIVE_SKILLS, ...AURA_SKILLS];
// 技能库完整索引（含血统技能），统一查询入口
const SKILL_LIBRARY = [...ALL_SKILLS, ...BLOODLINE_SKILLS];

function getSkillById(id) {
  // 优先从完整技能库查询（包含血统技能）
  return SKILL_LIBRARY.find(s => s.id === id) || ALL_SKILLS.find(s => s.id === id);
```

| 项目 | v2.0.5 前 | v2.0.5 后 |
|:---|:---|:---|
| `getSkillById` 查询范围 | `ALL_SKILLS` + fallback `BLOODLINE_SKILLS` | **优先查询 `SKILL_LIBRARY`**（含血统） |

`config.js` 顶部新增架构说明注释：

```js
// ===== config.js : 常量、地图、副本、技能库等数据定义 =====
// 需求12：图鉴库系统架构说明
// 本文件是游戏的核心数据图鉴库，所有宠物/怪物/技能数据均集中管理于此：
//   1. PET_DEX (宠物图鉴)：所有宠物的种族/资质/成长/专长/天生技能，getPetDex(name) 统一查询
//   2. SKILL_LIBRARY (技能库)：所有主动/被动/光环/血统技能，getSkillById(id) 统一查询
```

| 图鉴库 | 数据 | 查询入口 | 修改位置 |
|:---|:---|:---|:---|
| **PET_DEX** | 所有宠物的种族/资质/成长/专长/天生技能 | `getPetDex(name)` | `config.js` |

- **单一数据源**：所有宠物与怪物数据均**引用自 PET_DEX**，修改宠物数据只需改图鉴；

#### 1. 排序规则

`SKILL_BOOK_SHOP.active`（`config.js`）从原 `ACTIVE_SKILLS.slice().sort()` 改为：

```js
active: ACTIVE_SKILLS.slice().sort(function(a, b) {
  // 按品质正序：common → uncommon → rare → epic → legend
  var qualityOrder = ['common', 'uncommon', 'rare', 'epic', 'legend'];
  var qa = qualityOrder.indexOf(a.quality || 'common');
  var qb = qualityOrder.indexOf(b.quality || 'common');
  if (qa !== qb) return qa - qb;
  // 同品质按名称排序
  return a.name.localeCompare(b.name, 'zh-CN');
});
```
