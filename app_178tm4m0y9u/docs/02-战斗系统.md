## 1. 系统概述

战斗系统以 `liveBattle` 全局对象为核心状态机，由以下源文件协作实现：

| 文件 | 职责 |
| --- | --- |
| `src/js/battle.js` | 战斗核心逻辑：回合调度、伤害计算、技能释放、波次推进、奖励发放 |
| `src/js/config.js` | 静态配置：地图、Boss、怪物种族技能、宝箱掉落、天赋掉落、副本定义 |
| `src/js/ui.js` | 战斗渲染：`renderBattleArena()`、技能释放公告 `showSkillCastAnnouncement()`、速度/挂机按钮 |
| `src/index.html` | CSS 样式：`.battle-arena`、`.battle-log`、`.speed-btn`、各类动画关键帧 |

战斗整体流程为：`startLiveBattle()` → `spawnMonster()` → `buildTurnQueue()` → `scheduleNextTurn()` → `executeTurn()` → 行动者执行 → `nextFromQueue()` → 循环直至一方全灭 → `advanceWaveOrEndBattle()` → `processWaveCleared()`。

## 2. 回合制战斗机制

### 2.1 回合定义

每一回合（round）由 `buildTurnQueue()` 构建，将所有存活的玩家宠物与所有存活怪物加入行动队列，然后按速度降序排序。回合开始时打印日志：

```
── 第 N 回合 ──
```

### 2.2 行动执行流程（executeTurn）

`executeTurn()` 是核心调度函数，逻辑如下：

1. 若 `liveBattle.animating === true`，直接返回（防止重入）。
2. 若 `turnQueue` 为空：
   - 调用 `applyTurnRegen()` 处理回合结束再生（持续恢复、Hot、DoT、buff 回合递减、技能冷却递减）。
   - 若所有怪物已死（DoT 击杀），记录击杀日志后调用 `advanceWaveOrEndBattle()`。
   - 若所有宠物已死，调用 `processBattleDefeat()`。
   - 否则调用 `buildTurnQueue()` 开启新回合。
3. 从 `turnQueue` 队首 `shift()` 取出当前行动者。
4. 根据行动者类型调用 `executePetAction(pet)` 或 `executeMonsterAction(monsterIdx)`。
5. 行动完成后由行动函数内部调用 `nextFromQueue()` 推进下一名行动者。

### 2.3 宠物行动（executePetAction）

宠物行动按以下顺序判定：

| 优先级 | 判定项 | 说明 |
| --- | --- | --- |
| 1 | 状态阻止 | `stunned`/`frozen`/`sleeping`/`rooted` 任一 > 0 则跳过本回合，状态回合数 -1 |
| 2 | 光环技能 | `tryExecuteAuraSkills(pet)`：无冷却且未生效的光环当前回合必释放 |
| 3 | 主动技能 | `tryUseActiveSkill(pet)`：按概率触发，受魔法值与冷却限制 |
| 4 | 普通攻击 | `executeNormalAttack(pet)`：默认行动 |

### 2.4 怪物行动（executeMonsterAction）

| 优先级 | 判定项 | 说明 |
| --- | --- | --- |
| 1 | 死亡检查 | HP ≤ 0 跳过 |
| 2 | 状态阻止 | 同宠物（眩晕/冻结/睡眠/定身） |
| 3 | 主动技能 | 50% 概率使用 `monster.skills` 中随机一个主动技能（无需魔法值） |
| 4 | 普通攻击 | 50% 概率，按阵位顺序攻击首个存活宠物 |

## 3. 速度排序系统（buildTurnQueue）

`buildTurnQueue()` 负责构建每回合的行动顺序：

```javascript
actors.sort(function(a, b) { return b.speed - a.speed; });
```

### 3.1 行动者来源

| 行动者类型 | 速度来源 |
| --- | --- |
| 宠物 | `getPetStats(pet).速度`（默认 10） |
| 怪物 | `monster.speed`，生成时设为 `Math.floor((level || 1) * 2 + 10)` |

### 3.2 排序规则

- **降序排列**：速度高的先出手。
- 同速时由 JavaScript `Array.sort` 稳定性决定（V8 引擎稳定排序，按入队顺序执行）。
- 已死亡的单位不会入队（HP > 0 才加入）。
- 每回合重新构建队列（`liveBattle.round++`）。

## 4. 战斗时序机制

### 4.1 单位动画隔离

战斗采用「仅当前出手单位动画，其他不动」的设计：

- 全局 `liveBattle.animating` 标志位：行动开始置 `true`，行动结束（`nextFromQueue`）置 `false`。
- `executeTurn()` 在 `animating === true` 时直接返回，避免动画期间被打断。
- 攻击/技能动画通过 `setTimeout` 延迟推进（普攻 400ms，技能 400ms）。

### 4.2 回合延迟

`scheduleNextTurn()` 通过 `setTimeout` 在 `getTurnDelay()` 毫秒后触发下一次 `executeTurn()`。延迟数值见 [第 17 节 战斗速度设置](#17-战斗速度设置)。

### 4.3 关键时序常量

| 时序点 | 延迟 | 代码位置 |
| --- | --- | --- |
| 普攻/技能后推进下一行动 | 400ms | `setTimeout(() => nextFromQueue(), 400)` |
| 连击追加攻击 | 400ms | 二段/三段连击 |
| 自动战斗下一波生成 | 1000ms（主线）/ 1500ms（失败） | `battleSpawnTimer` |
| 技能释放公告显示 | 1500ms | `showSkillCastAnnouncement` |

## 6. 多怪物波次系统

### 6.1 波次生成（generateMonsterWave）

`generateMonsterWave(mapId)` 根据当前关卡阶段 `prog.phase` 生成多怪物波次：

| 阶段 | 怪物组成 | 总数 |
| --- | --- | --- |
| `mobs`（小怪关） | 3–6 个小怪 | `randomInt(3,6)` |
| `elite`（精英关） | 4 小怪 + 2 精英（精英最后出场） | 6 |
| `boss`（Boss 关） | 3 小怪 + 2 精英 + 1 Boss（Boss 最后出场） | 6 |

> **v2.5.0 修复（需求6）**：已通关关卡（`phase === 'cleared'`）此前在重新进入手动/挂机战斗时仅生成 1 只 Boss，导致敌方阵容不一致。v2.5.0 起统一 `generateMonsterWave` 行为：**所有 boss 关卡（无论是否通关、无论手动战斗还是挂机战斗）怪物阵容固定为 1 Boss + 2 精英 + 3 普通小怪 = 6 个敌方单位**。修复后进入已通关关卡的手动/挂机战斗，敌方单位数量均为 6 只，构成符合 1Boss+2精英+3小怪。

### 6.2 上限限制

```javascript
if (monsters.length > 6) monsters = monsters.slice(0, 6);
```

**最多 6 个敌人同时上场**，超出部分截断。

### 6.3 关卡进度推进（processWaveCleared）

每清空一波怪物后，根据本波中「最高等级类型」推进关卡进度（优先级 boss > elite > mob）：

| 当前阶段 | 击杀后行为 |
| --- | --- |
| `mobs` | `mobsKilled++`；达到 `mobsNeeded`（8–12 随机）后切到 `elite` 阶段 |
| `elite` | 若 `currentLayer >= map.layers` → 切到 `boss` 阶段；否则 `currentLayer++`，重置为 `mobs` 阶段 |
| `boss` | `phase = 'cleared'`，解锁下一张地图 |

### 6.4 特殊战斗例外

副本与藏宝图战斗**不使用波次系统**，保持单怪模式：

```javascript
if (liveBattle && (liveBattle.isDungeon || liveBattle.isTreasure)) {
  monsters = [getLayerMonster(G.player.currentMap)];
}
```

## 7. 怪物生成机制（getLayerMonster）

`getLayerMonster(mapId, forceType)` 是怪物生成的核心函数。

### 7.1 类型判定

- 若传入 `forceType`，直接使用该类型（用于波次系统生成混合波次）。
- 否则按 `prog.phase` 判定：`mobs` → `mob`，`elite` → `elite`，`boss` → `boss`。

### 7.2 倍率与配置

| 类型 | `hpMult` | `atkMult` | `defMult` | 配置来源 |
| --- | --- | --- | --- | --- |
| mob（小怪） | 5 | 5 | 1.2 | `map.monsters` + `MONSTER_RACE_SKILLS[race].mob` |
| elite（精英） | 12 | 8 | 1.6 | `map.eliteName` + `MONSTER_RACE_SKILLS[race].elite` |
| boss（首领） | 24 | 12 | 2.2 | `MAP_BOSSES[mapId]`（固定配置） |

> 代码注释中描述「怪物强度强化：小怪×5，精英×4，Boss×3（整体强度提升）」为整体强化概念；实际数值以上表 `hpMult/atkMult` 为准。

### 7.3 属性计算公式

连续成长曲线（避免跨图难度跳跃）：

```
lvScale = 1 + lv × 0.012
baseHp  = floor((40 + lv × 20) × hpMult × lvScale) ± 5% 随机
baseAtk = floor((4 + lv × 3.8) × atkMult × lvScale) ± 10% 随机
def     = floor(lv × 2.0 × defMult)
```

其中 `lv = randomInt(map.minLv, map.maxLv)`。

### 7.4 起源草地削弱

地图 1（起源草地）整体强度削弱一半：

```javascript
if (mapId === 1) {
  baseHp  = Math.floor(baseHp * 0.5);
  baseAtk = Math.floor(baseAtk * 0.5);
}
```

### 7.5 速度生成

怪物生成后若未设置 `speed`，则：

```
speed = floor((level || 1) × 2 + 10)
```

### 7.6 五维属性生成（v2.6.0 新增）

> **v2.6.0 新增（需求16）**：所有怪物在生成时同步生成五维属性（力量/敏捷/体质/耐力/魔力），复用与宠物同一套五维属性计算体系（`generateMonsterFiveDim` 函数）。按种族差异化加点：物理系（龙/恶魔）主加力量+耐力，法术系（天使/精灵）主加魔力+体质，高速系（哥布林）主加敏捷+体质，均衡型（史莱姆）以体质为主。怪物属性点倍率为小怪 ×1.5、精英 ×2.0、Boss ×3.0。详见 [第 30 节 五维属性体系](#30-五维属性体系v260-重构)。

`getLayerMonster` 返回值中新增字段：

```javascript
{
  // ...原有字段
  fiveDim: { 力量, 敏捷, 体质, 耐力, 魔力 },        // 五维属性值
  fiveDimAttrPts: { 力量, 敏捷, 体质, 耐力, 魔力 },  // 五维属性点
  fiveDimStats: { hp, atk, def, spd, lingli }        // 五维派生战斗属性
}
```

## 10. 伤害公式

### 10.1 通用伤害公式

```
基础伤害 = 攻击力来源 × 技能系数(dmgPct) × 技能伤害倍率(skillDmgMult) × (1 + 天赋法术穿透)
        − 目标防御 × (0.6 − 无视防御比例)
最终伤害 = round(基础伤害 × 随机波动(0.90~1.10)) × 暴击倍率（若暴击）
伤害下限 = max(1, 最终伤害)
```

### 10.2 攻击力来源

| 技能类型 | 攻击力来源 |
| --- | --- |
| 物理技能 | `getPetAtkPower()`：`stats.攻击力` × 各类加成 |
| 法术技能 | `getPetSpiritPower()`：`stats.灵力` × 各类加成 |
| 怪物法术技能 | `getMonsterAtkPower() × 1.2` |

### 10.3 防御减免系数

代码注释明确：**防御削减系数 0.7 → 0.6**（让防御收益降低，提升伤害体感）：

```
有效防御 = 目标防御 × (1 − defReduced)        // defReduced 来自破甲状态
减免后伤害 = 攻击力 × dmgPct × skillDmgMult × (1 + skillDmgBonus) − 有效防御 × (0.6 − ignoreDefPct)
```

### 10.4 技能伤害倍率（skillDmgMult）

```
skillDmgMult = 1 + passives.skillDmg
```

### 10.5 真实伤害

若 `skill.trueDmg` 为真，则**无视防御**：

```
rawDmg = power × dmgPct × skillDmgMult × (1 + skillDmgBonus)
```

### 10.6 多段技能（hits）

若技能有 `hits` 字段（如连击技能），按 `decayPerHit` 衰减：

```
第 i 段伤害 = round(单段伤害 × (1 - decayPerHit)^i)
```

### 10.7 斩杀机制

若 `skill.executeThreshold` 存在且目标当前 HP 低于阈值：

```
伤害 = round(伤害 × (skill.executeMult || 2))
```

### 10.8 装备特效绝杀

`execute` 特效对**小怪**有 5% 概率秒杀（对精英/Boss 无效）：

```javascript
if (equipSpecials.indexOf('execute') >= 0 && monster.enemyType === 'mob' && Math.random() < 0.05) {
  damage = liveBattle.monsterHpArray[targetIdx]; // 直接造成剩余 HP 伤害
}
```

## 11. 暴击系统

### 11.1 暴击率组成

```
critChance = 0.10（基础）
           + passives.critRate          // 被动暴击率
```

```
rawDmg = round(rawDmg × critMult)
```

## 12. buff/debuff 系统

### 12.1 buff 结构

每个单位（宠物或怪物）拥有独立的 buff 容器：

```javascript
{
  atk: 0, def: 0, spd: 0, all: 0,   // 属性百分比加成
  shield: 0, shieldTurns: 0,         // 护盾值与持续回合
  hotTurns: 0, hotPct: 0,            // 持续恢复（HoT）
  buffTurns: {},                     // 各 buff 的剩余回合
  counterBuff: 0, counterTurns: 0,   // 反击姿态
  reflectBuff: 0, reflectTurns: 0,   // 反射伤害
  defDebuff: 0,                      // 自身防御降低（副作用）
  stolenAtk: 0, stolenAtkTurns: 0    // 偷取的攻击力
}
```

### 12.2 各 buff 说明

| buff 字段 | 作用 | 触发技能示例 |
| --- | --- | --- |
| `atk` | 攻击力百分比加成（`×(1+atk)`） | 攻击强化类 |
| `def` | 防御力百分比加成 | 防御强化类 |
| `spd` | 速度百分比加成 | 加速 |
| `all` | 全属性百分比加成 | 祝福 |
| `shield` | 护盾值（先于 HP 扣除） | 守护盾 |
| `hotTurns` / `hotPct` | 每回合恢复 `maxHP × hotPct` | 生命绽放 |
| `counterBuff` | 受击时反击 `攻击力 × counterBuff` | 反击姿态 |
| `reflectBuff` | 反弹 `受到伤害 × reflectBuff` | 荆棘护盾 |
| `defDebuff` | 自身防御降低百分比 | 狂战之怒副作用 |
| `stolenAtk` | 偷取的攻击力（直接相加） | 偷取类技能 |

### 12.3 buff 回合递减

`applyTurnRegen()` 每回合对所有单位的 `buffTurns` 递减，归零时清除对应 buff：

```javascript
Object.keys(buffs.buffTurns).forEach(k => {
  if (buffs.buffTurns[k] > 0) {
    buffs.buffTurns[k]--;
    if (buffs.buffTurns[k] <= 0) {
      if (k === 'atk') buffs.atk = 0;
      // ...其他字段
    }
  }
});
```

`counterTurns` / `reflectTurns` / `stolenAtkTurns` / `shieldTurns` 独立递减，归零时清除效果。

### 12.4 护盾结算

受到伤害时，护盾优先于 HP 扣除：

```javascript
if (buffs.shield >= monsterDmg) { buffs.shield -= monsterDmg; monsterDmg = 0; }
else { monsterDmg -= buffs.shield; buffs.shield = 0; }
```

### 13.1 状态字段

```javascript
{

  poisoned: 0,  poisonPct: 0,  poisonStacks: 0,
```

### 13.2 状态效果一览

| 状态 | 效果 | 图标 | 持续递减 |
| --- | --- | --- | --- |
| `frozen` | 冻结，无法行动 | ❄️ | 每回合 -1 |
| `stunned` | 眩晕，无法行动 | 💫 | 每回合 -1 |
| `silenced` | 沉默，无法释放主动技能 | （无图标） | 每回合 -1 |
| `rooted` | 定身，无法行动 | 🌿 | 每回合 -1 |
| `sleeping` | 睡眠，无法行动 | 😴 | 每回合 -1 |
| `poisoned` | 中毒，每回合损失 `maxHP × poisonPct` | ☠️ | 每回合 -1，归零时清空层数 |
| `burning` | 燃烧，每回合损失 `maxHP × burnPct` | 🔥 | 每回合 -1 |
| `defReduced` | 防御降低百分比 | （无图标） | `defReduceTurns` 递减 |

### 13.3 中毒可叠加（stackPoison）

部分毒技能支持叠加：

```
poisonStacks = min(maxStacks || 5, 当前层数 + 1)
poisonPct = (poisonPctPerStack || 0.03) × poisonStacks
```

### 13.4 状态施加深渊

技能通过 `freezeChance` / `stunChance` / `silenceChance` / `rootChance` / `sleepChance` 等字段独立判定每个目标：

```javascript
if (skill.freezeChance && Math.random() < skill.freezeChance) {
  tgtMs.frozen = skill.freezeTurns || 1;
}
```

### 13.5 状态阻止行动

`executePetAction` 与 `executeMonsterAction` 在行动前检查阻止状态：

```javascript
if (status.stunned > 0) { addBattleLog(...); status.stunned--; nextFromQueue(); return; }
// frozen / sleeping / rooted 同理
```

注意：`silenced` **不阻止行动**，仅在 `tryUseActiveSkill` 中阻止技能释放。

### 13.6 净化（cleanse）

治疗技能若带 `cleanse` 字段，可清除目标所有负面状态：

```javascript
st.frozen = st.stunned = st.silenced = st.rooted = st.sleeping = 0;
st.poisoned = st.poisonPct = st.burning = st.burnPct = 0;
```

### 14.1 主动技能触发概率

```
passiveBonus = passives.skillTrigger
```

```javascript
var checkList = available.slice(0, Math.min(3, available.length));
```

### 14.5 法术连击（magic_double）

### 14.7 主动技能受被动效果加成（v2.5.0 修复，需求8）

> **v2.5.0 修复**：此前主动技能（`executeActiveSkill`）的伤害计算路径未读取 `passives.*` 字段，导致"必杀"、"强力"等被动提供的伤害加成、暴击加成、属性加成**只对普攻生效**，主动技能享受不到被动加成。v2.5.0 起统一伤害计算底层逻辑，所有被动技能提供的效果**同时作用于普攻与主动技能**。

#### 1. 修复前的伤害计算差异

| 计算项 | 普攻（executeNormalAttack） | 主动技能（executeActiveSkill） |
|:---:|:---:|:---:|
| `passives.atkBonus`（攻击力 +%） | ✅ 读取 | ❌ 不读取 |
| `passives.strBonus`（力量 +%） | ✅ 读取 | ❌ 不读取 |
| `passives.critRate`（暴击率 +%） | ✅ 读取 | ❌ 不读取 |
| `passives.critDmg`（暴击伤害 +%） | ✅ 读取 | ❌ 不读取 |
| `passives.dmgBonus`（伤害 +%，如"必杀"） | ✅ 读取 | ❌ 不读取 |
| `passives.ignoreDef`（无视防御 %） | ✅ 读取 | ❌ 不读取 |

#### 2. 修复后统一逻辑

```js
// 修复后：主动技能与普攻使用同一份 passives 读取逻辑
finalAtkPower = atkPower × (1 + passives.atkBonus)
                                × (1 + passives.strBonus)
                                × (1 + passives.dmgBonus)  // 必杀、强力等
                                × (1 + auras.teamAtk)
                                × (1 + buffs.atk)
                                × (1 + bloodline.atkPct);

finalCritRate   = 0.10 + passives.critRate + auras.teamCrit + bloodline.critRate + skill.bonusCrit;
finalCritDmg    = 1.8 + passives.critDmg + stats.critDmg;
finalIgnoreDef  = passives.ignoreDef + stats.ignoreDef;
```

#### 3. 校验标准

同一宠物使用主动技能时，有被动加成的伤害**显著高于**无被动加成的伤害，数值符合被动描述比例。例如"必杀"被动（+30% 伤害）现在对主动技能同样生效。

## 15. 光环技能机制

### 15.1 触发规则

`tryExecuteAuraSkills(pet)` 在宠物行动最早期执行（状态检查之后、主动技能之前）：

- 仅处理 `cd` 不存在或为 0 的光环技能。
- **当前回合必释放**：不经过概率判定（100% 触发）。
- 不占用主动技能触发次数。
- 通过 `liveBattle.activeAuras[skill.id]` 标记避免重复释放。

### 15.2 释放流程

```javascript
auraSkills.forEach(skill => {
  if (skill.cd && skill.cd > 0) return;          // 有冷却跳过
  if (liveBattle.activeAuras[skill.id]) return;   // 已生效跳过
  liveBattle.activeAuras[skill.id] = true;        // 标记生效
  executeAuraSkill(pet, skill);                   // 释放
});
```

### 15.3 光环效果

光环效果本身通过 `getAuraEffects(liveBattle.team)` 被动叠加到全队属性。`executeAuraSkill` 负责显示性「释放」动作：日志、动画、效果描述。

支持的光环效果字段（`skill.effect`）：

| 字段 | 作用 |
| --- | --- |
| `teamAtk` | 全队攻击力百分比加成 |
| `teamDef` | 全队防御力百分比加成 |
| `teamSpd` | 全队速度百分比加成 |
| `teamHp` | 全队气血上限百分比加成 |
| `teamCrit` | 全队暴击率加成 |
| `teamRegen` | 全队每回合回复比例 |

### 15.4 显示

光环技能释放时通过 `showSkillCastAnnouncement` 在战场中央显示，颜色为紫色（见 [第 15 节 光环技能机制](#15-光环技能机制) 与 [第 25 节 动画系统](#25-动画系统)）。

## 16. 法术群体技能 targetCount 系统

### 16.1 targetCount 字段

技能定义中 `targetCount` 表示「秒 N」（同时命中的目标数）：

| 技能 | targetCount | dmgPct | 类型 |
| --- | --- | --- | --- |
| 陨石天降（meteor） | 3（秒3） | 0.80 | 物理·群体 |
| 烈焰风暴（flame_storm） | 3（秒3） | 0.85 | 物理·群体·持续 |
| 奥术弹（arcane_bolt） | 3（秒3） | 0.15 | 魔法·群体 |
| 暴风雪（blizzard） | 4（秒4） | 0.60 | 法术·群体·控制 |
| 毒雾（poison_mist） | 4（秒4） | 0.50 | 法术·群体·持续 |
| 碧水滔天（water_deluge） | 4（秒4） | 0.55 | 物理·群体 |
| 蝶潮（butterfly_tide） | 5（秒5） | 0.45 | 法术·群体·控制 |
| 佛行震地（buddha_quake） | 5（秒5） | 0.50 | 法术·群体·治疗 |
| 圣光术（holy_light） | 3（秒3） | — | 治疗·单体（治疗生命最低 3 友方） |

### 16.2 目标数越多系数越低

由上表可见，dmgPct 随 targetCount 增加而递减：

```
秒3 → dmgPct ≈ 0.80~0.85
秒4 → dmgPct ≈ 0.50~0.60
秒5 → dmgPct ≈ 0.45~0.50
```

这是平衡设计：目标数越多，单目标伤害越低，总伤害大致持平。

### 16.3 实际作用目标

`executeActiveSkill` 中 AOE 技能按 `targetCount` 限制命中数：

```javascript
if (isAOE) {
  var aliveIndices = getAliveMonsterIndices();
  // 需求19：targetCount 限制（秒3/秒4/秒5），目标数越多伤害系数越低
  if (skill.targetCount && skill.targetCount > 0 && aliveIndices.length > skill.targetCount) {
    // 随机选取 targetCount 个目标
    targetIndices = shuffle(aliveIndices).slice(0, skill.targetCount);
  } else {
    targetIndices = aliveIndices;
  }
}
```

当存活敌人数 > `targetCount` 时，随机选取 `targetCount` 个目标命中；存活敌人数 ≤ `targetCount` 时命中全部。

### 16.4 天赋目标数加成

天赋「智力」3 阶描述：

> 智力+25%, 法术伤害+15%, **法术技能目标数+1（秒3变秒4）**

### 18.1 挂机机制

`autoBattleInterval` 是自动战斗的「心跳」标志（实际为 `setInterval(() => {}, 999999)` 的占位定时器，真正驱动靠 `scheduleNextTurn` 的 `setTimeout` 链）。

### 18.3 自动连战

- **主线**：清空一波后 `processWaveCleared` 在 1000ms 后自动 `spawnMonster()` 开启下一波。
- **失败**：`processBattleDefeat` 在 1500ms 后自动重生战斗（损失 5% 经验）。
- **藏宝图**：战斗结束后**停止**自动战斗（`clearInterval(autoBattleInterval)`）。

### 18.4 自动开箱

## 26. 战斗结算自动关闭与覆盖机制（v2.5.0 新增，需求1）

> **v2.5.0 新增**：战斗胜利结算面板支持自动关闭，且新结算触发时强制覆盖旧结算，避免连续战斗/挂机多场结算时弹窗叠加。

### 26.1 自动关闭倒计时

```js
window._battleSettlementTimer = setTimeout(function() {

}, 2000);
```

- 倒计时结束后自动关闭界面
- 倒计时内支持玩家手动点击关闭按钮

### 26.2 新旧覆盖机制

若当前已有结算界面展示时触发了新的战斗结算（如连续战斗、挂机多场结算排队）：

1. **立即销毁**上一个结算界面（清除 DOM 与定时器）
2. **直接展示**最新的结算面板
3. 新面板**重新开始 2 秒**自动关闭倒计时

```js
function showBattleSettlementModal(rewards) {
  // 若已有结算面板，先销毁

    clearTimeout(window._battleSettlementTimer);
    window._battleSettlementTimer = null;
  }
  closeBattleSettlementModal(); // 销毁旧面板
  // 渲染新面板并启动新的 2 秒倒计时
  renderSettlementModal(rewards);
  window._battleSettlementTimer = setTimeout(closeBattleSettlementModal, 2000);
}
```

### 26.3 校验标准

- 单场结算 2 秒后自动关闭
- 连续触发结算时不会叠加弹窗，仅保留最新一场

## 30. 五维属性体系（v2.6.0 重构）

> **v2.6.0 重构（需求16）**：战斗属性计算从原四维（体质/力量/敏捷/智慧）升级为五维——**力量 / 敏捷 / 体质 / 耐力（NEW）/ 魔力（原"智慧"重命名）**。正式移除原"智力"属性，原精神/智力相关法术效果全部整合至魔力属性中；新增"耐力"属性作为物理防御力的核心来源。玩家宠物、玩家人物、所有怪物、人物装备、宠物装备全部采用同一套五维属性定义与收益规则。

### 30.1 五维属性定义

| 属性 | 作用 |
| --- | --- |
| 力量 | 大幅提升物理攻击力，少量提升灵力 |
| 敏捷 | 大幅提升战斗速度，提升闪避率 |
| 体质 | 大幅提升生命值上限，少量提升灵力/防御 |
| 耐力（NEW） | 大幅提升物理防御力，少量提升灵力 |
| 魔力（原智慧） | 大幅提升法术攻击力/法术防御力，提升魔法值上限 |

### 30.2 属性点转化公式（getAttrPointValue）

每 1 点五维属性点转化为对应战斗属性值的计算公式全局统一：

```
单属性点对应战斗值 = 1 + (资质 / 10000) × (1 + (成长 + 种族值) / 10)
```

- **资质**：对应单项资质（力量资质/敏捷资质/体质资质/耐力资质/魔力资质）
- **成长**：宠物成长值
- **种族值**：对应种族单项种族值（`getRaceValues(race)`）

资质、成长、种族值越高，单属性点转化出的战斗属性值越高，实现同等级下高品质宠物属性收益更高的差异化。怪物固定使用资质 1500、成长 1.0 计算。

### 30.3 五维→战斗属性公式

战斗属性计算由五维属性派生，公式如下（`generateMonsterFiveDim` 与 `getCombatPowerBreakdown` 共用）：

| 战斗属性 | 公式 | 变化说明 |
| --- | --- | --- |
| 攻击力 | `力量 × 2 + 敏捷 × 0.5` | 不变 |
| 防御力 | `耐力 × 1.5 + 体质 × 0.3` | **耐力大幅提升防御**（原为 `体质 × 0.6`） |
| 灵力 | `魔力 × 1.5 + 体质 × 0.3 + 耐力 × 0.3 + 力量 × 0.2` | **魔力大幅提升**（原为 `智慧 × 2`） |
| 气血 | `体质 × 10 + 等级 × 20` | 不变 |
| 速度 | `敏捷 × 1.5` | 不变 |
| 魔法值 | `魔力 × 3 + 力量 × 0.5 + 等级 × 5` | **魔力主导**（原为 `力量 × 1.5 + 智慧 × 2 + 等级 × 5`） |

### 30.4 怪物五维属性（generateMonsterFiveDim）

所有怪物（主线野怪/副本怪/Boss/竞技场/血色要塞）全部使用与宠物同一套五维属性计算体系，按种族差异化加点：

| 种族定位 | 加点方向 | 适用种族 |
| --- | --- | --- |
| 物理系 | 力量 35% + 耐力 25% + 体质 20% + 敏捷 20% | 龙、恶魔 |
| 法术系 | 魔力 35% + 体质 25% + 耐力 20% + 敏捷 20% | 天使、精灵 |
| 高速系 | 敏捷 35% + 力量 25% + 体质 20% + 耐力 20% | 哥布林 |
| 均衡型 | 体质 30% + 力量 25% + 耐力 20% + 敏捷 15% + 魔力 10% | 史莱姆等 |

**怪物属性点倍率**（保持挑战度，但计算公式与宠物完全一致）：

| 怪物类型 | 属性点倍率 |
| --- | --- |
| mob（小怪） | ×1.5 |
| elite（精英） | ×2.0 |
| boss（首领） | ×3.0 |

```javascript
// battle.js → generateMonsterFiveDim(level, enemyType, race)
var typeMult = enemyType === 'boss' ? 3.0 : enemyType === 'elite' ? 2.0 : 1.5;
var totalPoints = Math.floor(level * 10 * typeMult);
```

### 30.5 战力计算重构（getCombatPowerBreakdown）

`getCombatPowerBreakdown`（`pet.js:660`）全面重构为五维体系：

- **基础属性**：使用 [30.2 属性点转化公式](#302-属性点转化公式getattrpointvalue) 计算五维，再按 [30.3 五维→战斗属性公式](#303-五维战斗属性公式) 派生。
- **战力权重不变**：

```javascript
var CP_WEIGHTS = { 气血: 0.3, 攻击力: 3, 防御力: 2, 灵力: 2.5, 速度: 1.5 };
```

- **加成层级**（均适配五维）：基础属性 → 宝石加成 → 人物属性加成 → 天赋倍率加成 → 装备特效百分比加成 → 宠物装备加成。
- **旧存档兼容**：`耐力资质` 缺失时 fallback 至 `智力资质`；`魔力资质` 缺失时 fallback 至 `智力资质`；旧 `int` 类型宝石自动映射为 `mag`。

### 30.6 旧存档迁移

`migrateFiveDimAttributes` 函数自动处理旧存档：

- 宠物资质：`智力资质 → 魔力资质`，新增 `耐力资质`（默认 = 体质资质）
- 人物属性：`智力 → 魔力`，新增 `耐力`（默认 = 体质）
- 旧装备 `int_flat`/`int_pct` 词条通过 fallback 兼容（`mag_flat || int_flat`）

## 31. 血色要塞强度与buff规则（v2.6.0 需求11）

> **v2.6.0 新增（需求11）**：血色要塞（Crimson Fortress）作为无限轮次 Roguelike 战斗活动，其怪物强度与 buff 生效规则独立于主线。本节描述血色要塞中怪物五维属性的三层强化机制，实现位于 `economy.js: generateCrimsonFortressMonster()`。

### 31.1 非领主怪基础强度提升 30%

血色要塞中**除领主/Boss 外的所有普通怪物**，五维属性（力量/敏捷/体质/耐力/魔力）在原有基础上整体提升 30%，同时 HP 和攻击力也同步提升 30%。

```javascript
// economy.js → generateCrimsonFortressMonster()
if (!isBoss) {
  finalHp = Math.floor(finalHp * 1.30);
  baseAtk = Math.floor(baseAtk * 1.30);
  // 五维属性整体提升30%
  Object.keys(fiveDim).forEach(function(k) { fiveDim[k] = Math.floor(fiveDim[k] * 1.30); });
}
```

- **触发条件**：`enemyType !== 'boss'`（即非每 5 关出现的血色领主）
- **影响范围**：五维全部 5 项属性 + HP + 攻击力
- Boss（血色领主）不享受此 30% 提升，而是使用专属倍率（HP ×3，攻击 ×2.5）

### 31.2 回合buff实时生效

每累计 5 回合怪物获得的增益 buff **实时计入五维属性计算**，buff 映射到对应五维属性：

| buff 类型 | 影响的五维属性 |
| --- | --- |
| 攻击 buff（`atkPct`） | 力量、魔力 |
| 防御 buff（`defPct`） | 耐力、体质 |
| HP buff（`hpPct`） | 体质 |
| 速度 buff（`spdPct`） | 敏捷 |

```javascript
// economy.js → generateCrimsonFortressMonster()
if (monsterAtkPct) {
  fiveDim.力量 = Math.floor(fiveDim.力量 * (1 + monsterAtkPct));
  fiveDim.魔力 = Math.floor(fiveDim.魔力 * (1 + monsterAtkPct));
}
if (monsterDefPct) {
  fiveDim.耐力 = Math.floor(fiveDim.耐力 * (1 + monsterDefPct));
  fiveDim.体质 = Math.floor(fiveDim.体质 * (1 + monsterDefPct));
}
if (monsterHpPct)  { fiveDim.体质 = Math.floor(fiveDim.体质 * (1 + monsterHpPct)); }
if (monsterSpdPct) { fiveDim.敏捷 = Math.floor(fiveDim.敏捷 * (1 + monsterSpdPct)); }
```

- **数据来源**：`getCrimsonFortressBuffEffects()` 返回当前累计的怪物 buff 加成
- **生效时机**：怪物生成时即计入，本场战斗后续回合中真实生效
- **同步影响**：HP/攻击/防御/速度 也按对应 buff 同步放大（如 `finalHp × (1 + monsterHpPct)`）

### 31.3 轮次强度递增 10%（永久叠加）

每通关 5 轮要塞，后续所有轮次的怪物五维属性在上一轮基础上**提升 10%**（原为 5%，v2.6.0 上调）。该增强为**轮次永久叠加**，轮次越高怪物强度越高。

```javascript
// economy.js → generateCrimsonFortressMonster()
var powerMult = diff.powerMult * (1 + Math.floor(round / 5) * 0.10);  // HP/攻击递增
// ...
var roundBoost = 1 + Math.floor(round / 5) * 0.10;  // 五维属性递增
Object.keys(fiveDim).forEach(function(k) { fiveDim[k] = Math.floor(fiveDim[k] * roundBoost); });
```

| 轮次区间 | 五维属性强化倍率 | HP/攻击强化倍率 |
| --- | --- | --- |
| 第 1–4 轮 | ×1.0 | ×1.0 |
| 第 5–9 轮 | ×1.1 | ×1.1 |
| 第 10–14 轮 | ×1.2 | ×1.2 |
| 第 15–19 轮 | ×1.3 | ×1.3 |
| 第 N 轮（N≥5） | ×(1 + floor(N/5) × 0.1) | ×(1 + floor(N/5) × 0.1) |

> 注：难度倍率（简单 0.7× / 普通 1.0× / 困难 1.5×）作为 `diff.powerMult` 与轮次递增倍率相乘，共同决定最终怪物强度。

### 31.4 三层强化叠加顺序

血色要塞怪物的五维属性按以下顺序依次叠加（均向下取整）：

1. **基础生成**：`generateMonsterFiveDim` 按种族差异化加点生成
2. **非领主 30% 提升**：非 Boss 怪物五维 ×1.30
3. **轮次 10% 递增**：`fiveDim[k] × (1 + floor(round/5) × 0.10)`
4. **回合 buff 实时计入**：按 [31.2 映射表](#312-回合buff实时生效) 分别乘以对应 buff 倍率

### 31.5 校验标准

- 血色要塞中普通怪物五维属性比同等级主线怪物高 30%（领主/Boss 除外）
- 每累计 5 回合触发的怪物增益 buff 在当回合即体现在五维属性数值上
- 通关 5 轮后，后续轮次怪物五维属性较第 1 轮提升 10%；通关 10 轮后提升 20%，以此类推永久叠加

## v2.7.0 更新

> **版本**：v2.7.0 属性展示与战斗力体系重构
> **更新日期**：2026-07-11

### 一、宠物战斗力评价体系重构（需求4.2）

v2.7.0 对 `getCombatPowerBreakdown(pet)`（`pet.js`）进行重构，在 v2.6.0 五维派生属性战力基础上**纳入二级战斗属性权重**并引入**寿命系数**，使战力数值更贴近实战表现。战力数值直接影响出战推荐排序、派遣奇遇战力校验、竞技场匹配等多个战斗相关模块。

#### 1. 战力计算新公式

```
最终战力 = (属性战力小计 + 二级属性战力) × 技能倍率 × 寿命系数
```

| 组成部分 | 说明 |
|---|---|
| 属性战力小计 | 沿用 v2.6.0 五维派生属性战力：气血 ×0.3 + 攻击力 ×3 + 防御力 ×2 + 灵力 ×2.5 + 速度 ×1.5 |
| 二级属性战力 | 10 项二级战斗属性按权重加权求和（见下表） |
| 技能倍率 | 主动技能平均 dmgPct 超 1.0 部分 ×0.5 + 技能数每个 +3%；被动技能按效果类型加权；异化品质额外 ×1.2 |
| 寿命系数 | 寿命 <1000 时按 `0.5 + lifespan/2000` 折减；神兽（99999）不受影响 |

#### 2. 二级属性战力权重表

| 二级属性 | 权重 | 战斗意义 |
|---|---|---|
| 暴击率 | ×1.0 | 直接提升期望伤害，权重最高 |
| 暴击伤害 | ×0.3 | 乘区属性，与暴击率叠加后边际递减 |
| 闪避率 | ×0.8 | 提升生存能力 |
| 命中率 | ×0.5 | 仅超过 0.8（80%）的部分计入，避免基础命中虚高战力 |
| 吸血 | ×0.6 | 兼顾输出与回复 |
| 减伤 | ×1.0 | 直接降低受到伤害，权重最高 |
| 技能触发 | ×0.8 | 提升主动技能释放频率 |
| 技能增伤 | ×0.5 | 乘区属性，权重适中 |
| 回血 | ×0.4 | 每回合稳定回复，长线战斗价值更高 |
| 法术增伤 | ×0.5 | 仅作用于法术类技能，权重适中 |

> 权重设计原则：暴击率/减伤等直接影响伤害与生存的属性权重最高（1.0）；暴击伤害/技能增伤等乘区属性权重较低（0.3~0.5），避免与暴击率叠加后战力虚高；命中率仅超额部分计入，防止基础命中虚增战力。

#### 3. 寿命系数

| 寿命区间 | 系数 | 说明 |
|---|---|---|
| `pet.lifespan >= 1000` | ×1.0 | 正常寿命，无折减 |
| `pet.lifespan < 1000` | `0.5 + lifespan/2000` | 寿命越低战力折减越大，最低 ×0.5 |
| 神兽（`lifespan === 99999`） | ×1.0 | 神兽寿命无限，不受折减影响 |

- 寿命系数作为最终乘数作用于战力总计，战斗中实际伤害/生存不受寿命影响（寿命仅影响战力评分，不影响实时战斗数值）；
- 该设计鼓励玩家通过月华露/元宵等道具维持宠物寿命，避免低寿命宠物占据高战力排名。

#### 4. 战力数值的下游影响

| 模块 | 影响方式 |
|---|---|
| 出战推荐 | 宠物列表按新战力排序，推荐战力最高的宠物出战 |
| 派遣奇遇 | `getDispatchTotalPower` 汇总派遣宠物战力，影响奖励加成（+5%/10% 战力溢出，上限 ×3.0） |
| 竞技场匹配 | 按战力段位匹配对手 |
| 宠物详情页 | 战力数值后新增「ⓘ」图标，点击展开战力构成详情（属性战力/二级属性战力/技能倍率/寿命系数/最终战力） |

> **注意**：v2.6.0 的五维派生属性战力计算（气血/攻击力/防御力/灵力/速度权重）保持不变，v2.7.0 仅在原基础上**新增二级属性战力与寿命系数两个维度**，不会导致旧宠物战力大幅波动。

#### 2. 战斗模拟逻辑

弹窗战斗使用简化版回合制模拟（非完整 `liveBattle` 状态机），但包含以下核心机制：

| 机制 | 说明 |
|---|---|
| **宠物属性** | 通过 `getPetStats(pet)` 获取真实属性（气血/攻击力/防御力/速度/暴击率/暴击伤害） |
| **回合上限** | 30 回合，超时按剩余HP比例判定胜负 |
| **暴击系统** | 按 `critRate` 概率触发暴击，伤害 × `critDmg` |
| **伤害分摊** | 敌人攻击随机选择存活宠物造成伤害 |
| **伤害浮动** | 基础伤害 × (0.8 + random × 0.4) |
| **伤害倍率** | `petDmgMult = 0.8 + winChance × 0.8`；`enemyDmgMult = 0.8 + (1-winChance) × 0.8` |
| **胜负判定** | 敌人HP≤0→胜；队伍HP≤0→败；回合结束→按剩余HP比例判定 |

### 二、团本Boss战斗（challengeRaidBoss）

#### 1. 战斗流程

```
challengeRaidBoss(raidId, bossId)
  → canChallengeBoss() 检查（小Boss可直接挑战，最终Boss需全部小Boss击败）
  → getTeamPets() 获取出战阵容
  → 计算monsterPower = boss.hp × 0.06 + boss.atk × 20 + boss.def × 30
  → 计算winChance = clamp(playerCp / (monsterPower × 1.2), 0.1, 0.95)
  → startPopupBattle({ type:'raid', onComplete: function(success) {...} })
  → 胜利：标记bossesDefeated[bossId]=true，最终Boss触发老虎机
  → 失败：提示可再次尝试
```

#### 2. Boss机制说明（数据定义，非战斗状态机实现）

团本Boss的机制（标记/护盾/召唤/灼烧/反弹/易伤）和阶段切换（70%/30% HP阈值）作为**数据定义**存储于 `RAID_DUNGEONS` 配置中，当前战斗模拟通过 `monsterPower` 和 `winChance` 简化处理。Boss机制描述在UI中展示给玩家，作为策略参考信息。
