## 一、系统总览

本游戏的「副本与活动」内容分为三大入口：

| 入口 | 屏幕名称 | 主要内容 |
|------|----------|----------|
| 主线战斗 | `main` | 11 张主线地图，按 mobs→elite→boss 三阶段推进，掉落宝箱与藏宝图 |
| 副本 | `dungeon` | 7 个特殊副本（含血统副本）+ 3 个团队副本，需门票与每日次数 |
| 活动 | `activity` | 7 个 Sheet 子页：打宝图 / 爬塔 / 竞技场 / 进阶试炼 / 押镖 / 技能秘境 / 宠物秘境 |
| 藏宝图 | `treasure` | 藏宝图背包、合成、购买、使用（进入守卫战斗） |

**关键设计**：
- **技能秘境（skill_tower）** 与 **宠物秘境（pet_equip_cave）** 已从特殊副本列表移除，分别迁移到活动页面的「技能秘境」Sheet 和「宠物秘境」Sheet。
- 新增 **血统副本（bloodline_dungeon）** 进入特殊副本列表，作为血统珠的稳定产出途径。
- 起源草地（地图 1）怪物强度削弱一半，方便新手过渡。
- 主线宝箱掉落已恢复（`dropChests` 在所有主线战斗胜利后调用）。

## 二、主线地图系统

### 2.1 地图列表（MAPS）

主线共 11 张地图，每张地图固定 10 层（`layers: 10`）。地图按等级阶梯解锁：

| ID | 地图名称 | 等级范围 | 精英名 | Boss 名 | 怪物种族池 |
|----|----------|----------|--------|---------|------------|
| 1 | 起源草地 | Lv.1~5 | 巨角野兔 | 草原因子 | 史莱姆 / 哥布林 |
| 2 | 新手森林 | Lv.1~15 | 森林巨狼 | 树精之王 | 精灵 / 哥布林 |
| 3 | 幽暗矿洞 | Lv.10~25 | 矿洞巨魔 | 暗影矿主 | 史莱姆 / 哥布林 |
| 4 | 亡灵墓地 | Lv.20~35 | 死亡骑士 | 巫妖之王 | 恶魔 / 史莱姆 |
| 5 | 烈焰火山 | Lv.30~45 | 熔岩巨人 | 炎魔领主 | 龙 / 恶魔 |
| 6 | 冰霜高原 | Lv.40~55 | 冰霜巨人 | 冰龙女王 | 龙 / 天使 |
| 7 | 暗影沼泽 | Lv.50~65 | 沼泽九头蛇 | 暗影之主 | 恶魔 / 哥布林 |
| 8 | 天空之城 | Lv.60~75 | 大天使长 | 天空之神 | 天使 / 龙 |
| 9 | 深渊裂隙 | Lv.70~85 | 深渊领主 | 混沌魔王 | 恶魔 / 精灵 |
| 10 | 龙之巢穴 | Lv.80~95 | 上古龙卫 | 万龙之祖 | 龙 / 恶魔 |
| 11 | 终焉神殿 | Lv.90~110 | 终焉使者 | 创世神 | 恶魔 / 龙 |

每张地图的 `monsters` 字段定义 3 种普通小怪名，`eliteName` 与 `bossName` 单独配置。

### 2.2 起源草地削弱机制

地图 1（起源草地）作为新手起点，怪物 HP 与攻击力额外乘以 0.5：

```js
// battle.js → getLayerMonster
if (mapId === 1) {
  baseHp = Math.floor(baseHp * 0.5);
  baseAtk = Math.floor(baseAtk * 0.5);
}
```

即起源草地的所有小怪、精英、Boss 的血量与攻击力均减半，防御不受影响。

### 2.3 怪物种族与固定 Boss（MAP_BOSSES）

- **怪物种族池（MAP_MONSTER_RACES）**：每张地图配置 2 个种族，小怪/精英从中随机选取，影响技能组合（`MONSTER_RACE_SKILLS`）。
- **固定 Boss 配置（MAP_BOSSES）**：每张地图的 Boss 拥有固定名称、种族、5 主动 + 3 被动技能组合、buff 加成与专属战利品。

Boss buff 字段含义：

| 字段 | 含义 |
|------|------|
| `atk` | 攻击力加成百分比 |
| `def` | 防御力加成百分比 |
| `spd` | 速度加成百分比 |
| `all` | 全属性加成百分比 |
| `reflect` | 反伤比例 |

Boss 专属战利品（`loot` 字段）：v2.0.3 起 boss1/boss2 的 `loot` 字段已**清空为空数组**，不再固定掉落 `hatch_stone`（孵化石）/ `fusion_stone`（融合石）等稀有道具。其他 Boss 的 `loot` 字段仍保留各类门票、`guixu_pill` 等奖励。

> 主线掉落的 `hatch_stone` 与 `fusion_stone` 已整体移除，改为通过 `maybeDropTalentLoot` 掉落新道具：精英怪掉落 `socket_nail`（打孔钉）、Boss 掉落 `repair_glue`（修补胶）（详见 [§16](#十六天赋额外掉落)）。

## 三、关卡进度系统

### 3.1 三阶段循环：mobs → elite → boss

每张地图的进度按「层（Layer）」推进，每层经历三个阶段：

```
mobs 阶段 → elite 阶段 → boss 阶段（仅最后一层）
     ↑                ↓
     └── 通关下一层 ──┘
```

1. **mobs 阶段**：击杀 8~12 只小怪后进入 elite 阶段。
2. **elite 阶段**：击败精英波后，若未到最后一层则进入下一层 mobs 阶段；若已到最后一层则进入 boss 阶段。
3. **boss 阶段**：击败 Boss 后该地图通关，解锁下一张地图。

### 3.2 层数与进度字段

进度数据存储在 `G.mapProgress[mapId]`，结构如下：

| 字段 | 类型 | 含义 |
|------|------|------|
| `currentLayer` | number | 当前层数（1 ~ map.layers） |
| `mobsKilled` | number | 当前层已击杀小怪波数 |
| `mobsNeeded` | number | 当前层需击杀的小怪波数（randomInt(8, 12)） |
| `phase` | string | 当前阶段：`'mobs'` / `'elite'` / `'boss'` / `'cleared'` |

阶段推进逻辑（`processWaveCleared`）：

| 当前阶段 | 击败怪物类型 | 推进结果 |
|----------|--------------|----------|
| mobs | mob | `mobsKilled++`，满 `mobsNeeded` 后进入 elite |
| elite | elite | 未到最后一层：`currentLayer++` 回到 mobs；到最后一层：进入 boss |
| boss | boss | `phase = 'cleared'`，解锁下一地图 |

### 3.3 怪物波次生成（generateMonsterWave）

主线采用「多怪同时上场」的波次系统，单场上限 6 个敌人：

| 阶段 | 波次构成 |
|------|----------|
| mobs | 3~6 个小怪 |
| elite | 4 个小怪 + 2 个精英（精英最后出场） |
| boss | 3 个小怪 + 2 个精英 + 1 个 Boss（Boss 最后出场） |

副本与藏宝图战斗不走波次系统，保持单怪模式。

### 3.4 怪物强度公式

怪物属性按连续成长曲线生成，避免跨图难度阶梯突变：

| 怪物类型 | hpMult | atkMult | defMult |
|----------|--------|---------|---------|
| 小怪 mob | 5 | 5 | 1.2 |
| 精英 elite | 12 | 8 | 1.6 |
| Boss | 24 | 12 | 2.2 |

公式（`getLayerMonster`）：

```
lvScale = 1 + lv * 0.012          // lv 为地图等级范围内的随机值
baseHp  = floor((40 + lv * 20) * hpMult * lvScale)
baseAtk = floor((4  + lv * 3.8) * atkMult * lvScale)
def     = floor(lv * 2.0 * defMult)
```

HP 与攻击力在 base 基础上有 ±5% / ±10% 的随机浮动。

## 四、主线宝箱掉落

### 4.1 宝箱品质（CHEST_RARITIES）

共 5 个品质，颜色与图标如下：

| ID | 名称 | 颜色 | 图标 |
|----|------|------|------|
| white | 白色宝箱 | #c0c0c0 | ⬜ |
| green | 绿色宝箱 | #22c55e | 🟩 |
| blue | 蓝色宝箱 | #3b82f6 | 🟦 |
| purple | 紫色宝箱 | #a855f7 | 🟪 |
| orange | 橙色宝箱 | #fb923c | 🟧 |

### 4.2 掉落率公式（getChestDropRate）

宝箱基础掉落率受怪物类型与天赋加成：

| 怪物类型 | 基础掉落率 |
|----------|------------|
| mob（小怪） | 5% |
| elite（精英） | 10% |
| boss（Boss） | 20% |

实际掉落率 = 基础率 × (1 + `chest_drop` 天赋 + `loot_mastery` 天赋)

### 4.3 dropChests 掉落规则

每次战斗胜利后调用 `dropChests(enemyType)`，先进行掉落率检定，再决定宝箱品质与数量：

| 怪物类型 | 掉落数量 | 品质分布 |
|----------|----------|----------|
| mob | 1 个 | white 80% / green 20% |
| elite | 1~2 个 | green 50% / blue 40% / purple 10% |
| boss | 1 个（固定） | purple 70% / orange 30% |

> 注：Boss 原本掉落 2~3 个宝箱，已在代码中修复为固定掉落 1 个。

宝箱掉落后，若 `G.autoOpenChests` 为 true 则自动开启，否则存入 `G.chests` 背包等待手动开启。

### 4.4 宝箱内容池（generateChestContents）

每个宝箱只掉落 1 个道具，按品质权重抽取。高品质物品权重更低（出现概率低）。

**物品品质权重**：

| 品质 | 权重倍率 | 颜色 |
|------|----------|------|
| low（低级） | 1.0 | #9ca3af |
| mid（中级） | 0.6 | #3b82f6 |
| high（高级） | 0.3 | #a855f7 |

**各品质宝箱内容池**（权重 w 为基础权重，实际概率 = w × quality 权重 / 总权重）：

> **v2.0.3 变更**：
> - `fusion_stone`（融合石）已从所有 5 个品质池中**移除**
> - `talent_point`（天赋点）已从所有 5 个品质池中**移除**（宝箱转盘不再产出天赋点）
> - 新增 `socket_nail`（打孔钉）/ `repair_glue`（修补胶）/ `advance_pill_*`（进阶丸）等道具进入对应品质池

| 宝箱 | 主要内容（按权重排序） |
|------|------------------------|

**普通词条（TREASURE_AFFIXES）**：8 种，每个词条随机生成数值：

| 词条 ID | 名称 | 数值范围 | 格式 |

| exp_bonus | 经验奖励 | 0.2~0.8 | 经验 +N% |
| equip_drop | 装备掉落率 | 0.3~1.0 | 装备掉落 +N% |
| egg_drop | 宠物蛋掉落率 | 0.3~1.0 | 蛋掉落 +N% |
| skill_drop | 技能书掉落率 | 0.3~1.0 | 技能书掉落 +N% |
| moon_dew | 月华露数量 | 1~5 | 月华露 +N |

| 词条 ID | 名称 | 效果 |
|---------|------|------|
| double_reward | 双倍奖励 | 所有奖励翻倍 |

| guaranteed_equip | 必出装备 | 必定掉落一件史诗以上装备 |

### 5.3 藏宝图生成（generateTreasureMap）

生成一张指定品质的藏宝图（`ui.js` ~line 6232）：
1. 按品质词条数（1~5）从 `TREASURE_AFFIXES` 随机不重复抽取词条，每个词条数值随机生成。
2. 若为 orange 品质，额外随机赋予 1 个特殊词条。
3. 记录 `playerLevel`（生成时的玩家等级，用于挖掘战斗强度）。
4. **v2.0.3 新增**：随机分配 `mapType`（1~11）与 `mapTypeName`，对应 11 张主线地图主题（详见 [§5.7](#57-地图类型与宠物池treasure_map_types)）。`mapType` 决定该藏宝图掉落宠物蛋的宠物池。

> 旧存档藏宝图通过 `state.js → migrateTreasureMaps()` 补设 `mapType` 字段。

### 5.4 藏宝图使用（useTreasureMap）

使用藏宝图后进入「藏宝图守卫」战斗，守卫按品质获得不同强度倍率：

| 品质 | hpMult | atkMult | defMult |
|------|--------|---------|---------|
| white | 1.0 | 1.0 | 1.0 |
| green | 2.4 | 1.6 | 1.2 |
| blue | 3.6 | 2.4 | 1.5 |
| purple | 5.4 | 3.6 | 2.0 |
| orange | 9.6 | 4.8 | 2.5 |

怪物等级 = 玩家等级 × (1 + 怪物强度词条总和)，战斗以 Boss 强度（×3）生成，使用 `enemyType: 'boss'`。

战斗胜利后由 `battle.js → processTreasureRewards()`（~line 2044）发放奖励，宠物蛋掉落使用 `getTreasureMapPetPool(tmap.mapType)` 取该地图类型对应的宠物池，再调用 `generateEggFromName(petName)` 生成指定宠物的蛋（详见《01-宠物系统》§2.5）。`guaranteed_egg` 特殊词条触发时，从池中挑选 T 级最高的宠物生成必出蛋。

### 5.5 藏宝图合成

**自动合成**：5 张同品质藏宝图 → 1 张更高品质。
- 相同词条取最大值并可能提升 10%（50% 概率）或取平均值。
- 不同词条随机保留，凑满目标品质词条数。
- 不足时随机生成新词条补齐。

**手动合成**：选择 2 张藏宝图进行针对性融合。

**购买**：在藏宝图页面直接用金币购买，价格 = (品质序号 + 1) × 500 金币（500/1000/1500/2000/2500）。

### 5.6 藏宝图掉落（maybeDropTreasureMap）

主线战斗胜利后（Lv.60 及以上）有概率掉落藏宝图：

| 品质 | 概率 |
|------|------|
| white | 15% |
| green | 10%（累计 25%） |
| blue | 15%（累计 40%） |

实际概率 = 基础概率 × (1 + `map_drop` 天赋 + `loot_mastery` 天赋)。

| 副本 ID | 每日次数 |
|---------|----------|
| bloodline_dungeon | 3 |
| 其他（默认） | 3 |

`getDungeonDailyLimit(dungeonId)`：若 `DUNGEON_DAILY_LIMITS` 中有配置则返回配置值，否则返回 3。

### 6.3 门票系统

每个特殊副本需消耗 1 张对应门票才能进入。门票道具 ID 见上表 `ticketItem` 字段。

门票获取途径：
- 主线宝箱掉落（purple/orange 宝箱可掉落各类门票）
- Boss 专属战利品（`MAP_BOSSES` 的 `loot` 字段）
- 战令奖励
- 兑换码

### 6.4 副本进入流程（enterSpecialDungeon）

```
1. 查找副本配置（DUNGEONS 数组）
   - 若未找到且 dungeonId === 'pet_equip_cave'，使用 PET_EQUIP_CAVE_FALLBACK 兜底
2. 等级检查（player.level >= dungeon.minLv）
3. 队伍检查（至少 1 只出战宠物）
4. 门票与次数检查（pet_equip_cave 跳过此步）
   - 检查门票数量 > 0
   - 检查今日次数 < dailyLimit
   - 扣除 1 张门票，dungeonDailyUsed[dungeon_id] += 1
5. 生成首波怪物：
   - 怪物等级：当前地图 minLv~maxLv 随机
   - 怪物名称：pet_equip_cave 固定「秘境守卫」，其他从地图 monsters 随机
   - 波次数：pet_equip_cave 固定 5 波，其他 randomInt(3, 5)
   - 怪物强度：副本小怪×5 难度强化
6. 进入 liveBattle 战斗
```

### 6.5 副本波次推进与奖励（processDungeonWaveComplete）

每击败一波怪物后调用，推进 `dungeonWave` 计数，并在最后一波发放通关奖励：

| 副本 ID | 通关奖励 |
|---------|----------|
| exp_cave | 经验 = 玩家等级 × 500 + randomInt(100, 500) |
| gold_mine | 金币 = 玩家等级 × 200 + randomInt(500, 2000) |
| egg_forest | 1 颗宠物蛋（tier = 地图等级对应 tier + 1，上限 T5） |
| forge_mine | 2~4 颗强化石（Lv.40+ 给中强石，Lv.70+ 给高强石） |
| treasure_ruin | 1~2 张藏宝图（按玩家等级决定品质，30% 概率提升一档） |
| gem_cavern | 1~3 颗宝石（10% 概率为 2 级，其余 1 级），受 `gem_drop_rate` 天赋影响 |
| pet_equip_cave | 宠物装备（品质随通关波数提升）+ 30% 概率额外掉材料 |
| bloodline_dungeon | 金币 = 玩家等级 × 100 + randomInt(200, 800)（血统珠已在波次中掉落） |

### 6.6 PET_EQUIP_CAVE_FALLBACK 兜底处理

由于 `pet_equip_cave` 已从 `DUNGEONS` 数组移除，但活动页面仍通过 `enterSpecialDungeon('pet_equip_cave')` 进入战斗，因此在 `ui.js` 中定义了兜底配置：

```js
const PET_EQUIP_CAVE_FALLBACK = {
  id: 'pet_equip_cave',
  name: '宠物秘境',
  type: 'special',
  minLv: 20,
  desc: '挑战神秘秘境，获取宠物装备',
  ticketItem: 'pet_ticket'
};
```

`enterSpecialDungeon` 在 `DUNGEONS` 中查找失败且 `dungeonId === 'pet_equip_cave'` 时，使用此兜底配置，并跳过门票与每日次数检查（次数由活动页面的 `petCaveUsed` 管理）。

## 十、爬塔系统（tower）

### 10.1 基础设定

| 项目 | 设定 |
|------|------|
| 最大层数 | 100 + 转生次数 × 50（`getMaxTowerFloors`） |
| 状态字段 | `G.towerProgress`（当前进度）、`G.towerMaxFloor`（最高记录） |
| 每周重置 | 每周 1 次手动重置（`G.towerWeeklyResetDate`） |
| 重置条件 | `towerProgress > 0` 且本周未重置 |

### 10.2 楼层数据（TOWER_FLOORS / getTowerFloorData）

前 100 层为静态配置（`TOWER_FLOORS`），超过 100 层动态生成：

| 层级 | 类型 | powerMult | HP 公式 | 攻击公式 |
|------|------|-----------|---------|----------|
| 每 10 层（10/20/30...） | Boss 层 | ×3 | (100 + i×50 + i×200) × 3 | (10 + i×3 + i×10) × 3 |
| 其他层 | 普通层 | ×5 | (100 + i×50) × 5 | (10 + i×3) × 5 |

防御 = floor(i × 1.5 × 1.5)

**奖励**：

| 层级 | 金币 | 钻石 | 经验 |
|------|------|------|------|
| Boss 层 | i × 200 | floor(i × 3) | i × 50 |
| 普通层 | i × 50 | floor(i × 0.5) | i × 20 |

### 10.3 战斗模拟（startTowerBattle）

爬塔采用回合制模拟战斗（非 liveBattle）：
- 宠物攻击 = 力量 × 2 + 10，15% 暴击（×1.5）
- 怪物攻击存活宠物（随机目标）
- 最多 30 回合，怪物 HP 归零则胜利
- 胜利后 `towerProgress++`，更新 `towerMaxFloor`，发放奖励
- 完成 5 层完成日常任务「爬塔先锋」（奖励 25 钻石）

### 10.4 每周重置

## 附录：参数公式速查表

### 主线怪物强度

```
lvScale = 1 + lv * 0.012
baseHp  = floor((40 + lv * 20) * hpMult * lvScale)
baseAtk = floor((4  + lv * 3.8) * atkMult * lvScale)
def     = floor(lv * 2.0 * defMult)
起源草地：baseHp *= 0.5, baseAtk *= 0.5
```

| 类型 | hpMult | atkMult | defMult |
|------|--------|---------|---------|
| mob | 5 | 5 | 1.2 |
| elite | 12 | 8 | 1.6 |
| boss | 24 | 12 | 2.2 |

### 宝箱掉落率

```
dropRate = baseRate * (1 + chest_drop + loot_mastery)
baseRate: mob=0.05, elite=0.10, boss=0.20
```

### 宝箱品质分布

| 怪物 | 数量 | 品质 |
|------|------|------|
| mob | 1 | white 80% / green 20% |
| elite | 1~2 | green 50% / blue 40% / purple 10% |
| boss | 1 | purple 70% / orange 30% |

### 藏宝图守卫强度

```
monsterLv = floor(playerLevel * (1 + monster_power 词条总和))
baseHp  = floor((40 + lv * 20) * hpMult * 3 * (1 + lv * 0.012) * monsterPower)
baseAtk = floor((4  + lv * 3.8) * atkMult * 3 * (1 + lv * 0.012) * monsterPower)
def     = floor(lv * 2.0 * defMult * 1.5)
```

| 品质 | hpMult | atkMult | defMult |
|------|--------|---------|---------|
| white | 1.0 | 1.0 | 1.0 |
| green | 2.4 | 1.6 | 1.2 |
| blue | 3.6 | 2.4 | 1.5 |
| purple | 5.4 | 3.6 | 2.0 |
| orange | 9.6 | 4.8 | 2.5 |

### 副本怪物强度

```
lvScale = 1 + lv * 0.012
baseHp  = floor((40 + lv * 20) * 5 * lvScale)
baseAtk = floor((4  + lv * 3.8) * 5 * lvScale)
def     = floor(lv * 2.0 * 1.2 * 1.5)
```

## v2.0.4 更新

### 一、押镖失败机制

为修复「故意失败刷阵法书」的卡 bug 行为，押镖活动的失败处理逻辑更新如下：

| 项目 | v2.0.4 前 | v2.0.4 后 |
|:---|:---|:---|
| 单阶段失败奖励 | 仍获得 1 本阵法书（consolation） | **不再奖励阵法书** |
| 累计失败处理 | 可无限继续挑战 | **累计失败 3 次 → 本次押镖彻底失败** |
| 失败代价 | 无 | 彻底失败时**消耗 1 次当日次数** |

**机制说明**：
- 单阶段失败不再发放任何阵法书奖励，仅推进至下一阶段；
- 单次押镖过程中累计失败 3 次时，本次押镖立即判定为彻底失败，进度清空并消耗 1 次当日押镖次数；

| 当日次数消耗 | 退出时 +1 | **失败即消耗 1 次当日次数** |
| 已获得技能书 | 保留 | **保留** |

**机制说明**：
- 战斗失败后**自动触发结算退出**，无需玩家手动操作；
- 失败会消耗 1 次当日技能秘境次数（与正常完成一致）；
- 已在前面阶段获得的技能书奖励**保留**，不会因失败丢失。

## 二十一、v2.1.0 更新：血色要塞 + 主线剧情 + 周常任务

### 21.1 血色要塞活动（Roguelike 无限挑战）

**新增活动**：血色要塞是一个无限轮次的 Roguelike 战斗活动，每 5 关可选增益 buff，怪物每 5 关整体强化 5%。战斗失败时按击杀数结算奖励，经验为正常的 3 倍。

#### 配置（`config.js`）

| 常量 | 说明 |
|:---|:---|
| `CRIMSON_FORTRESS_DIFFICULTIES` | 3 个难度：简单(0.7×)/普通(1.0×)/困难(1.5×)，经验倍率均为 3.0 |
| `CRIMSON_FORTRESS_BUFF_POOL` | 18 个增益 buff，按品质权重抽取 |
| `CRIMSON_FORTRESS_DAILY_MAX` | 每日 2 次挑战 |

#### Buff 池品质权重

| 品质 | 权重 | 数量 | 示例 |
|:---|:---:|:---:|:---|
| 普通(白色) | 50% | 5 | 攻击强化/防御强化/生命强化/速度强化/持续恢复 |
| 优秀(绿色) | 30% | 5 | 暴击强化/吸血强化/法术强化/减伤强化/闪避强化 |
| 稀有(蓝色) | 15% | 4 | 攻击精通/连击精通/穿透精通/暴伤精通 |
| 史诗(紫色) | 4% | 3 | 攻击大师/法术连发/不屈意志 |
| 传说(金色) | 1% | 2 | 全属性强化/战神降临 |

#### 核心机制

1. **每日 2 次**：每日重置，通过 `G.crimsonFortressUsed` 记录
2. **每 5 关增益选择**：弹出 3 个不重复 buff 供玩家 3 选 1，效果可累加
3. **怪物强化**：每 5 关整体强化 5%（`powerMult × (1 + floor(round/5) × 0.05)`）
4. **Boss 波**：每 5 关（第 5/10/15...关）出 Boss（血量 ×3，攻击 ×2.5）
5. **战斗失败结算**：按击杀数计算经验（×3 倍）和金币奖励

#### 核心函数（`economy.js`）

| 函数 | 说明 |
|:---|:---|
| `startCrimsonFortress(difficultyId)` | 开始活动（检查每日次数） |
| `generateCrimsonFortressMonster()` | 生成怪物（按轮次/难度/玩家等级） |
| `rollCrimsonFortressBuffs()` | 随机 3 个不重复 buff |
| `selectCrimsonFortressBuff(buffId)` | 选择 buff 并累加到列表 |
| `getCrimsonFortressBuffEffects()` | 获取所有已选 buff 的累加效果 |
| `beginCrimsonFortressBattle()` | 开始一轮战斗（活动战斗模态框） |
| `endCrimsonFortress()` | 结束活动（战斗失败时结算奖励） |
| `getCrimsonFortressRemaining()` | 获取今日剩余次数 |

#### UI（`ui.js`）

- 活动页新增 `fortress` tab：`🏰 血色要塞`
- `renderActivityCrimsonFortress()` 渲染函数：
  - 未开始时：显示难度选择界面 + 今日剩余次数 + 活动收获日志
  - 进行中时：显示当前关卡/击杀数/已选增益 + 挑战按钮
  - Buff 选择时：显示 3 张品质着色的 buff 卡片供选择

#### 状态字段（`state.js`）

- `G.crimsonFortress`：当前活动状态 `{ difficulty, round, kills, buffs, active, pendingBuffs, startTime }`
- `G.crimsonFortressUsed`：每日使用记录 `{ date: count }`

### 21.3 周常任务系统

**新增系统**：低频重时长玩法的周常任务池，每周一重置。

#### 任务池配置（`config.js` → `WEEKLY_TASKS`）

| 任务 ID | 任务名 | 描述 | 目标 | 奖励 |
|:---|:---|:---|:---:|:---|
| `tower_5` | 爬塔先锋 | 本周爬塔15层 | 15 | 💎100 |
| `treasure_hunt` | 寻宝猎人 | 本周完成3次打宝图 | 3 | 💎80 |
| `fuse_pet` | 融合实验 | 本周进行3次宠物融合 | 3 | 💎100 |
| `formation_escort` | 押镖勇士 | 本周参与3次押镖 | 3 | 💎80 |
| `skillbook_hunt` | 技能秘境 | 本周参与3次技能秘境 | 3 | 💎80 |
| `petcave_1` | 宠物秘境 | 本周参与3次宠物秘境 | 3 | 💎100 |
| `advance_trial` | 进阶试炼 | 本周参与1次进阶试炼 | 1 | 💎80 |
| `pet_advance` | 宠物进阶 | 本周完成1次宠物进阶 | 1 | 💎100 |
| `dispatch_finish` | 派遣归来 | 本周完成3次派遣奇遇 | 3 | 💎80 |
| `fortress` | 血色要塞 | 本周参与2次血色要塞 | 2 | 💎100 |

#### 智能路由

`updateDailyTask(taskId, amount)` 自动判断 taskId 属于日常还是周常：
- 先检查 `WEEKLY_TASKS`，匹配则写入 `G.weeklyTasks`
- 再检查 `DAILY_TASKS`，匹配则写入 `G.dailyTasks`
- 不在任何列表中的 taskId 被静默忽略

#### 重置机制

- **`getWeekKey()`**：获取当前周标识（周一日期字符串，同周内不变）
- **`checkDailyReset()`**：每周一重置 `G.weeklyTasks = {}` 并更新 `G.weeklyTaskDate`

#### 一键领取

- `claimAllDailyTasks()`：批量领取所有已完成的日常任务
- `claimAllWeeklyTasks()`：批量领取所有已完成的周常任务
- UI 显示可领取数量并提供一键领取按钮

#### 状态字段（`state.js`）

- `G.weeklyTasks`：周常任务进度对象
- `G.weeklyTaskDate`：周常任务重置标识（周一日期字符串）
