## 一、阵法系统总览

阵法系统参考《梦幻西游》玩法，是除装备、技能、血统之外的另一套团队增益体系。其核心特点如下：

| 维度 | 说明 |
|------|------|
| 阵法数量 | 7 个特色阵法（`FORMATIONS.length === 7`） |
| 阵法定位 | 全体攻击 / 全体防御 / 速度 / 法术 / 闪避 / 突击 / 平衡 |
| 位置数量 | 每个阵法定义 5 个位置加成：前 / 中 / 后 / 左 / 右 |
| 实际生效位置 | 出战队伍仅 3 个宠物，对应 `front / mid / back` 三位置 |
| 全体效果 | 每个阵法附带 `aura` 字段，定义全体型加成或机制 |
| 等级上限 | 5 级（`FORMATION_MAX_LEVEL = 5`） |
| 等级倍率 | Lv1 × 1.0，每升 1 级 +0.2，Lv5 × 1.8 |
| 升级素材 | 同名阵法书（每本 +1 经验） |
| 获取途径 | 押镖运送活动（每日 3 次，每阶段 1 本阵法书） |
| 入口位置 | 修炼入口位于「活动 → 押镖」Tab，阵法图鉴位于「阵法」主菜单 |

阵法系统代码层级：

```js
// config.js
const FORMATION_POSITIONS = [...];     // 站位定义（前/中/后）
const FORMATIONS = [...];               // 7 个阵法定义
const FORMATION_MAX_LEVEL = 5;          // 等级上限
function getFormationExpForLevel(level) // 升级所需经验
function getFormationLevelMult(level)   // 等级倍率
const FORMATION_BOOK_PREFIX = 'formation_book_';
const FORMATION_ESCORT = {...};         // 押镖活动配置

// economy.js
function learnFormation(id)             // 学习
function upgradeFormation(id)           // 单次升级（×1 书）
function upgradeFormationMax(id)         // 一键升级（连用所有书）
function setActiveFormation(id)         // 激活/切换
function getFormationBonusForPos(pos)   // 计算某位置的最终加成
function startFormationEscort()         // 押镖开始
function startFormationEscortBattle()   // 押镖战斗

// ui.js
function renderFormationScreen()        // 阵法图鉴主页
function renderActivityFormationEscort()// 押镖活动页
function changeFormation(slotIdx, newPos)// 调整站位
```

## 二、阵法位置与站位

### 2.1 三大战斗位置（front / mid / back）

游戏出战队伍固定为 3 只宠物，对应 3 个站位（`FORMATION_POSITIONS`）：

| id | 名称 | 图标 | order | 默认分配 |
|----|------|------|-------|----------|
| `front` | 前排 | 🛡️ | 0 | 出战队伍第 1 只 |
| `mid` | 中排 | ⚔️ | 1 | 出战队伍第 2 只 |
| `back` | 后排 | 🏹 | 2 | 出战队伍第 3 只 |

> 默认站位数组：`G.player.formation = ['front', 'mid', 'back']`，索引与 `G.player.activeTeam` 一一对应。

### 2.2 五位置设计（含左/右翼）

虽然实际生效位置只有 3 个，但每个阵法的 `bonus` 字段中**均定义了 5 个位置加成**（前 / 中 / 后 / 左 / 右），为后续扩展（如 5 宠队伍）预留：

```js
bonus: {
  front: { ... },   // 前排
  mid:   { ... },   // 中排
  back:  { ... },   // 后排
  left:  { ... },   // 左翼（数据预留，当前未生效）
  right: { ... },   // 右翼（数据预留，当前未生效）
}
```

> ⚠️ 当前 `getFormationBonusForPos(pos)` 只会被以 `front / mid / back` 调用，左/右翼加成数据存在但暂不参与计算。

### 2.3 出战队伍与站位绑定

宠物在出战队伍中的位置索引决定其站位：

```js
// economy.js - getPetFormationPos
function getPetFormationPos(petId) {
  if (!G.player.activeTeam) return -1;
  var idx = G.player.activeTeam.indexOf(petId);
  if (idx < 0) return -1;
  var formation = G.player.formation || ['front','mid','back'];
  return formation[idx] || 'mid';
}
```

- 出战队伍（`G.player.activeTeam`）有 3 个槽位
- 每个槽位对应 `G.player.formation[idx]` 中的站位
- 默认 `['front', 'mid', 'back']`，可在阵法界面通过下拉框调整

## 四、阵法 Aura 全体效果

每个阵法都附带 `aura` 字段，定义**不依赖站位、作用于全队**的增益（或减益）效果。该字段为数据定义，供战斗系统扩展使用。

| 阵法 | Aura 字段 | 数值 | 含义 |
|------|-----------|------|------|
| 天覆阵 | `dmgBonus` | +10% | 全体伤害加成 |
| 天覆阵 | `spdPct` | -5% | 全体速度惩罚（代价） |
| 地载阵 | `defPct` | +15% | 全体防御加成 |
| 地载阵 | `aoeReduce` | +10% | 受 AOE 伤害减免 |
| 风扬阵 | `spdPct` | +15% | 全体速度加成 |
| 风扬阵 | `critRate` | +5% | 全体暴击率加成 |
| 云垂阵 | `magicDmgPct` | +15% | 全体法术伤害加成 |
| 云垂阵 | `physDmgPct` | -5% | 全体物理伤害惩罚（代价） |
| 蛇蟠阵 | `dodgeRate` | +10% | 全体闪避加成 |
| 蛇蟠阵 | `immuneChance` | +10% | 被攻击时免疫概率 |
| 鸟翔阵 | `spdPct` | +10% | 全体速度加成 |
| 鸟翔阵 | `firstTurnExtra` | `true` | 首回合额外行动一次 |
| 龙飞阵 | `allPct` | +5% | 全属性加成 |

> **Aura 与位置加成的区别**：Aura 是阵法本身的全体效果（无需站位映射），位置加成 `bonus` 是按站位分配的差异加成。两者叠加生效。

## 五、阵法等级与经验系统

### 5.1 等级上限

```js
const FORMATION_MAX_LEVEL = 5;
```

每个阵法学习后初始为 Lv1，最高可升至 Lv5（满级）。

### 5.2 各级升级经验阈值

`getFormationExpForLevel(level)` 返回**从该级升至下一级所需经验**：

```js
function getFormationExpForLevel(level) {
  var expTable = { 1: 5, 2: 25, 3: 125, 4: 600 };
  return expTable[level] || 999999; // 满级后返回超大值
}
```

| 当前等级 | 升至下一级所需经验 | 对应阵法书数量（每本 +1 经验） |
|----------|---------------------|-------------------------------|
| Lv1 | 5 | 5 本 |
| Lv2 | 25 | 25 本 |
| Lv3 | 125 | 125 本 |
| Lv4 | 600 | 600 本 |
| Lv5 | —（已满级） | — |

### 5.3 等级倍率公式

阵法位置加成随等级线性提升：

```js
function getFormationLevelMult(level) {
  return 1 + (level - 1) * 0.2;
}
```

| 等级 | 倍率 | 计算式 |
|------|------|--------|
| Lv1 | ×1.0 | 1 + 0 × 0.2 |
| Lv2 | ×1.2 | 1 + 1 × 0.2 |
| Lv3 | ×1.4 | 1 + 2 × 0.2 |
| Lv4 | ×1.6 | 1 + 3 × 0.2 |
| Lv5 | ×1.8 | 1 + 4 × 0.2 |

**示例**：天覆阵 Lv5 时，前排攻击力加成 = `15% × 1.8 = 27%`。

> 注：`aura` 字段为固定值，**不随等级倍率变化**；等级倍率仅作用于 `bonus` 中的位置加成。

### 5.4 升级流程

`upgradeFormation(formationId)` 的核心流程：

1. 校验阵法已学习、未满级、持有 ≥1 本同名阵法书
2. 扣除 1 本阵法书
3. 经验 +1：`f.exp += 1`
4. 循环检查升级：`while (f.exp >= need) { f.exp -= need; f.level += 1; }`
5. 满级时清零经验：`f.exp = 0`
6. 保存游戏并刷新界面

**一键升级** `upgradeFormationMax(formationId)`：

- 连续消耗同名阵法书直到满级或书用尽
- 每本仍 +1 经验，达到阈值自动升级
- 安全上限 100 本/次调用，防止异常循环

### 5.5 满级总投入

将一个阵法从 Lv1 升至 Lv5 共需：

| 升级段 | 所需经验 | 累计经验 |
|--------|----------|----------|
| Lv1 → Lv2 | 5 | 5 |
| Lv2 → Lv3 | 25 | 30 |
| Lv3 → Lv4 | 125 | 155 |
| Lv4 → Lv5 | 600 | 755 |
| **合计** | — | **755 经验（即 755 本同名阵法书）** |

> 学习阵法另需消耗 1 本阵法书，所以**从零到满级共需 756 本同名阵法书**。

## 六、阵法书道具系统

### 6.1 道具 ID 规则


### 8.4 战斗胜率计算

押镖战斗为**模拟战斗**（不进入正式战斗界面），胜率公式：

```js
var cp = Math.floor(getPlayerCombatPower());              // 玩家战力
var monsterLv = playerLv + stage * 3;                     // 怪物等级
var monsterPower = monsterLv * 800 * (1 + stage * 0.3);   // 怪物强度

var winChance = 0.70 + (cp - monsterPower) / Math.max(monsterPower, 1) * 0.5;
winChance = Math.max(0.20, Math.min(0.95, winChance));   // 钳制在 [20%, 95%]
```

| 参数 | 公式 |
|------|------|
| 怪物等级 | 玩家等级 + 阶段 × 3 |
| 怪物强度 | 怪物等级 × 800 × (1 + 阶段 × 0.3) |
| 基础胜率 | 70% |
| 战力修正 | (玩家战力 - 怪物强度) / 怪物强度 × 50% |
| 胜率下限 | 20% |
| 胜率上限 | 95% |

**示例**（玩家 Lv30，第 1 阶段）：

- 怪物等级 = 30 + 1×3 = 33
- 怪物强度 = 33 × 800 × 1.3 = 34,320
- 若玩家战力 = 50,000，胜率 = 0.70 + (50000-34320)/34320 × 0.5 = 0.70 + 0.228 = 92.8%

## 九、阵法装配与生效

### 9.1 学习阵法

`learnFormation(formationId)` 流程：

1. 校验阵法存在、未学习过、持有 ≥1 本同名阵法书
2. 扣除 1 本阵法书
3. 写入学习记录：`G.formations[formationId] = { level: 1, exp: 0 }`
4. **自动激活首学的阵法**：`if (!G.activeFormation) G.activeFormation = formationId;`
5. 保存游戏，弹窗提示「已学习阵法：XX（Lv.1）」

> 学习只消耗 1 本阵法书，且初始即为 Lv1。后续每本阵法书用于升级，每本 +1 经验。

### 9.2 激活阵法

```js
function setActiveFormation(formationId) {
  if (formationId && !G.formations[formationId]) {
    showToast('未学习该阵法', 'error');
    return;
  }
  G.activeFormation = formationId || null;
  saveGame();
  showToast('🎴 已切换阵法', 'info');
}
```

- 一次只能激活 1 个阵法（`G.activeFormation` 为单值字段）
- 激活未学习的阵法会被拒绝
- 传 `null` 可取消激活

### 9.3 取消激活

阵法图鉴顶部「当前激活阵法」面板提供「取消激活」按钮，调用 `setActiveFormation(null)`，将 `G.activeFormation` 置空。取消后所有站位加成失效。

### 9.4 加成生效逻辑

```js
function getFormationBonusForPos(pos) {
  var result = { atkPct:0, defPct:0, hpPct:0, spdPct:0, intPct:0,
                 critRate:0, critDmg:0, dodgeRate:0, hitRate:0,
                 dmgReduce:0, magicDmgPct:0, mpPct:0, allPct:0 };
  if (!G.activeFormation) return result;            // 未激活：返回零加成
  var f = G.formations[G.activeFormation];
  if (!f) return result;
  var formDef = FORMATIONS.find(frm => frm.id === G.activeFormation);
  if (!formDef) return result;
  var posBonus = formDef.bonus[pos];
  if (!posBonus) return result;
  var mult = getFormationLevelMult(f.level);        // 等级倍率
  Object.keys(posBonus).forEach(function(key) {
    result[key] = (result[key] || 0) + posBonus[key] * mult;
  });
  return result;
}
```

**生效条件**：

1. `G.activeFormation` 不为空
2. `G.formations[G.activeFormation]` 存在（已学习）
3. 宠物处于出战队伍中（否则无站位）

### 9.5 阵法切换机制

- 阵法切换**无消耗、无冷却**，可在阵法图鉴界面任意切换
- 切换后立即生效（保存游戏并刷新界面）
- 切换不影响阵法等级与经验，仅替换 `G.activeFormation`
- 同一时刻仅 1 个阵法生效

## 十、阵法位置与站位效果

### 10.1 位置切换

`changeFormation(slotIdx, newPos)` 用于调整出战队伍的站位：

```js
function changeFormation(slotIdx, newPos) {
  const formation = G.player.formation || ['front','mid','back'];
  const oldPos = formation[slotIdx];
  const swapIdx = formation.indexOf(newPos);
  if (swapIdx >= 0 && swapIdx !== slotIdx) {
    formation[swapIdx] = oldPos;   // 交换：原位置宠物与目标位置宠物互换
  }
  formation[slotIdx] = newPos;
  G.player.formation = formation;
  saveGame();
  render();
}
```

**交换规则**：

- 若新位置已被其他宠物占据，则两者**互换站位**
- 若新位置未被占用，则直接设置
- 调用后立即保存并刷新

> 通过下拉框选择可调整的站位范围由 `FORMATION_POSITIONS` 决定，即 `front / mid / back` 三选一。
