## 1. 系统总览

宠物系统是本游戏的核心玩法，涵盖宠物生成、品质、资质、成长、技能、融合、重置、血统、装备、培养等子系统。所有宠物数据均由 `PET_DEX`（图鉴表）驱动，同名宠物具有确定性的种族、资质范围、成长范围、天生技能与技能库。

### 核心常量

| 常量 | 取值 | 说明 |
|---|---|---|
| `RACES` | 史莱姆 / 龙 / 恶魔 / 天使 / 哥布林 / 精灵 | 6 大种族 |
| `RARITIES` | common / uncommon / rare / epic / legendary / mythic / special | 7 种品质（含异化） |
| `RARITY_NAMES` | 普通 / 优秀 / 稀有 / 史诗 / 传说 / 神话 / 异化 | 中文映射 |
| `RARITY_COLORS` | #9ca3af / #22c55e / #3b82f6 / #a855f7 / #f59e0b / #ef4444 / #ec4899 | 品质色 |
| `APTITUDE_KEYS` | 力量资质 / 敏捷资质 / 体质资质 / 耐力资质 / 魔力资质 | 5 种资质（v2.6.0） |
| `ATTRIBUTES` | 力量 / 敏捷 / 体质 / 耐力 / 魔力 | 5 项基础属性（v2.6.0） |

## 2. 宠物生成机制

宠物主要通过 **抽蛋池（`generateEggFromPool`）** 与 **战斗掉蛋（`generateEgg`）** 两种途径获得。生成过程在 `generatePetBase` 中完成。

### 2.1 T 级概率分布（抽蛋池）

抽蛋池按 T 级抽取，T 越高越稀有。**代码实际仅到 T5，无 T6**（`ui.js` 注释明确：「最高 T5，无 T6」）。

| T 级 | 概率 | 累计阈值 | 成长上限范围 | 资质上限范围 |
|---|---|---|---|---|
| T1 | **45%** | < 0.45 | ≤ 1.4 | 800~1500 |
| T2 | **28%** | < 0.73 | 1.4 ~ 1.8 | 1000~1700 |
| T3 | **15%** | < 0.88 | 1.8 ~ 2.3 | 1300~2000 |
| T4 | **8%** | < 0.96 | 2.3 ~ 2.8 | 1600~2400 |
| T5 | **4%** | ≥ 0.96 | ≥ 2.8 | 1900~2800 |

> **关于 T6**：`getPetTier(name)` 最高返回 5，不存在 T6 宠物。但 **融合限定特殊宠物**（混元圣兽 / 灭世魔神 / 时空龙神）拥有 `tier: 6` 的专属血统，仅能通过融合 1.5% 概率获得（详见 [§12](#12-宠物融合机制)）。

### 2.2 战斗掉蛋 T 级（`getEggTierByMapLevel` / `getDropEggTier`）

> **v2.5.0 重构**：战斗掉蛋 T 级现在由**地图分段系统**统一管控，主线普通野怪掉落**封顶 T4**，T5/融合怪/神兽蛋**完全不进入**主线掉落池（仅可通过高阶副本、限时活动、限定活动转盘获取）。新增 `getDropEggTier(mapSegment)` 作为统一入口（位于 `config.js` ~line 2900），并叠加新手保护逻辑。

#### 2.2.1 旧版公式（已废弃）

旧 `getEggTierByMapLevel(mapLv)` 按地图等级线性放阶，存在「低级地图也可能掉 T5 蛋」的越阶问题，v2.5.0 起仅作内部参考，不再用于主线普通野怪掉落：

| T 级 | 旧触发条件 | 旧概率公式 |
|---|---|---|
| T5 | mapLv ≥ 25 | `min(0.10, max(0, (mapLv-25)/100 × 0.15))`，30 级 1%、60 级 5%、100 级 10% |
| T4 | mapLv ≥ 15 | `min(0.15, max(0, (mapLv-15)/100 × 0.20))` |
| T3 | mapLv ≥ 5 | `min(0.25, max(0, (mapLv-5)/100 × 0.30))` |
| T2 | 默认 | 50% |
| T1 | 兜底 | 剩余概率 |

#### 2.2.2 v2.5.0 新版地图分段掉蛋规则

11 张主线地图按解锁顺序划分为 4 个等级段（详见《05-副本与活动》§2.1），每个等级段对应固定的可掉落蛋 T 级范围：

| 地图分段 | 包含地图 | 可掉落蛋 T 级 | 说明 |
|---|---|---|---|
| 低阶段 | 1-3 图（起源草地 / 新手森林 / 幽暗矿洞） | **仅 T1** | 严禁越阶 |
| 中低阶段 | 4-6 图（亡灵墓地 / 烈焰火山 / 冰霜高原） | **T1-T2** | T2 权重随地图等级提升 |
| 中高阶段 | 7-9 图（暗影沼泽 / 天空之城 / 深渊裂隙） | **T2-T3** | T3 权重随地图等级提升 |
| 高阶段 | 10-11 图（龙之巢穴 / 终焉神殿） | **T3-T4** | 主线普通野怪**最高仅 T4** |

#### 2.2.3 新手保护规则

- 玩家通关主线地图不足 3 张时（`countClearedMaps() < 3`），所有掉落蛋强制降级为 **T1**，确保新手期不掉超阶蛋。
- T5 阶蛋、融合怪蛋、神兽蛋在**主线普通野怪掉落中权重为 0**，仅可通过：
  - 高阶副本（如血统副本、宠物秘境）
  - 限时活动
  - 限定活动转盘（常驻转盘永不产出）

#### 2.2.4 藏宝图必出蛋（`guaranteed_egg`）

藏宝图的「必出蛋」特殊词条也同步改用 `getDropEggTier()`，并额外**封顶 T3**（即使在高阶段地图也最多掉 T3 蛋），避免藏宝图成为 T5 蛋的越阶获取路径。

> 相关代码：`battle.js` ~line 2796-2799、`config.js` ~line 2900、`pet.js` `getEggTierByMapLevel`。

### 2.3 异化机制（1.5% 概率）

在 `generatePetBase` 中，每只宠物生成时进行异化判定：

```js
var isSpecial = Math.random() < 0.015;  // 1.5% 概率
```

- **异化时**：资质突破图鉴上限 10%~20%，成长突破上限 10%~20%（成长封顶 `gMax × 1.25`，资质封顶 3000）
- **正常时**：资质与成长均在图鉴上下限内随机
- **品质**：异化时直接设为 `rarity = 'special'`（异化品质），跳过品质评分流程

> 注：异化标记 `isSpecialTag` **不会**在 `generatePetBase` 中设置；该字段仅由融合路径或存档迁移设置（详见 [§19](#19-宠物展示名称规则)）。

### 2.4 抽蛋池品质加成

> **v2.5.0 调整**：抽蛋池的资质/成长额外加成比例由原 **20%** 下调至 **10%**，使抽蛋产出的宠物初始品质略降，更贴近主线掉蛋的产出曲线。其余蛋池规则（T 级概率、异化判定等）保持不变。

抽蛋池生成的宠物会传入 `qualityBoost = 0.10`，使品质评分提高 10%，更容易出高品质宠物。

```js
// ui.js → generateEggFromPool（~line 2429）
// 需求14：抽蛋池资质/成长额外加成由原20%下调至10%
var pet = generatePetBase(chosenName, 0.10);
```

> 历史值：v2.5.0 前为 `0.20`；中间曾短暂使用 `0.15`；v2.5.0 起固定为 `0.10`。

### 2.5 指定名生成（`generateEggFromName`）

`pet.js` 中新增 `generateEggFromName(petName)`（约 line 646），按指定宠物名生成蛋，用于**藏宝图奖励**：

- 直接以传入的 `petName` 为基础生成宠物
- 使用该宠物的 T 级（`getPetTier`）决定孵化时间与稀有度上限
- 由 `processTreasureRewards`（`battle.js` ~line 2044）在藏宝图掉蛋时调用，蛋池来源于 `getTreasureMapPetPool(tmap.mapType)`
- 传说藏宝图 `guaranteed_egg` 特殊词条触发时，从蛋池中挑选 T 级最高的宠物生成必出蛋

> 该函数与 `generateEgg`（按 T 级随机抽名）的区别：`generateEggFromName` 是「先有名再生成」，保证掉蛋落在该藏宝图地图对应的宠物池内。

## 3. 宠物品质系统

### 3.1 品质层级

| 索引 | 英文 ID | 中文名 | 颜色 | 说明 |
|---|---|---|---|---|
| 0 | common | 普通 | #9ca3af 灰 | 最低品质 |
| 1 | uncommon | 优秀 | #22c55e 绿 | |
| 2 | rare | 稀有 | #3b82f6 蓝 | |
| 3 | epic | 史诗 | #a855f7 紫 | |
| 4 | legendary | 传说 | #f59e0b 橙 | |
| 5 | mythic | 神话 | #ef4444 红 | 非异化最高品质 |
| 6 | special | 异化 | #ec4899 粉 | 仅 1.5% 异化或融合特殊触发 |

### 3.2 品质分布（qualityScore 阈值）

非异化宠物的品质由 `qualityScore`（成长×0.45 + 资质×0.55）决定，分布如下：

> **v2.9.0 重构**：还原完整 6 档品质体系，全部产出渠道（主线/转盘/藏宝图/副本）使用下表阈值。抽蛋池额外屏蔽「普通」档（详见 [§3.4](#34-产出渠道隔离v290)）。

| 品质 | qualityScore 区间（v2.9.0 全渠道） | 大致占比 |
|---|---|---|
| 普通 common | < 0.20 | 20% |
| 优秀 uncommon | 0.20 ~ 0.40 | 20% |
| 稀有 rare | 0.40 ~ 0.60 | 20% |
| 史诗 epic | 0.60 ~ 0.78 | 18% |
| 传说 legendary | 0.78 ~ 0.90 | 12% |
| 神话 mythic | ≥ 0.90 | 10% |

> 异化（special）独立判定，1.5% 概率，不计入上表。
> 历史阈值：v2.0.5 前（史诗 0.85~0.95 / 传说 0.95~0.99 / 神话 ≥0.99）；v2.0.5~v2.8.0（史诗 0.85~0.90 / 传说 0.90~0.95 / 神话 ≥0.95）。

### 3.3 品质刷新（`recalcRarity`）

宠物在融合、重生、月华露培养后，会根据当前成长/资质重新计算品质：

- 异化品质（special）**保持不变**，不参与重算
- 神兽品质固定为 mythic，不参与重算（修复品质衰减 Bug）
- 成长/资质超过图鉴上限时，对应得分按满分 1.0 计算
- 可选 `qualityBoost` 参数：`qualityScore = min(1, qualityScore × (1 + qualityBoost))`
- **v2.9.0**：`recalcRarity` 统一使用全 6 档阈值（普通<0.20 / 优秀<0.40 / 稀有<0.60 / 史诗<0.78 / 传说<0.90 / 神话≥0.90），与 `generatePetBase` 的全渠道产出路径一致

### 3.4 产出渠道隔离（v2.9.0）

`generatePetBase(name, qualityBoost, excludeCommon)` 新增第三个参数 `excludeCommon`，用于区分产出渠道的可掉落品质范围：

| 产出渠道 | excludeCommon | 可产出品质档 | 阈值（qualityScore） |
|---|---|---|---|
| 主线掉落 / 转盘 / 藏宝图 / 副本 | `false`（默认） | 全部 6 档 | 普通<0.20 / 优秀<0.40 / 稀有<0.60 / 史诗<0.78 / 传说<0.90 / 神话≥0.90 |
| 抽蛋池（`generateEggFromPool`） | `true` | 优秀及以上 5 档 | 优秀<0.35 / 稀有<0.55 / 史诗<0.72 / 传说<0.86 / 神话≥0.86 |

- 抽蛋池通过 `excludeCommon=true` 屏蔽「普通」档，并将优秀~神话档阈值整体上移，使抽蛋产出更稳定地落在中高品质
- 主线掉落、转盘、藏宝图、副本等渠道调用 `generatePetBase` 时不传 `excludeCommon`，按全 6 档分布产出
- 异化（special）独立判定，1.5% 概率，不受 `excludeCommon` 影响

## 4. 资质系统

### 4.1 五种资质（v2.6.0）

| 资质 | 影响属性 | 默认范围 |
|---|---|---|
| 力量资质 | 力量 | 1000~3000 |
| 敏捷资质 | 敏捷 | 1000~3000 |
| 体质资质 | 体质 | 1000~3000 |
| 耐力资质 | 耐力 | 1000~3000 |
| 魔力资质 | 魔力 | 1000~3000 |

> **v2.6.0 变更**：原「智力资质」重命名为「魔力资质」；新增「耐力资质」（默认 = 体质资质）。旧存档「智力资质」自动迁移为「魔力资质」，并补设「耐力资质」。

### 4.2 资质范围（图鉴上下限）

每只宠物在 `PET_DEX[name].aptRange` 中定义了 5 项资质的 `[下限, 上限]`（v2.6.0），例如：

| 宠物 | 力量资质 | 敏捷资质 | 体质资质 | 耐力资质 | 魔力资质 |
|---|---|---|---|---|---|
| 小火焰 | 1400~2000 | 1100~1500 | 1000~1400 | 1000~1400 | 1200~1600 |
| 创世神龙 | 2300~2900 | 2000~2600 | 2200~2800 | 2200~2800 | 2200~2800 |
| 混元圣兽（融合限定） | 2500~3000 | 2200~2800 | 2500~3000 | 2500~3000 | 2300~2900 |

- **正常生成**：`aptitude[k] = randomInt(lo, hi)`，在图鉴范围内随机
- **异化生成**：`aptitude[k] = min(3000, floor(hi × (1 + 10%~20%)))`，突破上限
- **融合生成**：在两只宠物该项资质的 `[min, max]` 区间内随机
- **重置生成**：在图鉴 `[下限, 上限]` 内重新随机
- 资质上限统一封顶 **3000**

### 4.3 资质得分（用于品质评分）

```
单项资质得分 = (当前资质 - 下限) / (上限 - 下限)   // 0~1
平均资质得分 aptScore = 5 项资质得分之和 / 5   // v2.6.0：4 项 → 5 项
```

## 5. 成长值系统

### 5.1 成长范围

每只宠物在 `PET_DEX[name].growthRange` 中定义 `[成长下限, 成长上限]`：

| T 级 | 成长范围典型值 |
|---|---|
| T1 | 0.5 ~ 1.2 |
| T2 | 0.9 ~ 1.7 |
| T3 | 1.4 ~ 2.2 |
| T4 | 1.8 ~ 2.6 |
| T5 | 2.3 ~ 3.0+ |
| 融合限定 | 2.5 ~ 3.5 |

### 5.2 成长生成规则

- **正常生成**：`growth = round(randomFloat(gMin, gMax) × 100) / 100`（保留 2 位小数）
- **异化生成**：`growth = min(gMax × (1 + 10%~20%), gMax × 1.25)`，突破上限封顶 1.25 倍
- **融合生成**：在两只宠物成长的 `[min, max]` 区间内随机
- **月华露培养**：每次 +0.02~0.05，封顶 3.5（详见 [§16](#16-月华露培养)）
- **全局成长上限**：3.5

### 5.3 成长得分

```
gScore = (当前成长 - 成长下限) / (成长上限 - 成长下限)   // 0~1
```

## 6. 品质评分公式

### 6.1 核心公式

```
qualityScore = gScore × 0.45 + aptScore × 0.55
```

- **成长占比 45%**
- **资质占比 55%**
- 抽蛋池额外加成（v2.5.0）：`qualityScore = min(1, qualityScore × (1 + 0.10))`（v2.5.0 前为 0.20）

### 6.2 品质判定流程

```
1. 计算 gScore（成长得分，0~1）
2. 计算 aptScore（5 项资质平均得分，0~1，v2.6.0）
3. qualityScore = gScore × 0.45 + aptScore × 0.55
4. 若 qualityBoost：qualityScore = min(1, qualityScore × (1 + qualityBoost))
5. 按 qualityScore 阈值映射到品质（详见 §3.2）
```

## 7. 属性计算公式

> **v2.6.0 重大变更**：属性体系由原「四维属性（力量/体质/敏捷/智力）」升级为「五维属性（力量/敏捷/体质/耐力/魔力）」。移除原「智力」属性，新增「耐力」属性（大幅提升物理防御力），原「智力」重命名为「魔力」（大幅提升法术攻击力/法术防御力）。属性点获取与转化规则全面重构。

### 7.1 属性点获取规则（v2.6.0）

宠物每提升 1 级，固定获得 **10 点属性点**，分配如下：

| 类别 | 分配方式 | 合计 |
|---|---|---|
| 固定属性点 | 力量、敏捷、体质、耐力、魔力各 +1 点 | 5 点 |
| 自由属性点 | 玩家可自由分配到任意五维属性上 | 5 点 |
| **合计** | | **10 点/级** |

- 固定属性点保证五维均衡成长，自由属性点体现玩家培养方向
- 玩家可通过分配自由属性点强化主属性（如物理系主加力量、法术系主加魔力、防御系主加耐力）

### 7.2 属性点转化公式（v2.6.0）

单点五维属性转化为战斗值的公式：

```
单属性点对应战斗值 = 1 + (对应单项资质 / 10000) × (1 + (成长值 + 种族值) / 10)
```

- **资质影响**：资质越高，单点属性转化的战斗值越高（资质 1000 时系数 0.1，资质 3000 时系数 0.3）
- **成长影响**：成长值越高，单点属性转化的战斗值越高
- **种族值影响**：种族值越高，单点属性转化的战斗值越高，体现种族特长

### 7.3 五维 → 战斗属性收益规则（v2.6.0）

| 五维属性 | 核心收益 | 次要收益 |
|---|---|---|
| 力量 | 大幅提升物理攻击力 | 少量提升灵力、命中 |
| 敏捷 | 大幅提升战斗速度 | 提升闪避率 |
| 体质 | 大幅提升生命值上限 | 少量提升灵力 |
| 耐力 | 大幅提升物理防御力 | 少量提升灵力 |
| 魔力 | 大幅提升法术攻击力、法术防御力 | 提升法力值上限 |

> **设计说明**：原「智力」同时影响法术攻击与法术防御；v2.6.0 重命名为「魔力」后定位不变，但与新增的「耐力」（专注物理防御）形成更清晰的攻防分工。

### 7.4 等效成长与资质（含装备加成）

宠物装备会改变等效成长与资质：

```
effectiveGrowth = growth + 装备成长加成
effectiveApt.X  = aptitude.X + 装备资质加成
```

然后再代入 §7.2 转化公式。

### 7.5 示例（体质属性点转化）

```
体质单点战斗值 = 1 + (体质资质 / 10000) × (1 + (成长值 + 种族值.体质) / 10)

例如：体质资质 2000、成长 2.0、种族值.体质 2.0
单点战斗值 = 1 + (2000/10000) × (1 + (2.0+2.0)/10)
          = 1 + 0.2 × 1.4
          = 1.28
```

## 8. 派生属性

> **v2.6.0 变更**：派生属性由原「四项基础属性」改为「五项基础属性（力量/敏捷/体质/耐力/魔力）」派生。新增「耐力」专门提供物理防御，原「智慧」相关收益由「魔力」承接。

由五项基础属性派生出的战斗属性（与 §7.3 收益规则对应）：

| 派生属性 | 主要来源 | 次要来源 |
|---|---|---|
| 气血（生命值上限） | 体质（大幅） | 等级加成 |
| 法力值上限 | 魔力（大幅） | 等级加成 |
| 物理攻击力 | 力量（大幅） | 敏捷（少量）、命中 |
| 物理防御力 | 耐力（大幅） | — |
| 法术攻击力 | 魔力（大幅） | — |
| 法术防御力 | 魔力（大幅） | — |
| 灵力 | 魔力（主要） | 力量、体质、耐力（少量） |
| 速度 | 敏捷（大幅） | — |
| 闪避率 | 敏捷（大幅） | — |
| 命中 | 力量（少量） | — |

> **与旧版差异**：旧版防御力仅由体质提供（`体质 × 0.6`），v2.6.0 起物理防御力主来源改为**耐力**；旧版灵力仅由智慧提供（`智慧 × 2`），v2.6.0 起灵力主来源改为**魔力**，且力量/体质/耐力提供少量灵力加成。

### 8.1 最终属性（含全部加成）

最终属性还需叠加：宝石加成、人物属性加成、天赋倍率、宠物装备基础词条、套装百分比、阵法百分比、血统加成等。

以最终气血为例：

```
finalHp = round( (气血 + 人物气血加成 + 装备气血基础词条) × hpMult )
finalHp = round( finalHp × (1 + petHpBonus + setHpPct + setAllPct) )
```

其中 `hpMult = 1 + 天赋加成(pet_hp 系列 + combat_mastery + combat_ultimate)`。

## 11. 宠物技能库

### 11.1 技能类型

| type | 说明 | 占技能格 |
|---|---|---|
| `bloodline` | 血统技能 | 否（不占格） |
| `active` | 主动技能 | 是 |
| `passive` | 被动技能 | 是 |
| `aura` | 光环技能 | 是 |

### 11.2 技能库生成（`getPetSkillLib`）

每只宠物有**固定的技能库**（同名宠物技能库一致），生成规则：

1. **必带天生技能**：`dex.innateSkills` 全部加入
2. **候选池累加**：按专长 `SPEC_SKILL_POOL` + T3/T4/T5 高阶池
3. **baseId 去重**：避免同 baseId 的多 tier 技能重复
4. **种子选取**：用宠物名哈希作确定性种子，从候选池补充到 `maxSkills` 个

### 11.3 技能库分层

| 候选池 | 适用 T 级 |
|---|---|
| `SPEC_SKILL_POOL` | T1~T2 基础池 |
| `SPEC_SKILL_POOL_T3` | T3+ 高阶池 |
| `SPEC_SKILL_POOL_T4` | T4+ 超高阶池 |
| `SPEC_SKILL_POOL_T5` | T5 神级池 |

每个专长（physical / magic / defense / speed / balanced）都有对应的池子。

### 11.4 技能获得规则（孵化时）

```
1. 天生技能（dex.innateSkills）必带
2. 总技能数 = randomInt(max(1, 天生数), maxSkills)
3. 额外技能数 = 总技能数 - 天生数
4. 从技能库剩余技能中随机抽取 extraCount 个
```

因此宠物可能只有 1 个天生技能，也可能满技能（4~6 个）。

### 11.5 技能等级（tier）

技能 ID 形如 `flame_strike`（tier 1）、`flame_strike_2`（tier 2）、`flame_strike_3`（tier 3）。`getSkillBaseId` 去除后缀得到 baseId，`getSkillTier` 返回 tier 数字。

## 12. 宠物融合机制

### 12.1 融合入口

消耗 **2 只宠物 + 1 个融合石** 进行融合。融合在 `fusePets(pet1, pet2)` 中实现。

### 12.2 融合路径分支

```
fusePets(pet1, pet2)
  ├─ 1.5% 概率 → 融合限定特殊宠物（generateFusionOnlyPet）
  └─ 98.5% → 普通融合路径
              ├─ 3% 概率 → 异化变异（rarity='special'，资质×1.2，成长×1.3）
              └─ 97% → 普通融合结果
```

### 12.3 融合限定特殊宠物（1.5%）

| 宠物名 | 种族 | 专属血统 | 定位 |
|---|---|---|---|
| 混元圣兽 | 龙 | 混元之血（chaos_blood） | 均衡 |
| 灭世魔神 | 恶魔 | 灭世之炎（inferno_blood） | 物理 |
| 时空龙神 | 龙 | 时空之血（timewalker_blood） | 法术 |

- 固定 `rarity = 'special'`（异化品质）
- 资质：`min(3000, max(两只较高值 × 1.2, 图鉴上限))`
- 成长：`min(3.5, round(max(两只较高成长 × 1.15, 图鉴成长上限) × 100) / 100)`
- 天生技能：图鉴独有技能全部必带

### 12.4 普通融合规则

**基础宠物**：50% 概率从 pet1 或 pet2 中随机选作基础（继承名字/种族/血统）。

**成长融合**：
```
gMin = min(pet1.growth, pet2.growth)
gMax = max(pet1.growth, pet2.growth)
newPet.growth = round(randomFloat(gMin, gMax) × 100) / 100
```

**资质融合**（每项独立）：
```
lo = min(pet1.aptitude[k], pet2.aptitude[k])
hi = max(pet1.aptitude[k], pet2.aptitude[k])
newPet.aptitude[k] = randomInt(lo, hi)
```

### 12.5 技能继承规则

收集两只宠物所有普通技能（不含血统），按 baseId 聚合：

| 情形 | 概率 | 结果 |
|---|---|---|
| 相同技能（两只都有，baseId 相同） | **50%** | 升级（tier+1，封顶 3）；否则保留原技能 |
| 不同技能（仅一只拥有） | **60%** | 保留；否则丢弃 |

- 至少保留 1 个技能（若全部失败则随机保留 1 个）
- **融合不限制技能数量**，可超过 6 格
- 所有保留技能标记为 `isInnate: true`

### 12.6 融合后品质

```
newPet.rarity = recalcRarity(newPet)   // 根据新成长/资质重算
```

### 12.7 融合异化变异（3%）

普通融合后有 3% 概率触发异化变异：

```
newPet.rarity = 'special'
newPet.isSpecialTag = true              // 展示层加 ★前缀·异
资质 × 1.2（封顶 3000）
成长 = min(成长 × 1.3, 3.5)
```

> 注意：融合路径不修改 `newPet.name`，避免 `getPetDex/getPetTier/getPetSkillLib` 查找失败。

### 13.2 重置规则（`resetPet`）

### 13.4 宠物洗点丹（v2.9.0 新增）

新增道具「宠物洗点丹」（`pet_reset_pill`），用于重置指定宠物的全部**自由属性点**，全额返还为未分配状态，供玩家重新分配。

#### 13.4.1 道具信息

| 字段 | 值 |
|---|---|
| 道具 ID | `pet_reset_pill` |
| 名称 | 宠物洗点丹 |
| 图标 | 🧪 |
| 商店价格 | 15 钻石 |
| 品质 | high |
| 描述 | 重置宠物全部自由属性点，全额返还 |

#### 13.4.2 使用规则（`resetPetAttrPoints`）

- **保留**：固定属性点（每维 = 宠物等级，每级各维固定 +1）
- **重置**：自由属性点全额返还到 `pet.freeAttrPoints`
- **不影响**：成长、资质、技能、等级、血统、符文、宝石、装备等
- 计算公式：
  - 总属性点 = `pet.level × getPetAttrPointsPerLevel()`（每级 10 点）
  - 固定点（每维）= `pet.level`
  - 可自由分配点数 = `总点数 - 固定点 × 5`
- 使用后清除 `pet._statsCache`，下次 `getPetStats` 调用时重新计算

| 渠道 | 配置 |

| 传奇宝箱奖池 | weight = 4 |
| 商城 | 15 钻石/个 |

#### 13.4.4 使用流程

1. 背包内点击道具 → `showPetResetPillModal()` 弹窗
2. 弹窗内选择目标宠物（显示当前自由点数、已分配五维）
3. 二次确认 → `applyPetResetPill(petId)` 执行洗点
4. 返回 `{ ok, pet, refunded }`，`refunded` 为返还的自由点数

## 18. 宠物战斗力计算

### 18.1 战力权重（`CP_WEIGHTS`）

| 属性 | 权重 |
|---|---|
| 气血 | 0.3 |
| 攻击力 | 3 |
| 防御力 | 2 |
| 灵力 | 2.5 |
| 速度 | 1.5 |

### 18.2 战力构成（6 部分）

```
总战力 = statCp × skillMult
statCp = baseCp + gemCp + charCp + talentCp + petBonusCp
skillCp = statCp × (skillMult - 1)
```

| 部分 | 说明 |
|---|---|
| 1. baseCp | 基础属性战力（资质+成长+等级+种族值，无加成） |
| 2. gemCp | 宝石加成战力 |
| 3. charCp | 人物属性加成战力（cb 中的 flat 属性） |
| 4. talentCp | 天赋倍率战力（倍率-1 部分） |
| 5. petBonusCp | 装备特效战力（petHp/petDef 百分比加成） |
| 6. skillCp | 技能加成战力（skillMult-1 部分） |

### 18.3 基础战力公式

```
基础属性 → 派生属性 → baseCp
baseCp = 气血×0.3 + 攻击力×3 + 防御力×2 + 灵力×2.5 + 速度×1.5
```

### 18.4 技能倍率（`getSkillPowerMultiplier`）

```
mult = 1.0
+ 主动技能平均伤害系数超出 1.0 部分 × 0.5
+ 主动技能数量加成（每个 +3%，上限 8 个）
+ 被动 atkBonus × 1.0
+ 被动 critRate × 0.6
+ 被动 critDmg × 0.3
+ 被动 skillDmg × 0.8
+ 被动 ignoreDef × 0.5
+ 被动 doubleChance × 0.6
+ 被动 tripleChance × 0.8
+ 被动 skillDouble × 0.5
+ 被动 vampPct × 0.4
+ 被动 firstStrike × 0.2
+ 生存型被动（dmgReduce/regenPct/parryChance/dodgeBonus/reflectPct/antiCrit）× 较小系数

若 rarity === 'special'：mult × 1.2
```

> **异化品质额外 ×1.2 加成**，体现异化宠物的战力优势。

### 18.5 总战力

```
total = round(statCp × skillMult)
```

可通过 `getCombatPowerBreakdown(pet)` 获取详细分解，`getPetCombatPower(pet)` 返回 `total`。

## 19. 宠物展示名称规则

### 19.1 `getPetDisplayName(pet)`

```js
function getPetDisplayName(pet) {
  if (!pet) return '';
  var base = pet.customName || pet.name || '';
  if (pet.isSpecialTag) return '★' + base + '·异';
  return base;
}
```

### 19.2 规则表

| 条件 | 展示名 |
|---|---|
| 无 customName 且非异化 | 真名（pet.name） |
| 有 customName 且非异化 | 自定义名（pet.customName） |
| 无 customName 且 isSpecialTag=true | `★真名·异` |
| 有 customName 且 isSpecialTag=true | `★自定义名·异` |

### 19.3 isSpecialTag 设置位置

`isSpecialTag` 字段在以下位置被设置为 `true`：

1. **融合异化变异**（`fusePets`，3% 路径）：`newPet.isSpecialTag = true`
2. **存档迁移**（`state.js`）：为已有 `rarity='special'` 的宠物补设该字段

> 注意：`generatePetBase` 中 1.5% 异化路径**仅设置** `rarity='special'`，**不设置** `isSpecialTag`。因此初始孵化出的异化宠物在展示名上不会自动加 `★前缀·异`（除非经过存档迁移补设）。展示名前缀仅用于展示，**不影响** `getPetDex / getPetTier` 查找。

### 21.2 进阶系统配置（`EVOLVE_SYSTEM_CONFIG`）

`config.js` 中定义全局进阶常量：

```js
var EVOLVE_SYSTEM_CONFIG = {
  ADVANCE_GROWTH_MULT: 1.3,       // 每次进阶成长提升倍率
  ADVANCE_APTITUDE_MULT: 1.3,     // 每次进阶资质提升倍率
  ADVANCE_VALUE_MAX_T1_TO_T3: 1000, // T1→T3 进阶值上限
  ADVANCE_VALUE_MAX_T3_TO_T5: 2000, // T3→T5 进阶值上限
  ITEMS: {
    low:  { id: 'evolution_crystal_low',  name: '低级进化晶石', baseValue: 15,  critChance: 0.08 },
    mid:  { id: 'evolution_crystal_mid',  name: '中级进化晶石', baseValue: 50,  critChance: 0.08 },
    high: { id: 'evolution_crystal_high', name: '高级进化晶石', baseValue: 150, critChance: 0.08 },
  },
  CRIT_MIN: 2,  // 暴击最低倍率
  CRIT_MAX: 9,  // 暴击最高倍率
};
```

| 参数 | 取值 | 说明 |
|---|---|---|
| 每次进阶成长倍率 | ×1.3 | 成长值在原基础上直接乘 1.3 |
| 每次进阶资质倍率 | ×1.3 | 5 项资质均在原基础上直接乘 1.3（非随机重置，v2.6.0：4 项 → 5 项） |
| T1→T3 进阶值上限 | 1000 | 累积 advanceValue 达到 1000 触发进阶 |
| T3→T5 进阶值上限 | 2000 | 累积 advanceValue 达到 2000 触发进阶 |
| 暴击触发概率 | 8% | 每次使用晶石时独立判定 |
| 暴击倍率范围 | 2~9 倍 | 触发暴击时随机取整数倍率 |

> **与旧系统的关键区别**：旧系统进阶时**重置**资质/成长（按新图鉴随机），新系统进阶时在**当前值基础上 ×1.3**（保留个体差异，培养投入不浪费）。

### 21.4 暴击机制（`calculateEvolutionCrystalValue`）

`pet.js → calculateEvolutionCrystalValue(itemTier)`：

```
baseValue = EVOLVE_SYSTEM_CONFIG.ITEMS[itemTier].baseValue
crit = Math.random() < 0.08          // 8% 概率触发暴击
critMult = crit ? randomInt(2, 9) : 1  // 暴击时随机 2~9 倍
value = baseValue × critMult          // 实际获得的进阶值
```

| 晶石 | 非暴击 | 最低暴击(×2) | 最高暴击(×9) |
|---|---|---|---|
| 低级 | 15 | 30 | 135 |
| 中级 | 50 | 100 | 450 |
| 高级 | 150 | 300 | 1350 |

> 暴击时 UI 显示「✨ 暴击×N！」提示。

### 21.5 进阶流程

#### 获取进阶信息（`getPetEvolveInfo`）

`config.js → getPetEvolveInfo(pet)` 返回该宠物的进阶链信息：

| 返回字段 | 含义 |
|---|---|
| `canEvolve` | 是否可继续进阶 |
| `maxed` | 是否已达到最高形态 |
| `evolveType` | 'T1' 或 'T3'（链起始 T 级） |
| `currentStage` | 当前进阶阶段（0=初始 / 1=第一次进阶 / 2=第二次进阶） |
| `maxStage` | 最大进阶阶段（T1 链=2，T3 链=1） |
| `nextName` | 下一阶段宠物名 |
| `targetTier` | 目标 T 级（3 或 5） |
| `chain` | 进阶链宠物名数组 |
| `advanceValueMax` | 当前阶段进阶值上限（1000 或 2000） |

#### 使用进化晶石（`useEvolutionCrystal`）

`economy.js → useEvolutionCrystal(petId, itemTier)`：

1. 校验宠物存在且 `evolveInfo.canEvolve` 为真
2. 检查道具库存，消耗 1 个对应品质的进化晶石
3. 调用 `calculateEvolutionCrystalValue(itemTier)` 计算获得进阶值（含暴击）
4. `pet.advanceValue += gain`
5. 若 `pet.advanceValue >= evolveInfo.advanceValueMax` → 自动调用 `advancePetEvolve(petId)` 执行进阶
6. 返回 `{ ok, msg, crit, critMult, gain, advanced }` 供 UI 展示

#### 执行进阶（`advancePetEvolve`）

`pet.js → advancePetEvolve(petId)`：

1. 校验 `advanceValue >= advanceValueMax`
2. **宠物名变更**：`pet.name = evolveInfo.nextName`（影响 `getPetDex / getPetTier / getPetSkillLib` 查找）
3. **`pet.advanceStage += 1`**
4. **属性提升**（非重置，在当前值基础上乘倍率）：
   - 资质：`pet.aptitude[k] = Math.floor(pet.aptitude[k] × 1.3)`
   - 成长：`pet.growth = Math.round(pet.growth × 1.3 × 100) / 100`
5. **技能槽解锁**：根据目标 T 级补充天生技能
   - 目标 T3：补充至至少 2 个天生技能（从 `SPEC_SKILL_POOL_T3` 按专长选取）
   - 目标 T5：补充至至少 3 个天生技能（从 `SPEC_SKILL_POOL_T5` 按专长选取）
   - T5 满技能格 = 6 格
6. **`pet.advanceValue = 0`**（清零，可继续累积到下一阶）
7. **更新 `pet.advanceable`**：检查是否还能继续进阶
8. **重新计算品质**：`pet.rarity = recalcRarity(pet)`

#### 动态图鉴生成（`getEvolvePetDex`）

进阶后的宠物名可能不存在于 `PET_DEX` 中（如「星光战鹰」「烈焰龙王」等），由 `config.js → getEvolvePetDex(baseName, stage)` 动态生成图鉴条目：

- 资质范围 = 基础图鉴 × `1.3^stage`
- 成长范围 = 基础图鉴 × `1.3^stage`
- 天生技能按目标 T 级从专长技能池补充
- `getPetDex(name)` 会自动检测新系统进阶名并调用此函数

> **旧系统兼容**：`PET_ADVANCE_CHAINS`（50 条旧链）仍保留于 `config.js`，`getPetDex` 会先检查新系统（`evolveChain`），再检查旧系统（`PET_ADVANCE_CHAINS`），确保旧存档中已进阶的宠物名仍可正确解析。

### 21.6 进阶页 UI（`renderAdvanceSheet`）

## v2.0.5 更新

v2.0.5 对宠物系统进行三项重要调整：宠物品质判断区间调整、血统技能重做（增加非属性加成效果）、建立统一图鉴库系统。本节详述三项变更的实现细节。

### 一、宠物品质判断调整（需求11）

v2.0.5 调整 `recalcRarity(pet, qualityBoost)` 中的品质分数区间，使史诗品质更容易达到，神话品质更稀有。

#### 1. 区间对比

| 品质 | v2.0.5 前 qualityScore 区间 | v2.0.5 后 qualityScore 区间 | 调整方向 |
|:---:|:---:|:---:|:---|
| 普通 common | < 0.40 | < 0.40 | 不变 |
| 优秀 uncommon | 0.40 ~ 0.65 | 0.40 ~ 0.65 | 不变 |
| 稀有 rare | 0.65 ~ 0.85 | 0.65 ~ 0.85 | 不变 |
| 史诗 epic | 0.85 ~ 0.95 | **0.85 ~ 0.90** | 区间收窄，更易达到 |
| 传说 legendary | 0.95 ~ 0.99 | **0.90 ~ 0.95** | 区间下移 |
| 神话 mythic | ≥ 0.99 | **≥ 0.95** | 阈值降低，但实际达到难度提升 |

#### 2. 调整逻辑

- **史诗更易达到**：区间从 0.10 宽度收窄至 0.05，玩家更容易进入史诗档；
- **神话更稀有**：阈值从 0.99 降至 0.95，看似更宽松，但传说档区间被压缩到 0.05，达到 0.95 难度大幅提升；
- **品质分数计算不变**：`qualityScore = gScore × 0.45 + aptScore × 0.55`，仅区间阈值调整。

#### 3. 影响范围

| 触发场景 | 函数 | 影响 |
|:---|:---|:---|
| 宠物生成 | `generatePetBase` | 生成时按新区间判定初始品质 |
| 月华露培养 | `applyMoonDew` | 提升成长后按新区间重算品质 |
| 宠物融合 | `fusePets` | 融合后按新区间重算品质 |
| 宠物重置 | `resetPet` | 重置后按新区间重算品质 |
| 宠物进阶 | `advancePetEvolve`（v2.4.0）/ ~~`advancePet`~~（旧，已移除） | 进阶后按新区间重算品质 |

#### 4. 异化品质不受影响

异化品质（special）独立判定（1.5% 概率），不参与 `recalcRarity` 重算，区间调整**不影响**异化宠物的生成与展示。

### 二、血统技能重做：5个专长招牌特殊效果（需求10）

v2.0.5 对宠物血统技能进行全面重做，**每个专长（specialty）都附带一个"招牌特殊效果"**，使血统技能不再只是单纯的属性加成，而是为宠物提供独特的战斗机制。

#### 1. 血统专长预设模板（BLOODLINE_EFFECT_PRESETS）

`config.js` 中 `BLOODLINE_EFFECT_PRESETS` 按 5 种专长预设效果模板，每个专长包含基础属性加成 + 招牌特殊效果：

```js
var BLOODLINE_EFFECT_PRESETS = {
  physical: { atkPct: 0.10, critRate: 0.05, critDmg: 0.10, lifestealPct: 0.08 },
  magic:    { intPct: 0.10, magicDmgPct: 0.10, mpRegenPct: 0.05,
              burnChance: 0.15, burnTurns: 3, burnPct: 0.05 },
  defense:  { defPct: 0.10, hpPct: 0.10, dmgReduce: 0.05,
              reflectChance: 0.20, reflectPct: 0.25 },
  speed:    { spdPct: 0.10, dodgeRate: 0.05, critRate: 0.05,
              counterChance: 0.15, counterPct: 0.50 },
  balanced: { allPct: 0.05, hpPct: 0.05, atkPct: 0.05, regenPct: 0.05 },
};
```

#### 2. 5 个专长招牌特殊效果

| 专长 | 招牌特殊效果 | 字段 | 数值 | 触发条件 |
|:---:|:---|:---|:---:|:---|
| 物理系 physical | **攻击吸血** | `lifestealPct` | 8% | 主动技能造成伤害时按伤害值吸血 |
| 法术系 magic | **攻击附带灼烧** | `burnChance` / `burnTurns` / `burnPct` | 15% / 3 回合 / 5% | 主动技能攻击时概率触发，每回合造成最大气血 5% 伤害 |
| 防御系 defense | **受击反弹** | `reflectChance` / `reflectPct` | 20% / 25% | 受到怪物攻击时概率反弹 25% 伤害 |
| 速度系 speed | **受击反击** | `counterChance` / `counterPct` | 15% / 50% | 受到怪物攻击时概率反击，造成攻击力 50% 伤害 |
| 均衡系 balanced | **每回合回血** | `regenPct` | 5% | 每回合恢复最大气血 5% |

#### 3. 血统品质放大规则

血统珠品质倍率（`BLOOD_ORB_QUALITY_MULT`）作用于血统效果：

| 项目 | 放大规则 |
|:---|:---|
| 数值类效果（atkPct/hpPct/defPct/lifestealPct/burnPct/reflectPct/counterPct/regenPct） | **按品质倍率放大** |
| 概率类效果（burnChance/reflectChance/counterChance） | **不放大**，保持基础概率 |

> 设计原则：概率类效果若放大，高阶血统珠会导致 100% 触发率，破坏平衡。

#### 4. 进阶 stage 倍率

血统效果按宠物进阶阶段（`advanceStage`）自动放大，无需重新抽取血统珠：

| advanceStage | 形态 | 倍率 |
|:---:|:---|:---:|
| 0 | 基础 T1 | ×1.0 |
| 1 | 中阶 T3 | ×1.5 |
| 2 | 顶阶 T5 | ×2.0 |

#### 5. 战斗系统实现

`battle.js` 中实现了血统特殊效果的运行时逻辑：

- **血统吸血**（`executeActiveSkill`）：所有主动技能造成伤害时按 `lifestealPct` 吸血；
- **血统灼烧**（`executeActiveSkill`）：主动技能攻击时按 `burnChance` 概率触发灼烧，3 回合内每回合造成 `burnPct` × 最大气血伤害；
- **血统法术伤害加成**（`executeActiveSkill`）：`magicDmgPct` 加成到法术伤害；
- **血统反击**（`executeMonsterAction`）：受击时按 `counterChance` 概率反击，造成攻击力 × `counterPct` 伤害；
- **血统反弹**（`executeMonsterAction`）：受击时按 `reflectChance` 概率反弹 `reflectPct` × 怪物伤害；
- **血统回血**（`applyTurnRegen`）：每回合按 `regenPct` × 最大气血恢复。

### 三、图鉴库系统建立（需求12）

v2.0.5 完善宠物图鉴系统，建立**统一数据图鉴库**架构，所有宠物和怪物数据集中管理于 `config.js`，确保数据唯一性与一致性。

#### 1. 四大图鉴库总览

| 图鉴库 | 类型 | 用途 | 查询入口 |
|:---|:---|:---|:---|
| `PET_DEX` | 宠物图鉴 | 所有宠物的种族/资质/成长/专长/天生技能 | `getPetDex(name)` |
| `SKILL_LIBRARY` | 技能库 | 所有主动/被动/光环/血统技能 | `getSkillById(id)` |
| `MONSTER_RACE_SKILLS` | 怪物技能池 | 按种族配置怪物技能 id | （引用技能库） |
| `MAP_BOSSES` | 地图Boss配置 | Boss技能/被动/buff | （引用技能库） |

#### 2. 设计原则

- **单一数据源**：所有数据定义在 `config.js` 中，其他文件（`battle.js`/`pet.js`/`ui.js`/`economy.js`）仅引用，不修改；
- **统一查询入口**：通过 `getPetDex(name)` 和 `getSkillById(id)` 查询，避免分散查询；
- **怪物与宠物共享图鉴**：怪物数据引用 `PET_DEX`，确保数据一致；
- **技能库统一引用**：商城/宠物/怪物技能均从 `SKILL_LIBRARY` 引用。

#### 3. SKILL_LIBRARY 统一技能库

v2.0.5 创建 `SKILL_LIBRARY` 统一技能库，包含所有技能类型：

```js
var SKILL_LIBRARY = [...ALL_SKILLS, ...BLOODLINE_SKILLS];
```

#### 4. 修复 BLOODLINE_SKILLS 未纳入查询入口

v2.0.5 前 `getSkillById` 仅从 `ALL_SKILLS` 查询，导致血统技能无法通过统一入口查询：

```js
// v2.0.5 前
function getSkillById(id) {
  return ALL_SKILLS.find(s => s.id === id) || BLOODLINE_SKILLS.find(s => s.id === id);
}

// v2.0.5 后
function getSkillById(id) {
  return SKILL_LIBRARY.find(s => s.id === id);
}
```

| 项目 | v2.0.5 前 | v2.0.5 后 |
|:---|:---|:---|
| 查询范围 | `ALL_SKILLS` + `BLOODLINE_SKILLS`（双查询） | `SKILL_LIBRARY`（单查询） |
| 维护性 | 双数组管理，易遗漏 | 单数组管理，统一维护 |
| 血统技能查询 | 通过 fallback 查询 | 直接查询，与其他技能一致 |

#### 5. PET_DEX 图鉴架构

`PET_DEX` 为对象，键为宠物名，值为该宠物的图鉴数据：

```js
var PET_DEX = {
  '小火焰': {
    race: '史莱姆',
    tier: 1,
    specialty: 'magic',
    growthRange: [1.0, 1.4],
    aptRange: { 力量资质: [1400, 2000], 体质资质: [1000, 1400], ... },
    innateSkills: ['fireball'],
    skillLib: ['fireball', 'flame_burst', ...],
    advanceable: true,
  },
  // ...所有宠物
};
```

**查询入口**：

| 函数 | 用途 |
|:---|:---|
| `getPetDex(name)` | 查询宠物图鉴数据 |
| `getPetTier(name)` | 查询宠物 T 级（1~5） |
| `getPetMaxSkills(name)` | 查询宠物技能格上限 |
| `getPetSkillLib(name)` | 查询宠物可学技能库 |

#### 6. 怪物数据引用

怪物数据（`MONSTER_RACE_SKILLS` / `MAP_BOSSES`）引用 `PET_DEX` 与 `SKILL_LIBRARY`：

- **怪物种族技能**：`MONSTER_RACE_SKILLS[race]` 返回该种族怪物可用的技能 id 列表，技能 id 引用 `SKILL_LIBRARY`；
- **地图 Boss 配置**：`MAP_BOSSES[mapId]` 包含 Boss 的种族、主动技能 id、被动技能 id、buff 加成，技能 id 引用 `SKILL_LIBRARY`。

#### 7. 修改规则

> **重要约束**：修改宠物或技能数据时，**只能修改 `config.js` 中的图鉴库**，不允许在其他文件中修改。

| 修改场景 | 修改位置 |
|:---|:---|
| 修改宠物种族/资质/成长/技能 | `PET_DEX` 对应条目 |
| 修改技能数据 | `ACTIVE_SKILLS` / `PASSIVE_SKILLS` / `AURA_SKILLS` / `BLOODLINE_SKILLS` |
| 修改怪物技能组合 | `MONSTER_RACE_SKILLS` / `MAP_BOSSES` |

#### 8. 图鉴库架构文档

`config.js` 顶部新增架构说明注释，明确四大图鉴库的职责与查询入口：

```
PET_DEX（宠物图鉴）：所有宠物的种族/资质/成长/专长/天生技能，getPetDex(name) 统一查询
SKILL_LIBRARY（技能库）：所有主动/被动/光环/血统技能，getSkillById(id) 统一查询
MONSTER_RACE_SKILLS（怪物技能池）：按种族配置怪物技能 id，引用技能库
MAP_BOSSES（地图Boss配置）：Boss技能/被动/buff，技能 id 引用技能库
```

> 相关文档：图鉴库系统同时记录于《04-技能系统》§v2.0.5 更新二，技能库查询入口修复见同节。

### 三、属性点转化公式

```
单属性点对应战斗值 = 1 + (对应单项资质 / 10000) × (1 + (成长值 + 种族值) / 10)
```

- **资质影响**：资质 1000 时系数 0.1，资质 3000 时系数 0.3
- **成长影响**：成长值越高，单点属性转化的战斗值越高
- **种族值影响**：种族值越高，单点属性转化的战斗值越高

> 详见 [§7.2](#72-属性点转化公式v260)。此公式取代 v2.5.0 前的 `基础属性 = (资质/100) × 成长 × (1+Lv×0.05) + Lv × 成长 × 种族值`。

### 一、批量加点与自动加点功能（需求1.1）

v2.7.0 新增**批量加点**与**自动加点**两种属性分配模式，减少玩家逐点分配属性的操作负担。两种模式均作用于宠物的自由属性点（`pet.freeAttrPoints`）。

#### 1. 批量加点 `batchAllocateAttrPoints(pet, allocations)`

- **定义位置**：`pet.js`
- **入参**：
  - `pet`：目标宠物对象
  - `allocations`：五维分配对象，形如 `{ str: 10, agi: 5, con: 5, end: 0, mag: 0 }`，五项之和不得超过 `pet.freeAttrPoints`
- **逻辑**：
  1. 校验分配总和 ≤ 剩余自由属性点，超出则按比例截断；
  2. 依次累加到 `pet.attrPoints.str/agi/con/end/mag`；
  3. 扣减 `pet.freeAttrPoints`；
  4. 调用 `syncPetAttrPoints(pet)` 同步派生属性。
- **UI 入口**：宠物详情页「批量加点」按钮 → `_batchAllocateAttrPoints(petId)`，弹出五维滑杆面板供玩家设定分配数量后确认。

#### 2. 自动加点 `autoAllocateAttrPoints(pet, ratios)`

- **定义位置**：`pet.js`
- **入参**：
  - `pet`：目标宠物对象
  - `ratios`：五维比例对象，形如 `{ str: 0.4, agi: 0.2, con: 0.2, end: 0.1, mag: 0.1 }`，五项之和应为 1.0（不满足时按比例归一化）
- **逻辑**：
  1. 按比例将 `pet.freeAttrPoints` 分配到五维属性；
  2. 每维实际分配数 = `floor(freeAttrPoints × ratio)`，余数按比例从大到小补齐；
  3. 调用 `syncPetAttrPoints(pet)` 同步派生属性。
- **UI 入口**：宠物详情页「自动加点」按钮 → `_autoAllocateAttrPoints(petId)`，内置推荐比例（物理系/法术系/防御系/均衡 4 套预设），玩家也可自定义比例后一键分配。

> **注意**：两种模式均不改变已分配的固定属性点（每级 5 点固定），仅作用于自由属性点部分。

### 二、修复未出战宠物属性点计算异常（需求1.2）

v2.7.0 修复了未出战宠物在等级变化后自由属性点未正确累加的问题，核心修复函数为 `syncPetAttrPoints(pet)`。

#### 1. 问题现象

- 未出战的宠物随角色升级同步等级时，`freeAttrPoints` 未按 `等级 × 10` 重新计算，导致部分等级的自由点数丢失；
- 转生后所有宠物等级重置为 1，已分配属性点未清零，出现负数自由点。

#### 2. 修复公式

```
剩余自由属性点 = 等级 × 10 - 已分配属性点总和
```

其中：
- `等级 × 10`：每级 10 点属性点（5 固定 + 5 自由），此处取总量 10；
- `已分配属性点总和 = pet.attrPoints.str + agi + con + end + mag`。

#### 3. `syncPetAttrPoints(pet)` 流程

| 步骤 | 操作 |
|---|---|
| 1 | 计算 `totalAllocated = sum(pet.attrPoints[五维])` |
| 2 | 计算 `expectedTotal = pet.level × 10` |
| 3 | `pet.freeAttrPoints = max(0, expectedTotal - totalAllocated)` |
| 4 | 若 `totalAllocated > expectedTotal`（转生等场景），按比例回扣已分配点至合法范围 |
| 5 | 重算派生属性（攻击力/防御力/灵力/气血/速度/魔法值） |

- 该函数在以下场景被调用：宠物升级、转生重置、批量加点、自动加点、存档迁移。
- 修复后未出战宠物的自由属性点与出战宠物保持一致计算口径。

### 四、宠物战斗力评价体系重构（需求4.2）

v2.7.0 对 `getCombatPowerBreakdown(pet)` 进行重构，在原五维派生属性战力基础上**纳入二级战斗属性权重**，并引入**寿命系数**，使战力数值更贴近实战表现。

#### 1. 战力计算新公式

```
最终战力 = (属性战力小计 + 二级属性战力) × 技能倍率 × 寿命系数
```

其中：
- **属性战力小计**：沿用 v2.6.0 五维派生属性战力（气血 ×0.3 + 攻击力 ×3 + 防御力 ×2 + 灵力 ×2.5 + 速度 ×1.5）；
- **二级属性战力**：10 项二级属性按权重加权求和；
- **技能倍率**：主动技能平均 dmgPct 超 1.0 部分 ×0.5 + 技能数每个 +3%；被动技能按效果类型加权；异化品质额外 ×1.2；
- **寿命系数**：见下节。

#### 2. 二级属性战力权重表

| 二级属性 | 权重 | 说明 |
|---|---|---|
| 暴击率 | ×1.0 | 每百分点折算战力 |
| 暴击伤害 | ×0.3 | 每百分点折算战力 |
| 闪避率 | ×0.8 | 每百分点折算战力 |
| 命中率 | ×0.5 | 仅超过 0.8（80%）的部分计入战力 |
| 吸血 | ×0.6 | 每百分点折算战力 |
| 减伤 | ×1.0 | 每百分点折算战力 |
| 技能触发 | ×0.8 | 每百分点折算战力 |
| 技能增伤 | ×0.5 | 每百分点折算战力 |
| 回血 | ×0.4 | 每百分点折算战力 |
| 法术增伤 | ×0.5 | 每百分点折算战力 |

> 权重设计原则：暴击率/减伤等直接影响伤害与生存的属性权重最高（1.0）；暴击伤害/技能增伤等乘区属性权重较低（0.3~0.5），避免与暴击率叠加后虚高。

#### 3. 寿命系数

| 寿命区间 | 系数 | 说明 |
|---|---|---|
| `pet.lifespan >= 1000` | ×1.0 | 正常寿命，无折减 |
| `pet.lifespan < 1000` | `0.5 + lifespan/2000` | 寿命越低战力折减越大，最低 ×0.5 |
| 神兽（`lifespan === 99999`） | ×1.0 | 神兽寿命无限，不受折减影响 |

- 寿命系数作为最终乘数作用于战力总计，鼓励玩家通过月华露/元宵等道具维持宠物寿命。

#### 4. 战力面板展示

- 宠物详情页战力数值后新增「ⓘ」图标，点击展开战力构成详情：
  - 属性战力小计
  - 二级属性战力小计（10 项分项）
  - 技能倍率
  - 寿命系数
  - 最终战力
- 战力排名/出战推荐列表同步采用新战力数值排序。

## v2.10.0 更新

> **版本**：v2.10.0 战力体系重构 + 刷新战力校验 + 上阵排序 + 打书技能展示

### 战斗力计算公式全维度重构（v2.10.0 需求4.1）

`getCombatPowerBreakdown` 函数重构为 **6 大模块加权求和**架构，确保所有宠物生效养成项均纳入战力计算：

| 模块 | 计算范围 | 公式 |
|------|----------|------|
| 基础属性战力 | 最终五维+战斗属性+二级战斗属性（含装备/符文加成） | `(baseCp + petEquipCp) × (1 + secondaryPct)` |
| 技能战力 | 血统天赋+主动/被动+附加技能 | `(baseCp + petEquipCp) × (skillMult - 1) + bloodlineCp` |
| 人物加成战力 | 人物额外属性加成+人物修炼加成 | `charCp + petBonusCp` |
| 阵法战力 | 当前阵法提供的属性/效果加成 | `formationCp` |
| 天赋战力 | 已点亮天赋中对宠物生效的加成 | `talentCp` |
| 神通战力 | 神通系统提供的全部加成 | `divineCp`（基于 `getAllDivinePowerEffects()`） |

**最终公式**：`total = baseAttrCp + skillCpTotal + charTotalCp + formationCp + talentCp + divineCp`

> **v2.8.0 对比**：v2.8.0 公式为 `(statCp + secondaryCp) × skillMult`，skillMult 乘以所有属性。v2.10.0 拆分为 6 个独立模块相加，skillMult 仅作用于基础属性部分，新增神通战力模块。

### 刷新战力功能属性校验（v2.10.0 需求4.2）

`recalcPetOnDeploy` 函数新增 **两步底层校验与自动修正**，在计算最终属性前执行：

1. **原生五维属性校验**：
   - 校验标准：原生五维属性总和 = 当前宠物等级 × 10
   - 单属性校验：五维单属性最低值 ≥ 当前宠物等级
   - 不满足时自动补齐差额（`pet.attrPoints[attr] = lv`）

2. **属性点总数校验**：
   - 校验标准：已分配属性点 + 剩余自由属性点 = 当前宠物等级 × 5（自由属性点配额）
   - 不满足时将差额自动补充到 `pet.freeAttrPoints`

> 校验修正完成后，再叠加所有外部加成（装备/符文/人物/阵法），计算最终属性与战斗力。

### 宠物列表上阵排序优化（v2.10.0 需求3.1）
