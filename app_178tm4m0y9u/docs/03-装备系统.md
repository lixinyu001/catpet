## 一、系统总览

游戏装备系统分为**两大独立模块**：

| 模块 | 入口 | 说明 |
|------|------|------|
| 角色装备系统 | 角色页 | 6 栏位、5 品质、词条+特效+宝石+强化 |
| 宠物装备系统 | 宠物页 | 3 栏位、4 品质、词条+套装+打造 |

二者数据结构、品质体系、加成计算完全独立，互不影响。

## 二、角色装备系统

### 2.1 装备品质

5 级品质定义（常量 `EQUIP_RARITIES`）：

| 顺序 | id | 中文名 | 颜色 | 词条数量 | 特效概率 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 0 | `white` | 白色 | `#c0c0c0` | 0 | 0% |
| 1 | `green` | 绿色 | `#22c55e` | 1 | 0% |
| 2 | `blue` | 蓝色 | `#3b82f6` | 2 | 3% |
| 3 | `purple` | 紫色 | `#a855f7` | 3 | 6% |
| 4 | `orange` | 橙色 | `#fb923c` | 4 | 12% |

对应常量：
- `EQUIP_RARITY_AFFIX_COUNT = [0, 1, 2, 3, 4]`
- `EQUIP_SPECIAL_CHANCE = { white: 0, green: 0, blue: 0.03, purple: 0.06, orange: 0.12 }`

> 仅蓝色及以上品质才可能出现装备特效与宝石孔。

### 2.2 装备栏位

6 个装备栏位（常量 `EQUIPMENT_SLOTS`）：

| id | 名称 | 图标 |
|:---:|:---:|:---:|
| `weapon` | 武器 | ⚔️ |
| `helmet` | 头盔 | 🪖 |
| `armor` | 衣服 | 👕 |
| `pants` | 裤子 | 👖 |
| `gloves` | 手套 | 🧤 |
| `shoes` | 鞋子 | 👟 |

### 2.3 装备基础属性差异化

按栏位类型差异化生成基础属性（函数 `generateEquipment`，level = 装备等级）：

| 栏位 | 定位 | 主属性 | 数值公式 | 次属性 | 数值公式 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| `weapon` 武器 | 输出 | 攻击力（高） | `randomInt(level*3, level*5)` | 气血 | `randomInt(level*1, level*2)` |
| `helmet` 头盔 | 法系 | 气血（高） | `randomInt(level*4, level*6)` | 智力 | `randomInt(level*2, level*3)` |
| `armor` 衣服 | 坦克 | 气血（高） | `randomInt(level*4, level*7)` | 防御力（高） | `randomInt(level*3, level*5)` |
| `pants` 裤子 | 生存 | 气血（中） | `randomInt(level*3, level*5)` | 防御力 + 体质 | 防 `randomInt(level*2, level*4)`，体 `randomInt(level*2, level*3)` |
| `gloves` 手套 | 暴击 | 攻击力（少） | `randomInt(level*2, level*3)` | 敏捷 | `randomInt(level*2, level*4)` |
| `shoes` 鞋子 | 速度 | 敏捷（高） | `randomInt(level*3, level*5)` | 气血 | `randomInt(level*1, level*2)` |

装备名按栏位 + 品质从 `EQUIP_BASE_NAMES` 中随机选取（如武器池：短剑/长剑/战斧/法杖/匕首/巨锤/长弓/魔杖）。

### 2.4 装备词条系统

词条池定义于 `AFFIX_TYPES`，共 19 种：

| 类别 | 词条 id | 显示格式 | 数值范围 |
|:---:|:---|:---|:---|
| 数值类 | `str_flat` / `con_flat` / `agi_flat` / `int_flat` | 力量/体质/敏捷/智力 +v | `randomInt(level, level*3)` |
| 数值类 | `hp_flat` / `atk_flat` / `def_flat` | 气血/攻击力/防御力 +v | `randomInt(level, level*3)` |
| 百分比类 | `str_pct` / `con_pct` / `agi_pct` / `int_pct` / `hp_pct` / `atk_pct` / `def_pct` | 属性 +v% | `randomFloat(0.03, 0.12)` |
| 战斗类 | `crit_rate` | 暴击率 +v% | `randomFloat(0.02, 0.08)` |
| 战斗类 | `dodge_rate` | 闪避率 +v% | `randomFloat(0.02, 0.08)` |
| 宠物专属 | `pet_dmg` / `pet_def` / `pet_hp` | 宠物伤害/防御力/气血 +v% | `randomFloat(0.05, 0.15)` |

**词条生成规则**：
1. 词条数 = `EQUIP_RARITY_AFFIX_COUNT[rarityIdx]`（白 0 / 绿 1 / 蓝 2 / 紫 3 / 橙 4）
2. 同件装备词条 id 不重复（最多 30 次重试避免冲突）
3. **橙色装备额外附赠 1 个特殊词条**：从 `pet_dmg` / `pet_def` / `pet_hp` 中随机挑选（不与已有词条冲突）

**词条应用顺序**（`getEquipStatBonus`）：
- 先累加所有 flat 数值词条
- 百分比词条最后统一应用：`final = base_flat × (1 + sum_pct)`

### 2.5 装备特效系统

8 种特效定义于 `EQUIP_SPECIALS`，仅蓝色及以上品质低概率出现 1 个：

| id | 名称 | 描述 | 颜色 |
|:---:|:---:|:---|:---:|
| `no_level_req` | 无级别限制 | 装备无等级限制，任何等级可穿戴 | `#f59e0b` |
| `simple` | 简易 | 强化消耗减少 50% | `#22c55e` |
| `gem_boost` | 宝石强化 | 宝石效果 +50% | `#a855f7` |
| `shennong` | 神农 | 气血回复 +20% | `#22c55e` |
| `treasure` | 珍宝 | 售价 +100% | `#f59e0b` |
| `sure_hit` | 必中 | 攻击必定命中 | `#ef4444` |
| `execute` | 绝杀 | 5% 概率秒杀小怪 | `#ef4444` |
| `block` | 格挡 | 10% 概率格挡 50% 伤害 | `#3b82f6` |

**特效出现概率**（`EQUIP_SPECIAL_CHANCE`）：

| 品质 | white | green | blue | purple | orange |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 概率 | 0% | 0% | 3% | 6% | 12% |

### 2.6 装备生成流程

`generateEquipment(rarity, level)` 函数完整流程：

1. 从 `EQUIPMENT_SLOTS` 随机选 1 个栏位
2. 从 `EQUIP_BASE_NAMES[slot.id]` 随机选基础名
3. 按 2.3 节公式生成基础属性
4. 按 2.4 节规则生成词条（含橙色额外词条）
5. 按 2.5 节概率随机 1 个特效（蓝+）
6. **宝石孔生成**（仅 `rarityIdx >= 2`）：
   - 概率分布：30% 0 孔 / 30% 1 孔 / 25% 2 孔 / 15% 3 孔
   - 每孔类型从 `['hp','mp','vit','str','agi','int']` 中随机
7. 返回装备对象：`{ id, slot, name, rarity, level, baseAtk, baseDef, baseHp, baseStr, baseCon, baseAgi, baseInt, affixes, special, gemSlots }`

## 三、宝石系统

### 3.1 宝石类型与栏位映射

6 种宝石类型（常量 `GEM_TYPES`）：

| id | 名称 | 图标 | 属性 | 每级加成 | 颜色 | 可镶嵌栏位 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---|
| `hp` | 气血宝石 | ❤️ | 气血 | +80 | `#ef4444` | 武器、头盔、衣服、裤子 |
| `mp` | 魔法宝石 | 💠 | 魔法值 | +30 | `#3b82f6` | 头盔、衣服、裤子 |
| `vit` | 体质宝石 | 🌿 | 体质 | +4 | `#22c55e` | 衣服、裤子 |
| `str` | 力量宝石 | 🔥 | 力量 | +4 | `#f59e0b` | 武器、手套 |
| `agi` | 敏捷宝石 | ⚡ | 敏捷 | +4 | `#eab308` | 鞋子、手套 |
| `int` | 智慧宝石 | 🔮 | 智慧 | +4 | `#a855f7` | 头盔、武器 |

**栏位→宝石类型映射汇总**：

| 栏位 | 可镶嵌宝石类型 |
|:---:|:---|
| 武器 weapon | 力量 / 智慧 |
| 头盔 helmet | 气血 / 魔法 / 智慧 |
| 衣服 armor | 气血 / 魔法 / 体质 |
| 裤子 pants | 气血 / 魔法 / 体质 |
| 手套 gloves | 力量 / 敏捷 |
| 鞋子 shoes | 敏捷 |

> 每个装备栏位允许镶嵌的宝石类型由 `GEM_TYPES[i].slots` 数组决定，调用 `getGemsForSlot(slotId)` 查询。

**等级上限**：`GEM_MAX_LEVEL = 15`（+15）

### 3.2 宝石孔生成

- 仅蓝色及以上品质装备有宝石孔
- 概率分布（详见 2.6 节）：0 孔 30% / 1 孔 30% / 2 孔 25% / 3 孔 15%
- 每个孔有固定类型（在装备生成时随机分配），仅可镶嵌同类型宝石

### 3.3 宝石升级

**升级公式**（函数 `upgradeGem`）：

```
消耗 = 3 颗同类型同等级宝石 + 2000 × level² 金币
产出 = 1 颗同类型（level + 1）宝石
```

常量：
- `GEM_UPGRADE_COST = 3`（消耗同级宝石数）
- `GEM_UPGRADE_GOLD = 2000`（基础金币系数）

**金币消耗示例表**：

| 当前等级 | 金币消耗 | 当前等级 | 金币消耗 |
|:---:|---:|:---:|---:|
| +1 → +2 | 2,000 | +8 → +9 | 128,000 |
| +2 → +3 | 8,000 | +9 → +10 | 162,000 |
| +3 → +4 | 18,000 | +10 → +11 | 200,000 |
| +4 → +5 | 32,000 | +11 → +12 | 242,000 |
| +5 → +6 | 50,000 | +12 → +13 | 288,000 |
| +6 → +7 | 72,000 | +13 → +14 | 338,000 |
| +7 → +8 | 98,000 | +14 → +15 | 392,000 |

**单属性最大加成**（+15 单孔）：
- 气血宝石：80 × 15 = 1200 气血
- 魔法宝石：30 × 15 = 450 魔法值
- 力量/体质/敏捷/智慧宝石：4 × 15 = 60 对应属性

### 3.4 宝石镶嵌与卸下

- **镶嵌**（`equipGem`）：在装备宝石孔中找到第一个类型匹配的空孔，从背包消耗 1 颗宝石放入
- **卸下**（`unequipGemFromSlot`）：将指定孔中的宝石退回背包
- **属性汇总**（`getGemStatBonus`）：遍历所有已装备物品的宝石孔，按 `def.perLevel × gem.level` 累加

### 3.5 打孔系统

v2.0.3 新增「打孔镶嵌」系统，允许玩家通过道具为装备**主动开孔**或**重置已开孔**，不再完全依赖装备生成时的随机宝石孔。

#### 3.5.1 道具

| 道具 ID | 名称 | 品质档位 | 作用 |
|:---:|:---:|:---:|:---|
| `socket_nail` | 打孔钉 | mid（中级） | 对装备进行打孔，按当前孔数决定成功率 |
| `repair_glue` | 修补胶 | high（高级） | 重置装备已开孔数回到 0（不影响宝石孔类型分配） |

道具获取途径：商城（`socket_nail` 2500 金币 / `repair_glue` 15 钻石）、宝箱掉落、藏宝图战斗掉落。

#### 3.5.2 打孔成功率（`socketEquipment`）

`economy.js → socketEquipment()`（~line 419）按装备当前已开孔数决定打孔成功率，单件装备最多 3 孔（受 `MAX_GEM_SLOTS` 限制）：

| 当前已开孔数 | 打孔成功率 |
|:---:|:---:|
| 0 → 1（第 1 孔） | **80%** |
| 1 → 2（第 2 孔） | **50%** |
| 2 → 3（第 3 孔） | **20%** |

- 打孔成功：装备 `gemSlots` 数组追加一个新孔，类型从 `['hp','mp','vit','str','agi','int']` 随机
- 打孔失败：消耗打孔钉但不开孔（无降级惩罚）
- 已达 3 孔上限时拒绝打孔

#### 3.5.3 重置孔数（`resetSockets`）

`economy.js → resetSockets()`（~line 468）消耗 1 个 `repair_glue`：

- 装备 `gemSlots` 数组清空为 `[]`
- 已镶嵌的宝石**全部退回背包**（避免损失）
- 不影响装备其他属性（基础属性/词条/特效/强化等级均不变）

#### 3.5.4 装备详情中的孔洞信息

`ui.js`（~line 3487）装备详情展示新增 **🔩 孔洞 X/MAX_GEM_SLOTS** 行：

- `X` = 当前装备已开孔数（`gemSlots.length`）
- `MAX_GEM_SLOTS` = 单件装备孔数上限（3）
- 装备列表项也会显示宝石孔数量徽章，便于快速识别可镶嵌宝石的装备

#### 3.5.5 角色页打孔镶嵌 sheet

角色页（`renderCharacterScreen`）sheet tabs 新增 `'socket'`（🔩 打孔镶嵌），由 `ui.js → renderSocketSheet()`（~line 3742）渲染：

- 选择装备（背包中的可打孔装备）
- 显示当前孔数 / 成功率 / 失败无惩罚说明
- 「🔩 打孔」按钮：调用 `socketEquipment(equipId)`，消耗 1 个 `socket_nail`
- 「🩹 重置孔数」按钮：调用 `resetSockets(equipId)`，消耗 1 个 `repair_glue`，需二次确认
- 道具库存展示（打孔钉 / 修补胶当前数量）

> 角色页 sheet tabs 现为：`equipment`（装备）/ `forge`（锻造强化）/ `socket`（打孔镶嵌）。

## 四、锻造强化系统

强化系统针对**装备栏位**而非物品本身（更换装备不影响强化等级），每级强化提升该位置装备**全部属性 10%**。

### 4.1 强化等级上限

**公式**（函数 `getMaxForgeLevel`）：

```
maxLevel = 12 + bonus
bonus = (转生次数 r >= 1) ? 1 + floor((r - 1) / 5) : 0
```

**转生加成规则**：
- 基础上限：+12
- 首次转生（r=1）：+1
- 之后每多 5 次转生再 +1（r=6 时再 +1，r=11 时再 +1，…）

**示例**：

| 转生次数 r | bonus | 强化上限 |
|:---:|:---:|:---:|
| 0 | 0 | +12 |
| 1 | 1 | +13 |
| 2~5 | 1 | +13 |
| 6 | 2 | +14 |
| 11 | 3 | +15 |

### 4.2 强化成功率

`getForgeSuccessRate(lv)` 返回当前等级→下一级的成功率：

| 当前等级 | 成功率 | 当前等级 | 成功率 |
|:---:|:---:|:---:|:---:|
| +0 → +1 | 100% | +8 → +9 | 60% |
| +1 → +2 | 100% | +9 → +10 | 40% |
| +2 → +3 | 100% | +10 → +11 | 30% |
| +3 → +4 | 100% | +11 → +12 | 20% |
| +4 → +5 | 100% | +12 → +13 | 15% |
| +5 → +6 | 100% | +13 → +14 | 10% |
| +6 → +7 | 80% | +14 → +15 | 6% |
| +7 → +8 | 70% | +15 → +16 | 3% |

### 4.3 强化石消耗

`getForgeStoneNeeded(lv)` 根据当前强化等级返回所需强化石：

| 强化等级范围 | 所需强化石 | 函数返回 |
|:---:|:---|:---|
| +0 ~ +5 | 低级强化石 `forge_stone_low` | 6 级以下用低级 |
| +6 ~ +8 | 中级强化石 `forge_stone_mid` | 6~8 级用中级 |
| +9 及以上 | 高级强化石 `forge_stone_high` | 9 级及以上用高级 |

### 4.4 失败惩罚与保底石

`getForgeFailPenalty(lv)` 返回失败惩罚：

| 强化等级范围 | 失败惩罚 |
|:---:|:---|
| +0 ~ +5 | 无（成功率 100%） |
| +6 ~ +8 | 等级 -1 |
| +9 及以上 | **等级清零（reset）** |

**保底石机制**：
- 勾选"使用保底石"后，强化时额外消耗 1 个保底石
- 强化失败时**等级保持不变**（不降级、不清零）
- 保底石不足则无法启用此选项
- 转生时不会退还已消耗的保底石

### 4.5 强化属性加成

**强化倍率公式**：

```
forgeMult = 1 + forgeLv × 0.10
```

应用范围：该位置装备的**全部属性**，包括：
- 基础属性（baseAtk / baseDef / baseHp / baseStr / baseCon / baseAgi / baseInt）
- 所有词条数值（flat 与 pct 词条的 base 累积值都参与乘算）

**示例**：+10 强化 → `forgeMult = 1 + 10×0.10 = 2.0`，装备全部属性 ×2。

### 4.6 自动强化

`startAutoForge(slotId, targetLv)` 函数：
- 每 500ms 执行一次 `doForge` 调用
- 触发停止条件：达到目标等级 / 手动停止 / 强化石不足 / 累计次数 > 200（安全阀）
- 可选保底石：若启用但库存不足，则继续不用保底石
- 自动强化期间不弹 toast，仅结束时提示

## 五、强化石合成

**合成规则**（函数 `composeForgeStone`）：

| 配方 | 输入 | 输出 |
|:---|:---|:---|
| 低级 → 中级 | 10 × 低级强化石 | 1 × 中级强化石 |
| 中级 → 高级 | 10 × 中级强化石 | 1 × 高级强化石 |

- 支持批量合成：输入数量 `count`，消耗 `count × 10` 个低级材料

- 合成无金币消耗，无失败概率

**入口位置**（已迁移到角色页"锻造强化"sheet，详见第七节）：

- 现位于角色页 → "锻造强化" sheet → "强化石合成" 区块

`addEquipmentToBag(equip)` 函数在装备入袋前检查自动分解设置：

**配置参数**（`G.autoDecompose`）：

| 字段 | 默认值 | 含义 |
|:---|:---:|:---|
| `enabled` | `false` | 是否启用自动分解 |
| `maxRarity` | `'green'` | 自动分解品质上限（含） |
| `maxLevel` | `10` | 自动分解等级上限（含） |

**自动分解产出**：与手动分解完全一致（按上表）

UI 入口：角色页 → 装备背包区 → "♻️ 自动分解"配置面板。

### 6.3 转生时装备处理

`doRebirth()` 函数在转生时自动处理装备：

1. **卸下所有装备**：6 个栏位的装备全部退回 `G.equipmentBag` 背包
2. **退还宝石**：遍历每件装备的 `gemSlots`，已镶嵌的宝石全部调用 `addGemToBag` 退回背包
3. **重置强化等级**：`G.player.forgeLevels[slotId] = 0`（强化等级绑定槽位，转生后清零）
4. **兼容旧存档**：若存在 `G.gems[slotId]` 旧数据，同样调用 `addGemToBag` 退还

> 转生后玩家等级归 1，装备等级不变，但玩家可重新穿戴原装备（受装备等级限制）。

| 3 | `mythic` | 神话 | `#ef4444` | 4 |

### 8.3 基础词条与随机词条

`generatePetEquip(rarity, slot, level)` 函数：

**等级缩放系数**：`lvlScale = 1 + level × 0.08`

**基础词条**（按 slot 固定 1 个，rarityIdx 为品质序号 0~3）：

| 栏位 | 基础属性 | 数值公式 |
|:---:|:---:|:---|
| attack | 攻击力 | `floor((10 + level×2) × (1 + rarityIdx×0.3) × lvlScale)` |
| hp | 气血 | `floor((30 + level×6) × (1 + rarityIdx×0.3) × lvlScale)` |
| defense | 防御力 | `floor((6 + level×1.5) × (1 + rarityIdx×0.3) × lvlScale)` |

**随机词条**（`PET_AFFIX_TYPES`，共 13 种，不重复）：

| 类别 | 词条 id | 属性 |
|:---:|:---|:---|
| 属性类 | `p_str` / `p_con` / `p_agi` / `p_int` | 力量 / 体质 / 敏捷 / 智力 |
| 属性类 | `p_hp` / `p_atk` / `p_def` / `p_spd` | 气血 / 攻击力 / 防御力 / 速度 |
| 资质类 | `p_apt_str` / `p_apt_con` / `p_apt_agi` / `p_apt_int` | 力量/体质/敏捷/智力 资质 |
| 成长类 | `p_growth` | 成长 |

**词条数值范围**（`valScale = 1 + rarityIdx × 0.4`）：

| 词条类别 | 属性 | 数值公式 | 乘以 valScale |
|:---:|:---|:---|:---:|
| 属性类 | 气血 | `randomInt(20, 50) × level` | ✓ |
| 属性类 | 攻击力 / 防御力 | `randomInt(5, 12) × level` | ✓ |
| 属性类 | 速度 | `randomInt(2, 6) × max(1, floor(level/5))` | ✓ |
| 属性类 | 力量/体质/敏捷/智力 | `randomInt(3, 8) × max(1, floor(level/3))` | ✓ |
| 资质类 | 4 种资质 | `randomInt(20, 100)` | ✓ |
| 成长类 | 成长 | `round(randomFloat(0.05, 0.20) × valScale × 100) / 100` | ✓ |

### 8.9 装备加成计算

`getPetEquipBonus(pet)` 函数返回宠物的装备总加成：

```javascript
{
  statAdditions: { 力量, 体质, 敏捷, 智力, 气血, 攻击力, 防御力, 速度 },
  aptAdditions: { 力量资质, 体质资质, 敏捷资质, 智力资质 },
  growthAddition: number,  // 成长加成
  setBonuses: { [setId]: { count, bonus, name, color } },
  skillAdditions: []
}
```

**计算流程**：
1. 初始化结果对象（所有属性归零）
2. 遍历 3 个装备栏（attack / hp / defense）：
   - 收集套装计数（`setCounts[setId]++`）
   - 遍历装备 `affixes`，按 `kind` 累加到对应字段：
     - `stat` → `statAdditions[affix.stat]`
     - `aptitude` → `aptAdditions[affix.stat]`
     - `growth` → `growthAddition`
3. 套装加成判定：
   - `count >= 3` → 应用 3 件套加成 `bonus3`（含独特效果）
   - `count >= 1` → 应用 1 件套加成 `bonus`

**装备/卸下接口**：
- `equipPetEquipment(petId, equipId)`：装到指定宠物的对应栏位（旧装备退回背包）
- `unequipPetEquipment(petId, slot)`：卸下指定栏位装备到背包

## v2.0.4 更新

### 宠物装备打造系统重做

v2.0.4 对宠物装备打造系统进行了全面重做，引入材料分级与三档打造等级机制，使高品质宠物装备的获取路径更清晰、更具挑战性。

#### 1. 材料分级体系

神秘水晶与远古符文各分为低 / 中 / 高三级，与兽皮共同构成 7 种打造材料：

| 材料 ID | 名称 | 档位 |
|:---:|:---|:---:|
| `mystic_crystal_low` | 低级神秘水晶 | 低级 |
| `mystic_crystal_mid` | 中级神秘水晶 | 中级 |
| `mystic_crystal_high` | 高级神秘水晶 | 高级 |
| `ancient_rune_low` | 低级远古符文 | 低级 |
| `ancient_rune_mid` | 中级远古符文 | 中级 |
| `ancient_rune_high` | 高级远古符文 | 高级 |
| `beast_leather` | 兽皮 | 通用 |

**旧材料迁移**：旧版 `mystic_crystal`（神秘水晶）与 `ancient_rune`（远古符文）在加载存档时**自动转换为对应 `*_low`**（低级）版本，旧存档玩家不会损失材料。

#### 2. 三档打造等级

新打造配方由原 4 档（rare/epic/legend/mythic）改为 **3 档（低级 / 中级 / 高级）**，每档消耗不同级别的材料与金币：

| 打造档位 | 神秘水晶消耗 | 远古符文消耗 | 金币消耗 | 产出品质分布 |
|:---:|:---:|:---:|---:|:---|
| 低级打造 | 2 × 低级水晶 | 1 × 低级符文 | 5,000 | 100% rare（稀有） |
| 中级打造 | 3 × 中级水晶 | 2 × 中级符文 | 30,000 | 60% rare + 30% epic + 10% legend |
| 高级打造 | 5 × 高级水晶 | 4 × 高级符文 | 150,000 | 20% epic + 40% legend + 40% mythic |

> 高级打造不再保底稀有，但有可能直接产出 mythic（神话）品质装备，是顶级宠物装备的核心来源。

#### 3. 栏位选择

打造时玩家可**指定栏位**（attack / hp / defense），不再随机分配栏位。这一改动让玩家可以针对性补齐宠物装备缺口。

#### 4. 分解产出分级

宠物装备分解的材料产出按**装备品质**对应**材料等级**，与打造配方形成闭环：

| 装备品质 | 分解产出材料档位 |
|:---:|:---|
| rare（稀有） | 低级材料 |
| epic（史诗） | 中级材料 |
| legend（传说）/ mythic（神话） | 高级材料 |

> 旧版分解按 `PET_EQUIP_DECOMP_YIELD` 表随机产出 mystic_crystal / ancient_rune / beast_leather 三种材料；v2.0.4 起按装备品质映射到对应档位的分级材料。

#### 5. 宠物秘境掉落按波次分级

宠物秘境（petcave）的材料掉落现按波次分级，鼓励玩家挑战更高波次：

| 波次 | 掉落材料档位 |
|:---:|:---|
| 第 1 ~ 2 波 | 低级材料 |
| 第 3 ~ 4 波 | 中级材料 |
| 第 5 波及以后 | 高级材料 |

> 这一改动使宠物秘境成为中级 / 高级材料的稳定产出途径，玩家可通过推进波次获取更高级材料。

## v2.0.5 更新

### 一、装备卸下时宝石一起卸下（需求1）

v2.0.5 修复了卸下装备会导致已镶嵌宝石丢失的问题，`unequipItem` 函数在卸下装备时会自动退还所有宝石。

#### 1. 卸下流程

`unequipItem(slotId)`（`economy.js`）流程更新：

1. 获取该栏位的装备 `equip = G.equipment[slotId]`；
2. **遍历 `equip.gemSlots` 数组**：
   - 若该孔已镶嵌宝石（`slot.gem` 存在），按 `slot.type` 调用 `addGemToBag(slot.type, slot.gem.level, 1)` 退还到宝石背包；
3. 装备退回 `G.equipmentBag`；
4. 清空 `G.equipment[slotId]`。

#### 2. 影响范围

| 操作 | v2.0.5 前 | v2.0.5 后 |
|:---|:---|:---|
| 卸下装备 | 已镶嵌宝石丢失 | **所有宝石按类型退还背包** |
| 转生时卸下装备 | 已通过 `doRebirth` 退还 | 仍正常退还（与本次修复一致） |
| 重置孔数（`resetSockets`） | 已退还宝石 | 仍正常退还 |

> 该修复确保玩家在切换装备时不会损失宝石资源，与转生时的宝石退还逻辑保持一致。

### 二、宠物装备封印系统（需求5）

v2.0.5 引入宠物装备**封印状态**，转生后所有宠物装备会被封印，属性按人物等级/装备等级比例解封，避免转生后人物等级归 1 仍享受满级装备属性。

#### 1. 封印状态字段

`petEquip` 对象新增字段：

| 字段 | 类型 | 含义 |
|:---|:---|:---|
| `sealed` | boolean | 是否处于封印状态 |
| `sealedLevel` | number | 装备被封印时的等级（用于解封比例计算） |

#### 2. 转生时设置封印

`doRebirth()`（`economy.js`）在转生流程中遍历所有宠物的所有装备，设置封印状态：

```js
Object.keys(G.pets).forEach(petId => {
  var pet = G.pets.find(p => p.id === petId);
  if (!pet || !pet.petEquipment) return;
  ['attack', 'hp', 'defense'].forEach(slot => {
    var equip = pet.petEquipment[slot];
    if (equip) {
      equip.sealed = true;
      equip.sealedLevel = equip.level;
    }
  });
});
```

#### 3. 解封比例计算（getPetEquipUnsealRatio）

`economy.js → getPetEquipUnsealRatio(equip)` 返回当前解封比例（0.0 ~ 1.0）：

```js
function getPetEquipUnsealRatio(equip) {
  if (!equip.sealed) return 1.0;
  var playerLevel = G.player.level;
  var equipLevel = equip.sealedLevel || equip.level;
  if (equipLevel <= 0) return 1.0;
  return Math.min(1.0, playerLevel / equipLevel);
}
```

| 人物等级 | 100级装备解封比例 | 实际属性 |
|:---:|:---:|:---|
| 1 | 1% | 1% |
| 5 | 5% | 5% |
| 50 | 50% | 50% |
| 100 | 100% | 100%（完全解封） |
| ≥100 | 100% | 100% |

#### 4. 属性按比例缩放

`getPetEquipBonus(pet)`（`economy.js`）计算装备加成时：

1. **stat 类词条**：按 `unsealRatio` 缩放（基础属性 + flat 词条）；
2. **aptitude / growth 类词条**：**不受封印影响**，资质与成长始终全额生效；
3. **套装属性加成**：通过 `scaleSetBonus(bonus, ratio)` 按比例缩放，使用 3 件装备中**最低**的解封比例。

```js
function scaleSetBonus(bonus, ratio) {
  if (!bonus || ratio >= 1.0) return bonus;
  var scaled = {};
  Object.keys(bonus).forEach(function(k) {
    var v = bonus[k];
    if (typeof v === 'number') scaled[k] = v * ratio;
    else scaled[k] = v;
  });
  return scaled;
}
```

#### 5. UI 显示

宠物装备管理界面显示封印状态：

- 已封印装备：显示 `🔒 封印中（解封 X%）`
- 已解封装备：无额外标记
- 解封比例实时随人物等级变化

#### 6. 不受影响的属性

| 属性 | 是否受封印影响 |
|:---|:---:|
| 基础属性（攻击/防御/气血等） | ✅ 受影响 |
| 随机词条（stat 类） | ✅ 受影响 |
| 套装属性加成 | ✅ 受影响（按最低比例） |
| 资质词条 | ❌ 不受影响 |
| 成长词条 | ❌ 不受影响 |
| 技能加成 | ❌ 不受影响 |

> 这一设计确保转生后宠物装备的「战力继承」与「资质/成长继承」分离：属性需要人物重新升级解锁，但宠物资质与成长作为长期投入不会因转生而损失。

### 三、装备上锁系统（需求6）

v2.0.5 新增装备**上锁**功能，防止重要装备被误分解，转生后卸下的装备自动上锁。

#### 1. 上锁状态字段

`equip` 对象新增字段：

| 字段 | 类型 | 默认 | 含义 |
|:---|:---|:---:|:---|
| `locked` | boolean | `false` | 是否已上锁 |

#### 2. 切换上锁状态（toggleEquipLock）

`economy.js → toggleEquipLock(equipId)` 切换装备的上锁状态：

1. 在 `G.equipmentBag` 中查找装备；
2. 若未找到，遍历 `G.equipment`（已装备）查找；
3. 切换 `equip.locked` 字段；
4. 调用 `saveGame()` 持久化。

#### 3. 转生后自动上锁

`doRebirth()`（`economy.js`）在转生时：

1. 卸下所有 6 个栏位的装备到 `G.equipmentBag`；
2. **对每件卸下的装备自动设置 `locked = true`**；
3. 退还宝石（见 v2.0.5 需求1）；
4. 重置强化等级。

> 这一机制确保转生后玩家不会因误操作分解原穿戴的高价值装备。

#### 4. 分解保护

##### 4.1 手动分解（decomposeEquipById）

`economy.js → decomposeEquipById(equipId)` 在分解前检查：

```js
if (equip.locked) {
  showToast('该装备已上锁，无法分解！', 'error');
  return;
}
```

##### 4.2 批量分解（batchDecomposeSelected）

`ui.js → batchDecomposeSelected()` 在批量分解时：

1. 遍历选中的装备；
2. **跳过已上锁装备**（不分解）；
3. 提示玩家有多少件装备因上锁被跳过：
   ```
   showToust('已跳过 X 件上锁装备', 'info');
   ```

##### 4.3 自动分解（addEquipmentToBag）

`economy.js → addEquipmentToBag(equip)` 在装备入袋前：

1. 检查 `equip.locked` 字段；
2. **已上锁装备直接入袋，不触发自动分解**；
3. 仅未上锁装备参与 `G.autoDecompose` 的自动分解判定。

#### 5. UI 显示

装备背包（`renderCharacterScreen → equipment` sheet）显示上锁状态：

- 已上锁装备：显示 `🔒` 图标 + 「解锁」按钮
- 未上锁装备：显示 「上锁」按钮
- 上锁按钮位于装备卡片右下角，与「分解」「穿戴」按钮并列

#### 6. 上锁与自动分解的关系

| 装备状态 | 自动分解是否生效 | 手动分解是否生效 | 批量分解是否生效 |
|:---|:---:|:---:|:---:|
| 未上锁 | ✅ 生效 | ✅ 生效 | ✅ 生效 |
| 已上锁 | ❌ 跳过 | ❌ 拒绝 | ❌ 跳过 |

#### 7. 上锁与分解的关系示例

| 场景 | 结果 |
|:---|:---|
| 转生后 100 级橙装被自动上锁 | 自动分解配置（maxLevel=10）即使匹配等级也不会分解 |
| 玩家手动给紫装上锁 | 批量分解时被跳过，提示跳过数量 |
| 玩家解锁后分解 | 正常按品质返还强化石 |

> 上锁系统是 v2.0.5 对装备资产保护的强化，与转生系统联动，确保玩家在转生后不会因自动分解设置丢失原穿戴的高价值装备。
