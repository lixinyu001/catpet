# 血统技能逻辑定义说明书

> 本文档将 `docs/技能.log` 中所有宠物的血统技能 desc（自然语言描述）转化为开发可直接实现的结构化逻辑语言。
>
> **代码定位**：血统效果常量定义于 `src/js/config.js`（`PET_BLOODLINE_EFFECTS` / `PET_BLOODLINE_DEX`）；触发与结算位于 `src/js/battle.js`；技能生成位于 `src/js/config.js`（`generatePetBloodlineSkill`）。
>
> **关联文档**：`04-技能系统.md` §五/§14.2、`01-宠物系统.md` §22.1

---

## 核心摘要

本文档定义了血统技能的**结构化逻辑语言规范（Schema）**，并将技能.log中全部211个宠物的血统技能desc转化为可编程逻辑对象。每个血统技能由两部分组成：①`effects`（被动属性加成，沿用现有`PET_BLOODLINE_EFFECTS`字段体系）；②`mechanics`（触发型特殊效果，新增结构化定义，包含trigger/condition/action/params等字段）。开发者可据此在`battle.js`中实现对应的触发逻辑。

**关键词标签**：#血统技能 #逻辑定义 #mechanics #trigger #condition #action #状态系统

---

## 一、逻辑语言规范（Schema）

### 1.1 顶层结构

每个宠物的血统技能逻辑定义格式如下：

```js
'宠物名': {
  // 被动属性加成（沿用现有 PET_BLOODLINE_EFFECTS 字段体系，经进阶倍率缩放）
  effects: {
    atkPct: 0.10,       // 攻击力+10%
    defPct: 0.10,       // 防御力+10%
    hpPct: 0.10,        // 气血+10%
    spdPct: 0.10,       // 速度+10%
    intPct: 0.10,       // 灵力+10%
    allPct: 0.05,       // 全属性+5%
    critRate: 0.05,     // 暴击率+5%
    critDmg: 0.10,      // 暴击伤害+10%
    dodgeRate: 0.05,    // 闪避率+5%
    dmgReduce: 0.05,    // 伤害减免+5%
    regenPct: 0.05,     // 每回合回血5%
    lifestealPct: 0.08, // 吸血8%
    magicDmgPct: 0.10,  // 法术伤害+10%
    skillTrigger: 0.08, // 技能触发+8%
    healBoost: 0.15,    // 治疗效果+15%
    mpRegenPct: 0.05,   // 魔法回复+5%
    reflectChance: 0.20,// 反弹概率20%
    reflectPct: 0.25,   // 反弹伤害25%
    counterChance: 0.15,// 反击概率15%
    counterPct: 0.50,   // 反击伤害50%
    burnChance: 0.15,   // 灼烧概率15%
    burnTurns: 3,        // 灼烧持续3回合
    burnPct: 0.05,       // 灼烧伤害5%最大气血
    reviveChance: 0.20,  // 复活概率20%
    reviveHpPct: 0.30,   // 复活血量30%
    curseChance: 0.15,   // 诅咒概率15%
    // ... 其他现有字段
  },
  // 触发型特殊效果（新增，battle.js 需实现对应触发逻辑）
  mechanics: [
    {
      id: 'petName_effect1',   // 唯一标识
      trigger: 'onXxx',        // 触发时机（见1.2枚举）
      chance: null,            // 触发概率（null=100%必触发）
      condition: null,         // 前置条件（null=无条件，见1.3枚举）
      target: 'target',        // 目标选择（见1.4枚举）
      action: 'actionType',    // 动作类型（见1.5枚举）
      params: {},              // 动作参数（见1.6按action类型定义）
      cooldown: null,          // 冷却回合数（null=无冷却）
      maxTrigger: null,        // 每场战斗最大触发次数（null=无限）
    }
  ]
}
```

### 1.2 trigger 枚举（触发时机）

| trigger | 含义 | battle.js 插入点 |
|---------|------|------------------|
| `onBattleStart` | 进场时 | `startBattle` 初始化后 |
| `onTurnStart` | 回合开始时 | `executeTurn` 开头 |
| `onTurnEnd` | 回合结束时 | `executeTurn` 末尾 |
| `onAttackHit` | 攻击命中时（普攻+技能） | `executeNormalAttack` / `executeActiveSkill` 命中后 |
| `onPhysicalAttackHit` | 物理攻击命中时 | 同上，`!isMagic` 分支 |
| `onMagicAttackHit` | 法术攻击命中时 | 同上，`isMagic` 分支 |
| `onNormalAttackHit` | 普攻命中时 | `executeNormalAttack` 命中后 |
| `onSkillCast` | 释放技能时 | `executeActiveSkill` 开头 |
| `onHealCast` | 释放治疗时 | `executeActiveSkill` heal分支 |
| `onBuffCast` | 释放增益时 | `executeActiveSkill` buff分支 |
| `onHit` | 受击时 | `executeMonsterAction` 受伤后 |
| `onPhysicalHit` | 受物理攻击时 | 同上，物理伤害分支 |
| `onMagicHit` | 受法术攻击时 | 同上，法术伤害分支 |
| `onMeleeHit` | 受近战攻击时 | 同上，近战判定分支 |
| `onRangedHit` | 受远程攻击时 | 同上，远程判定分支 |
| `onCritReceived` | 被暴击时 | 同上，`isCrit` 分支 |
| `onDodge` | 闪避时 | 闪避判定成功后 |
| `onKill` | 击杀目标时 | 目标HP归零后 |
| `onDeath` | 自身死亡时 | 自身HP归零后 |
| `onFatalDamage` | 受致命伤害时 | 伤害≥当前HP时（死亡前拦截） |
| `onFirstAttack` | 首次攻击时 | 首次攻击标记 |
| `onFirstHit` | 首次受击时 | 首次受击标记 |
| `onStatusTick` | 状态每回合触发时 | `applyTurnStatusEffects` 中 |
| `onStatusApply` | 被附加状态时 | `applyStatus` 函数中 |
| `onAllyDeath` | 友方死亡时 | 友方HP归零后 |
| `onShieldBreak` | 护盾被击破时 | 护盾吸收归零后 |
| `onChargeAttack` | 冲锋攻击时 | 冲锋类技能命中后 |
| `onDiveAttack` | 俯冲攻击时 | 俯冲类技能命中后 |
| `onMove` | 移动时 | 移动逻辑中 |
| `passive` | 被动常驻 | 属性计算时始终生效 |

### 1.3 condition 枚举（前置条件）

| condition | 含义 | 判定表达式 |
|-----------|------|------------|
| `{ selfHpBelow: 0.30 }` | 自身血量低于30% | `petHp.current / petHp.max < 0.30` |
| `{ selfHpAbove: 0.50 }` | 自身血量高于50% | `petHp.current / petHp.max > 0.50` |
| `{ targetHpBelow: 0.50 }` | 目标血量低于50% | `targetHp.current / targetHp.max < 0.50` |
| `{ targetHpAbove: 0.80 }` | 目标血量高于80% | `targetHp.current / targetHp.max > 0.80` |
| `{ targetHpHigherThanSelf: true }` | 目标血量高于自身 | `targetHp.current > petHp.current` |
| `{ hasStatus: 'burn' }` | 自身拥有某状态 | `ms.burning > 0` |
| `{ targetHasStatus: 'freeze' }` | 目标拥有某状态 | `tgtMs.frozen > 0` |
| `{ statusStackGTE: { status:'poison', count:5 } }` | 状态叠层≥N | `tgtMs.poisonStack >= 5` |
| `{ notControlled: true }` | 未被控制 | `!ms.stunned && !ms.frozen && !ms.silenced` |
| `{ isFirstAttack: true }` | 首次攻击 | `!pet._bloodlineFirstAttackDone` |
| `{ isFirstDeath: true }` | 首次死亡 | `!pet._bloodlineFirstDeathDone` |
| `{ isFirstHit: true }` | 首次受击 | `!pet._bloodlineFirstHitDone` |
| `{ isNight: true }` | 夜间 | `battleTime === 'night'` |
| `{ isDay: true }` | 白天 | `battleTime === 'day'` |
| `{ isDusk: true }` | 黄昏 | `battleTime === 'dusk'` |
| `{ terrain: 'forest' }` | 特定地形 | `currentTerrain === 'forest'` |
| `{ terrainIn: ['grass','swamp'] }` | 地形在列表中 | `['grass','swamp'].includes(currentTerrain)` |
| `{ everyNTurns: 3 }` | 每N回合 | `turnNumber % 3 === 0` |
| `{ turnLTE: 2 }` | 回合数≤N | `turnNumber <= 2` |
| `{ turnGTE: 4 }` | 回合数≥N | `turnNumber >= 4` |
| `{ targetRace: 'dragon' }` | 目标种族 | `targetPet.race === 'dragon'` |
| `{ targetAlignment: 'evil' }` | 目标阵营 | `targetPet.alignment === 'evil'` |
| `{ targetIsFlying: false }` | 目标非飞行 | `!targetPet._isFlying` |
| `{ targetSize: 'small' }` | 目标体型 | `targetPet.size === 'small'` |
| `{ dmgType: 'fire' }` | 伤害类型 | `skill.element === 'fire'` |
| `{ isMagic: true }` | 法术攻击 | `isSkillMagic(skill)` |
| `{ isSingleTarget: true }` | 单体攻击 | `!isAOE` |
| `{ isAoeHit: true }` | 范围攻击命中 | `isAOE` |
| `{ notMovedFor: 2 }` | N回合未移动 | `pet._turnsNotMoved >= 2` |
| `{ wasControlledThisTurn: true }` | 本回合受过控制 | `pet._wasControlledThisTurn` |
| `{ shieldExists: 'bloodShield' }` | 某护盾存在 | `ms.shields.bloodShield > 0` |
| `{ totemAlive: true }` | 图腾存活 | `pet._totemHp > 0` |
| `{ phantomAlive: true }` | 幻影存活 | `pet._phantomHp > 0` |
| `{ bothMinionAlive: true }` | 两个分身都存活 | `pet._minion1Hp > 0 && pet._minion2Hp > 0` |
| `{ aliveAllyLTE: 1 }` | 存活友方≤N | `aliveAllyCount <= 1` |
| `{ targetUnaware: true }` | 目标未察觉 | `!targetPet._awareOfAttacker` |
| `{ targetFemale: true }` | 目标为女性 | `targetPet.gender === 'female'` |
| `{ killByMagic: true }` | 法术击杀 | `lastKillByMagic` |
| `{ killByExecute: true }` | 处决击杀 | `lastKillByExecute` |
| `{ purifySuccess: true }` | 净化成功 | `debuffsRemoved > 0` |
| `{ overheal: true }` | 溢出治疗 | `healAmount > hpMissing` |
| `{ isFlying: true }` | 自身飞行中 | `pet._isFlying` |
| `{ inField: 'fire_field' }` | 在某场地效果中 | `activeField === 'fire_field'` |

### 1.6 params 子对象规范（按 action 类型）

#### applyStatus params

```js
{
  status: 'burn',           // 状态名（见1.7状态枚举）
  duration: 3,              // 持续回合（0=永久/跟随状态）
  stack: 1,                 // 本次附加层数
  maxStack: 3,              // 最大叠层
  // --- 状态效果字段（按状态类型不同）---
  dotPct: 0.10,             // DOT伤害%（灼烧/中毒等）
  dotSource: 'lastHitDmg',  // DOT伤害来源：'lastHitDmg'=本次伤害, 'maxHp'=最大气血, 'atk'=攻击力
  spdPct: -0.10,            // 速度增减%
  atkPct: -0.10,            // 攻击力增减%
  defPct: -0.10,            // 防御力增减%
  intPct: -0.10,            // 灵力增减%
  hitRate: -0.10,           // 命中率增减%
  critRate: -0.10,          // 暴击率增减%
  healReduce: 0.50,         // 治疗减免%（焚魂/火焰伤口）
  noAction: true,           // 无法行动（眩晕/冻结/麻痹等）
  noNormalAttack: true,     // 无法普攻（风旋）
  skillRetarget: 'random',  // 技能目标重定向（恶作剧/迷乱）
  attackAllyFirst: true,    // 优先攻击友方（迷魂）
  noCrit: true,             // 无法暴击（不祥）
  undispellable: true,      // 不可驱散
  ignoreImmunity: true,     // 无视免疫（圣洁眩晕）
  defPctPerStack: 0.03,     // 每层防御加成%（岩壳）
  atkPctPerStack: 0.05,     // 每层攻击加成%（炎能/尾巴）
  dmgPctPerStack: 0.03,     // 每层法术伤害加成%（毒素）
  regenPct: 0.20,           // 每回合回血%（蛰伏）
  dmgReduce: 0.80,          // 伤害减免%（蛰伏/金甲）
  absorbPct: 0.05,          // 吸收伤害%=最大气血%（护盾类）
  missChance: 0.50,         // 攻击落空概率%（时空裂隙）
  skillDmgToSelf: 0.50,     // 技能反噬伤害%（虚空裂隙）
}
```

#### dealDamage / dealDamagePct params

```js
{
  dmgPct: 0.20,             // 伤害系数（dealDamage用，基于攻击力/灵力）
  powerAttr: 'atk',         // 威力来源：'atk','int','def','spd','hp','maxHp'
  dmgType: 'lightning',     // 伤害类型/元素
  asTrue: true,             // 真实伤害（无视防御）
  aoe: true,                // 范围伤害
  canChain: true,           // 可连锁触发（灭世炎爆）
}
```

#### addShield params

```js
{
  shieldType: 'moonShield', // 护盾类型
  absorbPct: 0.30,          // 吸收比例
  base: 'healAmount',       // 基准值：'maxHp','healAmount','atk','def'
  duration: 2,              // 持续回合
  noStack: true,            // 不可叠加
  hotPct: 0.03,             // 护盾存在时每回合回血%（祥瑞护盾）
  immuneControl: true,      // 护盾存在时免疫控制（血盾）
  breakEffect: {            // 护盾被击破时触发
    action: 'applyStatus',
    params: { status:'hitRateReduce', hitRate:-0.10, duration:2 }
  }
}
```

#### summon params

```js
{
  summonType: 'phantom',    // 召唤类型：'phantom','miniSlime','totem','remnant','antAlly'
  inheritPct: 0.30,         // 继承属性百分比
  count: 2,                 // 召唤数量
  syncAttack: true,         // 同步普攻
  duration: 3,              // 存在回合数
  aoeDmgPerTurn: { pct:0.30, base:'atk' }, // 每回合AOE伤害（残影）

    action: 'applyStatus',
    params: { status:'dodgeBonus', dodgeRate:0.30, duration:2 },
    target: 'self'
  }
}
```


### 1.7 status 枚举（状态名称）

| 类别 | status | 含义 | 核心效果字段 |
|------|--------|------|-------------|
| **DOT类** | `burn` | 灼烧 | `dotPct, dotSource, duration` |
| | `poison` | 中毒 | `dotPct, dotSource, duration, stack` |
| | `corpsePoison` | 尸毒 | `dotPct, duration, spreadOnDeath:true` |
| | `fireWound` | 火焰伤口 | `healReduce:0.50, duration` |
| | `soulBurn` | 焚魂 | `noHeal:true, duration, undispellable:true` |
| **控制类** | `freeze` | 冻结 | `noAction:true, duration` |
| | `stun` | 眩晕 | `noAction:true, duration` |
| | `paralyze` | 麻痹 | `noAction:true, duration` |
| | `petrify` | 石化 | `noAction:true, defPct:0.30, duration` |
| | `confuse` | 混乱 | `skillRetarget:'random', duration` |
| | `daze` | 迷乱/迷魂 | `skillRetarget:'random'\|attackAllyFirst, duration` |
| | `fear` | 恐惧 | `noAction:true, duration` |
| | `sleep` | 睡眠 | `noAction:true, wakeOnHit:true, duration` |

```js
// 1. 血统机制触发入口（在现有血统效果触发点扩展）
function triggerBloodlineMechanics(pet, triggerType, context) {
  var bloodline = getPetBloodlineSkill(pet);
  if (!bloodline || !bloodline.mechanics) return;
  bloodline.mechanics.forEach(function(m) {
    if (m.trigger !== triggerType) return;
    if (m.cooldown && pet._mechanicCooldowns[m.id] > 0) return;
    if (m.maxTrigger && (pet._mechanicTriggerCount[m.id] || 0) >= m.maxTrigger) return;
    if (m.chance && Math.random() >= m.chance) return;
    if (m.condition && !checkMechanicCondition(pet, m.condition, context)) return;
    executeMechanicAction(pet, m, context);
    if (m.cooldown) pet._mechanicCooldowns[m.id] = m.cooldown;
    if (m.maxTrigger) pet._mechanicTriggerCount[m.id] = (pet._mechanicTriggerCount[m.id] || 0) + 1;
  });
}
```

## 附录C：与现有 PET_BLOODLINE_EFFECTS 的关系

| 项目 | 现有 effects | 新增 mechanics |
|------|-------------|----------------|
| 数据位置 | `config.js` `PET_BLOODLINE_EFFECTS` | `config.js` `PET_BLOODLINE_MECHANICS`（新增） |
| 生效方式 | `generatePetBloodlineSkill` 读取并经进阶倍率缩放 | `battle.js` `triggerBloodlineMechanics` 按trigger触发 |
| 倍率规则 | 数值类放大/概率类不放大/回合类不放大 | mechanics内数值参数同样受倍率规则约束 |
