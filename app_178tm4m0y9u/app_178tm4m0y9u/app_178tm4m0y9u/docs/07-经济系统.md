# 经济系统设定说明书

> 本文档基于 `src/js/config.js`、`src/js/economy.js`、`src/js/ui.js`、`src/js/battle.js` 的实际代码编写，描述梦幻西游风格宠物养成游戏的经济系统完整设定。
>
> **代码定位**：
> - 货币与道具常量、宝箱/血统珠/宠物装备配方常量定义于 `config.js`（`SKILL_QUALITY_PRICES` / `CHEST_RARITIES` / `PET_EQUIP_CRAFT_RECIPES` / `PET_EQUIP_DECOMP_YIELD` / `BLOOD_ORB_RANDOM_QUALITY` / `BLOOD_ORB_DECOMPOSE_RULES` / `ADVANCE_TRIAL`（v2.0.3 新增）等）；
> - 道具/技能书商店、宝石、技能抽取池、市场界面定义于 `ui.js`（`SHOP_ITEMS` / `GEM_TYPES` / `SKILL_BOOK_SHOP` / `SKILL_POOL_CONFIG` / `renderShopScreen` / `renderSkillBookCards` / `switchSkillBookTab` / `renderMarketScreen`）；
> - 强化石合成、宠物装备打造/分解/出售、阵法书与血统珠等逻辑位于 `economy.js`（`composeForgeStone` / `craftPetEquip` / `sellPetEquip` / `decomposePetEquip` / `getItemName` / `useAdvancePill` / `advancePet`（v2.0.3 新增）/ `startAdvanceTrial`（v2.0.3 替换 `startBloodlineTrial`））；
> - 装备打孔/重置孔数位于 `economy.js`（`socketEquipment` / `resetSockets`，v2.0.3 新增）；
> - 战斗掉落与开箱逻辑位于 `battle.js`（`giveMonsterKillRewards` / `dropChests` / `openChestDirect` / `toggleAutoChests`）与 `config.js`（`dropChests` / `generateChestContents` / `maybeDropTalentLoot`）。

## 一、经济系统总览

本作经济系统围绕「金币 + 钻石 + 门票」三重货币体系展开，搭配道具商城、技能书商店、技能抽取池、玩家交易市场、宠物装备打造/分解/出售、宝箱掉落等模块，形成完整闭环：

- **生产**：主线战斗掉落（金币/经验/宝箱/宠物蛋/装备/材料）、副本奖励、活动奖励、市场出售；
- **消耗**：商城购买道具、技能书商店购买、技能池抽卡、宠物装备打造、强化石合成、装备强化、宝箱开启等；
- **转换**：装备分解换强化石、宠物装备分解换材料、强化石合成升级、血统珠分解返还等。

主要数值与常量集中于 `config.js`，主要 UI 与商品列表集中于 `ui.js`，主要经济行为（合成/打造/分解/上架）位于 `economy.js`。

## 二、货币系统

游戏内货币分为「基础货币」「高级货币」与「门票凭证」三类。

### 2.1 金币（gold）

| 字段 | 说明 |
| --- | --- |
| 字段 | `G.player.gold` |
| 用途 | 商城中以 `currency: 'gold'` 标记的道具购买、玩家装备强化、宠物装备打造（配方中的 `gold` 字段）、市场上架/购买宠物蛋、商店批量购买等 |
| 主要来源 | 主线战斗掉落、副本奖励、出售装备/宠物装备、藏宝图奖励、活动奖励、金币箱开启 |
| 增减函数 | `addGold(amount)`（增加）；购买时直接 `G.player.gold -= price` |
| 显示 | 顶部状态栏 `🪙`，商城/市场页头部展示 |

主线每只怪物掉落金币公式（`battle.js → giveMonsterKillRewards`）：

```
goldGain = floor(monster.level * 5 * (1 + G.player.rebirth * 0.1))
```

### 2.2 钻石（diamond）

| 字段 | 说明 |
| --- | --- |
| 字段 | `G.player.diamond` |
| 用途 | 商城中以 `currency: 'diamond'` 标记的道具（经验书、宝石、稀有蛋、双倍卡、归虚丹、金币箱等）、技能书商店购买（全部钻石定价）、技能抽取池抽卡、激活高级战令（50 钻） |
| 主要来源 | 每日任务奖励、宝箱开启（紫/橙宝箱）、藏宝图、活动奖励、竞技场奖励 |
| 增减函数 | 直接 `G.player.diamond += amount` / `G.player.diamond -= price` |
| 显示 | 顶部状态栏 `💎` |

钻石掉落受天赋加成（`diamond_find` + `diamond_mastery` + `loot_mastery`）：

```
actualDiamond = floor(amount * (1 + diamond_find + diamond_mastery + loot_mastery))
```

## 三、商城系统（renderShopScreen）

### 3.1 商城结构

商城页面（`renderShopScreen`，`ui.js:1903`）包含两大模块：

1. **📦 道具与宠物蛋**：分类筛选 + 道具网格，分页（每页 12 项，复用通用分页 `paginateList('shop', ...)`）；
2. **📖 技能书商店**：7 个分类标签 + 技能书网格，分页（每页 8 项，独立分页 `window._skillBookPage`）。

页面顶部展示当前金币 `🪙` 与钻石 `💎`，便于购买决策。

### 3.2 道具分类筛选（8 类）

道具与宠物蛋模块提供 8 个筛选标签（v2.2.0 重构），状态保存在 `window._shopItemFilter`：

| 筛选 ID | 标签 | 命中规则（SHOP_ITEMS.filter） |
| --- | --- | --- |
| `all` | 全部 | 全部商品 |
| `ticket` | 门票 | `ticketIds` 数组中的7种副本门票 |
| `forge` | 强化 | `forgeIds` 数组中的7种强化相关道具（强化石/保护石/打孔钉/修补胶/洗练石） |
| `pet` | 宠物 | `petIds` 数组中的14种宠物相关道具（孵化/融合/月华露/元宵/进阶丸等） |
| `exp` | 经验 | `expIds` 数组中的11种经验/金币加成道具 |
| `gem` | 宝石 | `item.action === 'gem'` 或 `item.id` 前缀为 `gem_` |
| `gold` | 金币 | `goldIds` 数组中的3种金币宝箱 |
| `other` | 其他 | 不属于上述任一分类的剩余道具（含挖密藏道具 dig_map/dig_shovel/dig_lens/dig_key） |

切换筛选按钮时调用 `window._shopItemFilter = '<id>'; resetPage('shop'); renderShopOnly();`，即同步重置道具分页到第 1 页。

### 3.3 window._shopItemFilter 状态

- 全局变量：`window._shopItemFilter`，默认 `'all'`；
- 在 `renderShopScreen` 内部首帧时若未定义则初始化为 `'all'`；
- 每次切换会触发 `resetPage('shop')`（道具分页重置为 1）后调用 `renderShopOnly()` 局部重渲染；
- 状态在页面切换/刷新后保留（除非刷新整个浏览器）。

### 3.5 购买流程与 buyShopItem

购买入口：`buyShopItem(itemId, price, action, tier, currency)`（`ui.js:6712`）

1. 读取 `getShopQty(itemId)` 数量（1~99）；
2. 校验 `G.player.diamond`/`G.player.gold` 是否足够 `price * qty`；
3. 扣费后按 `action` 分发：
   - `egg`：生成 `qty` 个 `generateEgg(tier)` 推入 `G.eggs`；
   - `item`：根据 ID 走特例（`hatch_stone` → `G.hatchStones`，`exp_book` 系列 → 直接 `addExp(...)`），其余推入 `G.inventory`；
   - `skill_random`：随机 `qty` 本技能书推入 `G.skillBooks`；
   - `yuanxiao`：元宵直接入背包；
   - `buff`：调用 `activateBuff(buffType, buffMult, buffDuration)`；
   - `gold_chest`：直接 `addGold(goldAmount * qty)`；
   - `gem`：调用 `addGemToBag(gemType, gemLevel, qty)`；
   - `rare_egg`：生成 `qty` 个 T2-T4 蛋；
   - `rename_card`：入背包；
   - `exp_book_bulk`：每包给 5 本 `exp_book`。

购买数量通过 `+/-` 按钮调节，调用 `setShopQty` 限制在 1~99。

## 八、强化石合成（composeForgeStone）

### 8.1 合成规则

`economy.js:80 → composeForgeStone(fromTier, count)`：

## 九、宠物装备打造系统（craftPetEquip）

### 9.1 四品质打造配方（PET_EQUIP_CRAFT_RECIPES）

`config.js:1882 → PET_EQUIP_CRAFT_RECIPES`，所有配方均消耗「神秘水晶 + 远古符文 + 金币」（不含兽皮，兽皮仅用于装备分解产出）：

| 目标品质 | 神秘水晶 mystic_crystal | 远古符文 ancient_rune | 金币 gold |
| --- | --- | --- | --- |
| rare（稀有） | 1 | 0 | 5,000 |
| epic（史诗） | 1 | 1 | 20,000 |
| legend（传说） | 1 | 3 | 80,000 |
| mythic（神话） | 1 | 8 | 300,000 |

打造入口 `craftPetEquip(rarity)`：

1. 校验材料与金币；
2. 扣除材料（`G.petEquipMaterials[mat] -= recipe[mat]`）和金币（`G.player.gold -= recipe.gold`）；
3. 调用 `rollCraftRarity(rarity)` 决定实际产出品质；
4. 随机选栏位（attack/hp/defense）调用 `generatePetEquip(outputRarity, slot, G.player.level)` 生成装备；
5. 推入 `G.petEquipBag`。

| 配方品质 | rare | epic | legend | mythic |
| --- | --- | --- | --- | --- |
| rare 配方 | 100% | — | — | — |
| epic 配方 | 20% | 80% | — | — |
| legend 配方 | — | 25% | 70% | **5%**（爆种） |
| mythic 配方 | — | 10% | 30% | 60% |

`rollCraftRarity(recipeRarity)` 累加权重并 `Math.random()` 抽取，是高品质宠物装备的核心获取途径之一。

### 9.3 出售价格（sellPetEquip）

### 10.3 宝箱掉落概率（dropChests / getChestDropRate）

`config.js:1543 → getChestDropRate(enemyType)`：

```

| 怪物类型 | 基础掉宝率 | 掉落宝箱品质 |
| --- | --- | --- |

### 10.4 自动开启设置（toggleAutoChests）

`battle.js:1991 → toggleAutoChests()`：

- 状态字段：`G.autoOpenChests`（boolean）；
- 默认值：`false`（手动开箱）；
- 开启后：战斗中 `dropChests` 掉落的宝箱**立即调用 `openChestDirect(c)` 开启**，不进入 `G.chests`；
- 关闭后：宝箱推入 `G.chests`，需在主页"未开宝箱"区域手动点击或「一键开启」。

主页底部有切换按钮，文本随状态切换为「📦 自动开箱」或「🎒 手动开箱」。

未开宝箱区域提供「一键开启」按钮（`openAllChests()`）：循环 `openChestDirect` 直至 `G.chests` 清空。手动点击单个宝箱调用 `openChestFromBag(chestId)`，会触发转盘动画（`showChestRoulette`）。

## 十二、战斗掉落系统

### 12.1 主线掉落（giveMonsterKillRewards）

`battle.js:254 → giveMonsterKillRewards(monster)` 处理每只怪物击杀的奖励发放：

| 奖励项 | 公式 | 日志类型 |
| --- | --- | --- |
| 经验 | `expGain = max(5, (monster.level × 8 + lvDiff × 5) × (1 + rebirth × 0.2))` | `⭐ 获得 X 经验` |
| 金币 | `goldGain = floor(monster.level × 5 × (1 + rebirth × 0.1))` | `🪙 获得 X 金币` |
| 宝箱 | `dropChests(enemyType)`，按 mob/elite/boss 概率 | `⬜/🟩/🟦/🟪/🟧 掉落了 X 宝箱！` |
| 天赋额外掉落 | `maybeDropTalentLoot(enemyType)` | 装备/打孔钉/修补胶/宝石（v2.0.3 替换原融合石/孵化石） |
| 战令经验 | `G.player.battlePassExp += 10` | （无日志，每 1000 升 1 级） |

其中 `lvDiff = monster.level - G.player.level`，鼓励挑战高等级怪物。同一怪物不会重复给奖励（`monster._rewarded` 去重）。

宝箱掉落后根据 `G.autoOpenChests` 决定是否立即开启：

```
if (G.autoOpenChests) droppedChests.forEach(c => openChestDirect(c));
else G.chests.push(...droppedChests);
```

### 12.2 天赋额外掉落（maybeDropTalentLoot）

`config.js:1594 → maybeDropTalentLoot(enemyType)` 在每场战斗后触发：

> **v2.0.3 变更**：原 `fusion_stone`（融合石）与 `hatch_stone`（孵化石）掉落已整体移除，替换为新道具：精英怪（elite）掉落 `socket_nail`（打孔钉）、Boss 掉落 `repair_glue`（修补胶），用于装备打孔系统。

| 掉落类型 | 概率（含天赋加成） | 数量 |
| --- | --- | --- |
| 装备 | `equip_drop + loot_mastery × 0.5`（boss +30%，elite +10%） | 1 件，rarity 随机 |
| 打孔钉 socket_nail | `socket_drop + loot_mastery × 0.5`（elite +20%） | elite 1~2 / mob 1（v2.0.3 替换融合石，仅 elite/mob 掉落） |
| 修补胶 repair_glue | `repair_drop + loot_mastery × 0.5`（boss +40%） | boss 1~2（v2.0.3 替换孵化石，仅 boss 掉落） |
| 宝石 | `gem_drop_rate × 0.3`（boss +8%，elite +3%） | 1 颗，boss 有 15% 概率为 2 级 |

装备 rarity 随机规则：5% orange → 20% purple → 50% blue → 25% green（即橙色 5%、紫色 19%、蓝色 38%、绿色 38%）。

> 原融合石/孵化石仍可在商城购买（融合石 500 金币、孵化石 800 金币），但不再从主线战斗掉落。融合/孵化功能仍正常消耗对应道具。

### 12.3 藏宝图掉落（maybeDropTreasureMap）

`config.js:1575 → maybeDropTreasureMap()`：仅当 `G.player.level >= 60` 触发，掉宝率受天赋 `map_drop` + `loot_mastery` 影响：

roll < 0.25 × (1 + mapBonus) → green 藏宝图
roll < 0.40 × (1 + mapBonus) → blue 藏宝图
```

即整体 40% 概率掉落一张藏宝图，写日志 `🗺️ 获得 X 藏宝图！`。

### 一、装备上锁系统对分解的影响（需求6）

  }
  
  // ... 原有分解逻辑 ...

##### 1.2 批量分解（batchDecomposeSelected）

新增上锁跳过逻辑：

```js
function batchDecomposeSelected() {
  var skipped = 0;
  var toDecompose = [];
  selectedEquips.forEach(equipId => {
    var equip = G.equipmentBag.find(e => e.id === equipId);
    if (equip && equip.locked) {
      skipped++;
    } else {
      toDecompose.push(equipId);
    }
  });
  
  if (skipped > 0) {
    showToast(`已跳过 ${skipped} 件上锁装备`, 'info');
  }
  
  // 仅分解未上锁装备
  toDecompose.forEach(id => decomposeEquipById(id));
}
```

#### 2. 上锁与分解的完整关系表

| 装备状态 | 自动分解 | 手动分解 | 批量分解 | 出售（如适用） |
|:---|:---:|:---:|:---:|:---:|
| 未上锁 + 满足自动分解条件 | ✅ 分解 | ✅ 分解 | ✅ 分解 | ✅ 出售 |
| 未上锁 + 不满足自动分解条件 | ❌ 入袋 | ✅ 分解 | ✅ 分解 | ✅ 出售 |
| 已上锁 | ❌ 入袋 | ❌ 拒绝 | ❌ 跳过 | ✅ 出售 |

#### 3. 转生后自动上锁机制

`doRebirth()`（`economy.js`）在转生时：

1. 卸下所有 6 个栏位的装备到 `G.equipmentBag`；
2. **对每件卸下的装备自动设置 `locked = true`**；
3. 退还宝石（见 v2.0.5 需求1）；
4. 重置强化等级。

| 场景 | 上锁状态 | 后续行为 |
|:---|:---|:---|
| 玩家穿戴 100 级橙装 → 转生 | 卸下后自动上锁 | 不会被自动分解 |
| 玩家手动给紫装上锁 | 上锁 | 无法手动分解，需先解锁 |
| 玩家解锁装备 | 未上锁 | 可正常分解 |

#### 4. 强化石产出影响

由于转生后高等级装备自动上锁，**自动分解产出的强化石数量**会减少：

| 项目 | v2.0.5 前 | v2.0.5 后 |
|:---|:---|:---|
| 转生后自动分解触发 | 转生后卸下的高等级装备可能被自动分解 | **不再被自动分解**（已上锁） |
| 玩家强化石获取 | 转生后获得一波强化石 | **需手动解锁后才能分解**，强化石获取延后 |

> 这一调整降低了转生后玩家误操作损失高价值装备的风险，但同时也要求玩家主动管理装备库存（解锁 → 分解）以获取强化石。

### 15.2 离线挂机收益

#### 计算规则

| 参数 | 值 | 说明 |

| 每分钟蛋掉落 | 0.1 个 | floor(0.1 × 分钟数) |

#### 核心函数

| 函数 | 说明 |
|:---|:---|

### 三、宠物品质判断调整（需求11）

v2.0.5 调整宠物品质分数区间，使史诗品质更容易达到，神话品质更稀有。

#### 1. recalcRarity 品质分数区间

`pet.js → recalcRarity(pet, qualityBoost)` 中的品质分数区间调整：

| 品质 | v2.0.5 前 | v2.0.5 后 | 调整 |
|:---|:---|:---|:---|
| 史诗 Epic | 0.85 ~ 0.95 | **0.85 ~ 0.90** | 区间缩小（更易达到） |
| 传说 Legend | 0.95 ~ 0.99 | **0.90 ~ 0.95** | 区间下移（更易达到） |
| 神话 Mythic | 0.99+ | **0.95+** | 阈值降低（更易达到） |

> 注：实际表现为史诗品质更容易达到，神话更稀有——这是因为调整后史诗的区间从 0.10 缩小到 0.05，传说从 0.04 扩大到 0.05，神话的阈值从 0.99 降到 0.95。

#### 2. 品质分数计算

品质分数 = `(sumAptitudeScore + growthScore + bonusScore) / maxScore`

| 分数组成 | 说明 |
|:---|:---|
| `sumAptitudeScore` | 4 项资质得分之和 |
| `growthScore` | 成长得分 |
| `bonusScore` | 加成分（如血统珠、装备等） |
| `maxScore` | 最大可能分数（用于归一化） |

#### 3. 实际影响

| 项目 | v2.0.5 前 | v2.0.5 后 |
|:---|:---|:---|
| 普通 Common | < 0.85 | < 0.85（不变） |
| 稀有 Rare | 0.85 ~ 0.95 | 0.85 ~ 0.90（不变，0.85 起算） |
| 史诗 Epic | 0.85 ~ 0.95 | **0.85 ~ 0.90**（区间缩小） |
| 传说 Legend | 0.95 ~ 0.99 | **0.90 ~ 0.95**（下移） |
| 神话 Mythic | 0.99+ | **0.95+**（阈值降低） |

> 这一调整使宠物品质分布更合理，玩家更容易获得传说品质宠物，但神话品质仍保持稀有性。
