### 六、挖密藏九宫格系统（`economy.js` / `ui.js` / `config.js` / `state.js` / `index.html`）

#### 核心配置（`economy.js`）
- `DIG_DAILY_LIMIT = 10`：每日最多10次
- `DIG_INITIAL_DIGS = 4`：初始4次挖掘机会
- `DIG_CELL_TYPES`：8种格子类型（空地/金币/宝石/道具/陷阱/银宝箱/锁宝箱/黄金宝藏），按权重随机生成

#### 核心逻辑（`economy.js`）
- `generateDigGrid()`：生成3×3九宫格，确保至少有1个非空格子
- `startDigSession()`：功能等级检查 → 每日次数检查 → 消耗密藏图 → 初始化会话 → 自动导航到挖密藏页面
- `digCell(idx)`：挖掘格子，锁宝箱需消耗钥匙，陷阱额外损失1次挖掘
- `processDigReward(cellType)`：按格子类型发放奖励
  - 空地：无奖励
  - 金币：`playerLevel × random(50,200)`
  - 宝石：随机宝石（15%概率2级）
  - 道具：强化石/孵化石/打孔钉/元宵等随机
  - 陷阱：无奖励 + 额外损失1次挖掘
  - 银宝箱：钻石1~3 + 金币
  - 锁宝箱：钻石5~10 + T3~T4宠物蛋（需钥匙）
  - 黄金宝藏：钻石10~20 + T4~T5宠物蛋 + 月华露×2
- `useDigLens(idx)`：透视格子，不消耗挖掘次数
- `useDigShovel()`：增加1次挖掘机会（上限9次）
- `endDigSession()`：结束会话，显示收获总结

#### UI（`ui.js`）
- 导航栏新增"⛏️ 挖密藏"入口
- `renderDigScreen()`：渲染挖密藏界面（入口页/进行中页）

- `selectDigCell(idx)`：选中/取消选中格子（用于透视功能）
- 背包界面密藏图添加"🗺️ 开始挖宝"按钮

#### CSS（`index.html`）

- `.dig-cell` / `.dig-cell-hidden` / `.dig-cell-peeked` / `.dig-cell-revealed` / `.dig-cell-selected`：格子状态样式

### 七、藏宝图新词缀扩展（`config.js` / `battle.js` / `ui.js`）

#### 新增普通词缀（`TREASURE_AFFIXES`）
- `gem_drop`：宝石掉落率 +
- `pet_exp_bonus`：宠物经验加成 +
- `forge_stone_drop`：强化石掉落率 +
- `dig_item_drop`：密藏道具掉落 +

#### 新增特殊词缀（`TREASURE_SPECIAL_AFFIXES`）
- `triple_reward`：所有奖励三倍！！
- `guaranteed_dig_map`：必定掉落一张密藏图

#### 词缀处理逻辑（`battle.js` `processTreasureRewards`）
- 宝石掉落：`Math.random() < gemDrop` 时掉落随机宝石
- 宠物经验：`petExpBonus > 0` 时队伍宠物获得额外经验
- 强化石掉落：`Math.random() < forgeStoneDrop` 时掉落1~3颗强化石
- 密藏道具掉落：`Math.random() < digItemDrop` 或 `guaranteed_dig_map` 时掉落密藏道具
- 三倍奖励：`mult = isTriple ? 3 : (isDouble ? 2 : 1)` 应用到金币/经验/钻石/强化石

#### 紫色藏宝图特殊词缀（`ui.js` `generateTreasureMap`）
- 紫色品质藏宝图30%概率获得特殊词缀（不含三倍奖励）
- 橙色品质藏宝图100%获得特殊词缀（含三倍奖励）

### 二、宠物派遣奇遇系统

#### 核心机制
- **闲置宠物派遣**：派遣至已通关的地图区域探索，最多 3 只组队
- **战力门槛**：派遣宠物总战斗力需达到地图 `minPower` 标准才能开始
- **战力加成**：超出最低战力每 10% 额外 +5% 奖励倍率（封顶 ×3.0）
- **探索时长**：可选 1/4/8 小时（倍率 1.0/4.5/10.0）
- **基础产出**：金币、经验、养成材料；低概率（5%×战力倍率）掉落稀有道具

#### 配置（`config.js`）
- `DISPATCH_DURATIONS`：3 种时长与倍率
- `DISPATCH_MAPS`：11 个地图派遣区域，按 `MAPS` 顺序配置
  - mapId 1 起源草地（minPower 500）→ mapId 11 终焉神殿（minPower 250,000）
- `DISPATCH_BASE_REWARD`：基础金币 1000 / 经验 500 / 材料 2 个 / 稀有概率 5%
- `DISPATCH_POWER_BONUS_PER_10PCT = 0.05`
- `DISPATCH_MAX_SLOTS = 3`：同时派遣上限

#### 状态字段（`state.js`）
- `G.dispatches`：当前进行中的派遣列表
- `G.dispatchHistory`：已领取的派遣历史（保留最近 50 条）
- `G.dispatchDailyUsed`：每日派遣计数（仅记录，不硬性限制）
- 旧存档自动迁移

#### 核心逻辑（`economy.js`）
- `getDispatchMapConfig(mapId)`：获取地图配置
- `isMapUnlockedForDispatch(mapId)`：地图是否已通关（`mapProgress[mapId].phase === 'cleared'`）
- `getAvailableDispatchPets()`：可派遣宠物（不在出战阵容、不在其它派遣中）
- `getDispatchTotalPower(petIds)`：总战斗力（`getPetCombatPower` 求和）
- `getDispatchPowerBonus(total, min)`：战力超额加成倍率
- `getDispatchProgress(dispatch)`：进度 0~1（基于真实时间）
- `getDispatchExpectedReward(dispatch)`：预计奖励预览
- `startDispatch(mapId, petIds, durationIdx)`：开始派遣（含完整校验）
- `collectDispatch(dispatchId)`：领取奖励（完成时）
- `recallDispatch(dispatchId)`：强制召回（无奖励）
- `applyDispatchRareDrop(rareId)`：稀有掉落应用（rare_egg/blood_orb_*/skill_random）

#### UI（`ui.js`）
- 活动页新增 `dispatch` tab：`🎒 派遣奇遇`
- `renderActivityDispatch()` 渲染函数：
  - 进行中的派遣列表（进度条、预计完成时间、领取/召回按钮）
  - 新建派遣面板（地图选择、宠物多选、时长选择、战力检查、奖励预览）
  - 派遣历史（近 10 条）
  - 派遣收获日志
- 交互函数：`_dispatchPickMap`、`_dispatchPickDuration`、`_dispatchTogglePet`、`_dispatchStart`、`_collectDispatch`、`_recallDispatch`

#### 日常任务（`config.js`）
- 新增 `dispatch_finish` 任务：完成 1 次派遣奇遇，奖励 20 钻石

### 三、平衡说明
- 派遣系统天然受限于：3 并发槽位 + 1/4/8 小时时长，不设硬性每日次数限制
- 稀有掉落概率随战力加成提升，但封顶 50%
- 派遣宠物可获得 30% 经验奖励，鼓励派遣主力外的宠物
